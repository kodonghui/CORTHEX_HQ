# CORTHEX 투자분석 품질 채점 — 통합 프롬프트

> **용도 A — 샘플 1개 중간점검**: 특정 모델의 분석 결과를 채점 후 시스템 개선 방향 도출
> **용도 B — 3라운드 블라인드 비교**: 모델 변경 후 3회 실행, 라운드별 순위 산정
>
> 프롬프트 상단에 "중간점검" 또는 "블라인드 비교" 중 어떤 용도인지 명시할 것.

---

## 시스템 배경

CORTHEX HQ는 AI 투자참모 시스템입니다.

```
사용자 요청 → CIO(투자최고책임자)
                 ├─ CIO 독자분석 (5번째 분석가 역할)
                 ├─ 시황분석 전문가 (MC) — 거시경제, FILM 프레임워크
                 ├─ 종목분석 전문가 (SA) — 재무제표, DCF, 밸류에이션
                 ├─ 기술분석 전문가 (TA) — 차트, RSI, MACD, 볼린저밴드
                 └─ 리스크관리 전문가 (RM) — VaR, MDD, 시나리오 분석

CIO + 전문가 4명 = 총 5명이 병렬 분석 → CIO가 종합 보고서 작성
전문가끼리 cross_agent_protocol로 서로 스폰(호출) 가능
검수 통과 못하면 → 반려 → 전문가 재작업 → 재보고
```

각 전문가와 CIO는 **도구(tool)**를 호출해서 실시간 데이터를 가져옵니다.
도구 호출 내역은 **활동 로그**에 기록됩니다.

### 데이터 2: 활동 로그 (도구 호출 기록)

> ⚠️ **로그 제약**: 활동 로그 API는 **최신 100건만** 반환합니다.
> 100건 초과 시 초반 호출이 잘릴 수 있습니다.
> - 로그에 나타난 호출만으로 중복률 계산, "실제 총 호출은 이보다 많을 수 있음" 명시

## 채점 절차

### Step 1: 보고서 품질 채점 (Q0~Q9, 전문가 4명 각각)

전문가별로 아래 10개 차원을 **1/3/5 점수**로 채점합니다.
**반드시 1, 3, 5 중 하나만** 부여. 2, 4 같은 중간값 금지.

| 차원 | 가중치 | 핵심 질문 |
|------|--------|-----------|
| Q0. 전체 만족도 | 별도 | "이 보고서만 보고 매매 결정 가능?" |
| ★Q1. 도구활용/정확성 | 20% | 도구 3개+ 사용? 수치 정확? (★1점이면 Q0 최대 2점) |
| Q2. 데이터 근거성 | 15% | 모든 판단이 도구 데이터에 매핑? |
| Q3. 수치 정밀성 | 15% | 단위/시점 명확? 구체적 숫자 10개+? |
| Q4. 분석 논리 | 10% | 데이터→해석→결론 재현 가능? |
| Q5. 분석 완결성 | 10% | 필수 영역 빠짐없이, 균일한 깊이? |
| Q6. 리스크/반대논거 | 15% | 리스크를 수치로? "확실히/반드시" 0건? |
| Q7. 전문적 통찰 | 5% | 프레임워크를 실제 데이터에 적용? |
| Q8. 가독성 | 5% | BLUF + 표/목록? 반복 최소? |
| Q9. 형식 준수 | 5% | CIO 보고 형식 정확히 준수? |

### 채점 기준 상세

#### 【Q0. 전체 만족도】 가중치: 별도 독립 평가
> "이 보고서만 보고 바로 매매 결정을 내릴 수 있는가?"

| 점수 | 기준 |
|------|------|
| **1** | 매매 결정 불가. 도구 미사용, 분석 구조 없음, 또는 할루시네이션으로 신뢰 불가 |
| **3** | 참고는 되지만 단독 결정 근거로 부족. 방향성은 잡히지만 빠진 분석/논리 비약 존재 |
| **5** | 무편집 즉시 활용 가능. 데이터 충분, 논리 명확, 리스크 명시, 실행 전략 구체적 |

#### 【★Q1. 도구 활용 & 데이터 정확성】 가중치: 20%, critical
> Q1이 1점이면 Q0 최대 2점 제한.

| 점수 | 기준 |
|------|------|
| **1** | 도구 전혀 사용 안 함(0회), 또는 존재하지 않는 수치를 지어냄 |
| **3** | 핵심 도구 1~2개 사용. 절반 이상 미사용 또는 도구 데이터 해석 일부 부정확 |
| **5** | 핵심 도구 3개+ 체계적 사용. 모든 수치가 도구 결과와 일치 |

#### 【Q2. 데이터 근거성】 가중치: 15%
| 점수 | 기준 |
|------|------|
| **1** | 도구 밖 상식/추측을 핵심 판정 근거로 사용 ("일반적으로 반도체는...") |
| **3** | 주된 근거는 도구 데이터이나 핵심 판단 1~2개가 도구 밖 상식에 기댐 |
| **5** | 모든 핵심 판단이 도구 데이터에 직접 매핑. 추적 가능한 형태 |

#### 【Q3. 수치 정밀성】 가중치: 15%
| 점수 | 기준 |
|------|------|
| **1** | 수치 오류 2건+. 단위 혼동, 시점 오류. 또는 구체적 숫자 3개 미만 |
| **3** | 큰 오류 없으나 시점 모호한 곳 1~2개. 구체적 숫자 5~10개 |
| **5** | 모든 수치에 단위/시점 명확. 구체적 숫자 10개+, 모든 핵심 수치에 출처/시점 명시 |

#### 【Q4. 분석 논리】 가중치: 10%
| 점수 | 기준 |
|------|------|
| **1** | 왜 그렇게 판단했는지 재현 불가. 데이터 나열만. 또는 데이터↔결론 모순 |
| **3** | 흐름은 있으나 중간 연결 1~2곳 생략/비약 |
| **5** | 데이터→해석→시사점→결론이 단계별로 선명. 독자가 같은 결론 도달 가능 |

#### 【Q5. 분석 완결성】 가중치: 10%
| 점수 | 기준 |
|------|------|
| **1** | 핵심 분석 영역의 절반 이하만 다룸 |
| **3** | 대부분 다루나 일부가 매우 피상적. 균형 안 맞음 |
| **5** | 분석 영역을 빠짐없이, 각 항목 깊이 일관됨 |

#### 【Q6. 리스크/반대논거 구체성】 가중치: 15%
| 점수 | 기준 |
|------|------|
| **1** | 리스크 언급 없거나 "주의 필요" 수준. "확실히/반드시" 같은 확언 |
| **3** | 리스크 언급하나 수치 없이 정성적. "하락할 수 있다" (얼마나?) |
| **5** | 리스크 각각을 구체적 수치로 제시(MDD, 손절가 등). 매수/리스크 비율 6:4~7:3. "확실히/반드시" 0건 |

#### 【Q7. 전문적 통찰】 가중치: 5%
| 점수 | 기준 |
|------|------|
| **1** | 검색 결과 요약 수준. 전문 프레임워크 미사용 |
| **3** | 프레임워크 이름만 언급. 실제 데이터에 적용 안 함 |
| **5** | 전문 프레임워크를 실제 데이터에 적용하여 구체적 수치 도출 |

#### 【Q8. 가독성/간결성】 가중치: 5%
| 점수 | 기준 |
|------|------|
| **1** | 문장 과밀/중복/반복. 읽는 비용이 큼 |
| **3** | 읽히지만 일부 장황/반복 |
| **5** | 정보 밀도 높고 반복 최소. BLUF + 표/목록 적절 활용 |

#### 【Q9. CIO 보고 형식 준수】 가중치: 5%
| 점수 | 기준 |
|------|------|
| **1** | 보고 형식 전혀 따르지 않음. 필수 섹션 대부분 누락 |
| **3** | 큰 틀은 맞으나 필수 항목 1~2개 누락 |
| **5** | 보고 형식 정확히 준수. 모든 필수 섹션 포함 |

### 전문가 유형별 보너스 (±0.5점 보정)

| 전문가 | +0.5 조건 | -0.5 조건 |
|--------|-----------|-----------|
| 시황분석 | FILM 4항목 수치화 / 거시지표 3개+ / CEO 비유 | FILM 미사용 / 거시 1개↓ / 비유 없음 |
| 종목분석 | DCF 실제 계산 / 피어 2개+ 비교 / 재무 3개+ 수치 | 밸류에이션 없이 "저평가" / 단독 / 재무 없음 |
| 기술분석 | 지표 2개+ 수치 / 진입가 구체적 / 거래량 분석 | 지표 0개 / "적절한 시점" / 거래량 없음 |
| 리스크관리 | VaR/MDD 등 2개+ / 3시나리오 수치 / 포트폴리오 관점 | "리스크 있다" / 시나리오 없음 / 개별만 |

**계산**:
```
가중 평균 = (Q1×20 + Q2×15 + Q3×15 + Q4×10 + Q5×10 + Q6×15 + Q7×5 + Q8×5 + Q9×5) / 100
최종 점수 = clip(가중 평균 + Σ보너스, 1.0, 5.0)
```

### Step 3: 효율성 채점 (E1~E5) — 활동 로그 필수

| 차원 | 가중치 | 데이터 | 핵심 질문 |
|------|--------|--------|-----------|
| E1. 전문가 간 중복 위임 | 30% | 보고서+로그 | 서로 다른 전문가가 동일 도구+파라미터 호출? |
| E2. CIO 독자분석 중복 | 25% | 보고서+로그 | CIO 도구 호출 중 전문가와 겹치는 비율? |
| E3. 도구 호출 중복률 | 20% | 로그 | 전체 파이프라인에서 동일 호출 반복 비율? |
| E4. 불필요한 전문가 동원 | 15% | 보고서+로그 | 도구 0~1회 사용 + 실질 기여 없는 전문가? |
| E5. 교차 스폰 순환/낭비 | 10% | 로그 | A→B→A 순환? 스폰 결과 실질 반영? |

#### E1 기준
| 점수 | 기준 |
|------|------|
| **1** | 동일 분석 중복 지시 3건+. 로그: 서로 다른 전문가가 동일 도구+파라미터 5건+ |
| **3** | 중복 지시 1~2건. 경계 영역 의도치 않은 중복. 로그: 교차 호출 2~4건 |
| **5** | 중복 지시 0건. 각 전문가 고유 영역만 지시. 로그: 교차 호출 0~1건 |

#### E2 기준
| 점수 | 기준 |
|------|------|
| **1** | CIO 독자분석 70%+가 전문가와 동일. 비용만 2배, 새 정보 0 |
| **3** | CIO 독자분석 30~70%가 전문가와 중복 |
| **5** | CIO 독자분석이 전문가와 실질 0% 중복. 로그: 중복률 10% 미만 |

#### E3 기준 (로그에서 직접 계산)
```
1. 로그에서 모든 (도구명, 핵심파라미터) 쌍을 추출
2. 동일 쌍이 2회+ 나타나면 "중복"
   - 시각/포맷 차이는 무시
   - 같은 전문가가 다른 파라미터로 호출 = 중복 아님
3. 중복률 = 중복 호출 수 / 전체 호출 수 × 100%
```

| 점수 | 기준 |
|------|------|
| **1** | 전체 중복률 30%+. 동일 종목+도구를 3명+ 전문가가 각각 호출 |
| **3** | 중복률 10~30%. 핵심 데이터 호출 다소 겹침 |
| **5** | 중복률 10% 미만. 각 전문가 도구 호출이 자기 영역에 특화 |

#### E4 기준
| 점수 | 기준 |
|------|------|
| **1** | 2명+ 전문가가 실질 기여 없이 동원. 로그: 도구 0~1회 전문가 2명+ |
| **3** | 1명 기여 미미. 다른 전문가 내용의 하위호환. 로그: 도구 2회 이하 1명 |
| **5** | 4명 전원 고유 가치 기여. 한 명 빼면 전체 분석에 공백 발생. 로그: 전원 도구 3회+ |

#### E5 기준
| 점수 | 기준 |
|------|------|
| **1** | A→B→A 순환 발생. 또는 교차 스폰 50%+가 새 정보 없이 토큰만 소모 |
| **3** | 순환 없으나 교차 스폰 1~2건이 불필요 |
| **5** | 모든 교차 스폰이 명확한 필요에 의한 것. 또는 교차 스폰 0건 |

**경고 규칙**:
- E3이 1점 → 🚨 비용 경고
- E1+E2 합산 ≤ 4점 → ⚠️ 구조적 중복 경고

**계산**: `효율성 점수 = E1×30% + E2×25% + E3×20% + E4×15% + E5×10%`

---

### Step 4: 최종 점수 계산

```
품질 점수 = 전문가 평균 × 70% + CIO 종합 × 30%
라운드 최종 점수 = 품질 점수 × 75% + 효율성 점수 × 25%
```

### 판정 기준

| 등급 | 점수 | 판정 |
|------|------|------|
| **S** | 4.5~5.0 | 탁월 — 전문 애널리스트급, 즉시 매매 결정 가능 |
| **A** | 3.5~4.4 | 우수 — 실전 투입 가능, 경미한 보완 필요 |
| **B** | 3.0~3.4 | 보통 — 참고 수준, 단독 의사결정 근거로 부족 |
| **C** | 2.0~2.9 | 미흡 — 재작업 필요, 핵심 항목 누락 다수 |
| **F** | 1.0~1.9 | 부적격 — 도구 미사용, 할루시네이션, 구조 없음 |

## 채점 규칙 (반드시 준수)

1. **1/3/5만 부여**. 애매하면 낮은 쪽으로.
2. **근거 1줄 필수**. 모든 점수에 이유 기재.
3. **로그 없이 E3 채점 금지**. 로그가 없으면 효율성을 "로그 미제공 — 채점 불가"로 표기.
4. **Q1 critical 규칙 준수**. Q1이 1점이면 → Q0은 무조건 최대 2점.
5. **보고서 내용과 로그 교차 검증**. 보고서에 "DART 확인"했는데 로그에 없으면 → Q1+Q2 감점.

## Step 5 (중간점검 전용): 시스템 개선 방향 도출

> **블라인드 비교 모드에서는 3라운드 전부 완료 후 이 단계 수행.**
> **중간점검 모드에서는 1개 샘플 채점 직후 이 단계 수행.**
>
> 채점 결과를 근거로 CORTHEX 시스템의 구체적 개선 방향을 제시합니다.
> **추측 금지. 채점에서 관찰한 사실만 근거로 사용.**

### 5-1. 전문가 품질 진단

각 전문가별로:
- **반복적으로 낮은 점수를 받은 차원**: (Q1~Q9 중 어디?)
- **원인 추정**: 프롬프트 문제? 도구 연결 문제? 모델 한계?
- **수정 대상 파일**: 해당 전문가의 soul 파일 또는 agents.yaml

### 5-2. CIO 종합 진단

- C2(빈 곳 보완력)가 낮다 → CIO 독자분석 프롬프트 강화 필요
- C3(종합 판단력)이 낮다 → 충돌 해소 지시 추가 필요
- C4(실행 구체성)가 낮다 → 실행 파라미터 필수 출력 지시 추가 필요

### 5-3. 효율성 진단

- E3(도구 중복) 문제 → 공유 캐시 도입? 전문가 역할 재분리?
- E1(위임 중복) 문제 → CIO의 전문가 지시 프롬프트 개선?
- E4(불필요 동원) 문제 → 해당 전문가 역할 재정의?

## SketchVibe가 뭔가?

비개발자(대표님)가 말+손그림으로 AI 에이전트 시스템을 설계하면,
Claude가 정식 다이어그램으로 변환 → 승인 시 실제 시스템(YAML)에 자동 반영하는 방식.

```
대표님 손그림 + 말 → Claude 정식 다이어그램 → 대표님 승인 → agents.yaml 자동 반영
```

자세한 개념: `docs/knowhow/15-sketchvibe.md`

## 특허 가능성 판정

```
발명 성립성:   ██████████ 확실 (100%)
신규성:        ████████░░ 높음  (80%)  — 통합 파이프라인 조합이 핵심
진보성:        █████░░░░░ 중간  (50%)  — 청구항 설계가 전부
종합:          65~75점 / 100 (중상)
```

**핵심 차별점 3가지 (이게 특허 생사를 가름):**
1. 자연어 + 손그림 **동시** 입력 (기존은 둘 중 하나만)
2. AI가 "설계 의도"를 추론 (단순 변환이 아님)
3. 사용자 승인 → 실시간 에이전트 배포 (기존은 파일 저장만)

## 청구항 핵심 (CLO 초안)

```
(a) 자연어 + 자유형 스케치 동시 수신
(b) AI가 설계 의도 추론 (비개발자 언어를 시스템 구조로)
(c) 손그림 위에 정식 다이어그램 덧그리기 (overlay)
(d) 대표님 승인(확정 트리거) 수신
(e) 실행 중인 AI 에이전트 시스템에 즉시 반영
```

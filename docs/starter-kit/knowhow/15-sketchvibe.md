## 개념

기존 바이브 코딩(Andrej Karpathy, 2025): **말**로 AI에게 코딩 지시
**SketchVibe**: **말 + 그림**으로 AI에게 시스템 설계 지시

```
[기존 바이브코딩]
말 → AI → 코드

[SketchVibe]
말 + 손그림(화살표/박스) → AI 해석 → 정식 다이어그램 → 시스템 반영
```

---

## 작동 방식

1. 대표님이 캔버스에 **초록색**으로 느낌만 스케치 (화살표, 박스, 메모)
2. Claude가 읽고 **정식 다이어그램**으로 덧그리기
3. 대표님: "맞아" → **승인**
4. Claude가 `agents.yaml` 등 시스템 config 업데이트
5. 2D 시스템 맵 **자동 반영**

## 특허 가능성

- 한국 소프트웨어 특허 요건: "기술적 효과가 있는 방법" → 해당
- 핵심 청구항 후보: *"비개발자가 자연어+자유형 스케치로 의도 표현 시 AI가 시스템 config 자동 반영하는 방법 및 시스템"*
- 조각 선행기술은 있으나 **통합 경험으로서의 신규성** 있음

## CLO 선행특허 조사 프롬프트

```
다음 아이디어(SketchVibe)에 대해 한국 특허 선행기술 조사를 해줘.

【조사 대상】
"비개발자(non-technical user)가 자연어(말) + 자유형 스케치(손그림 화살표/박스)로
시스템 설계 의도를 표현하면, AI가 이를 해석하여 정식 다이어그램으로 변환하고,
확정 시 실제 시스템 구성 파일(YAML/JSON/코드)에 자동 반영되는 방법 및 시스템"

【조사 항목】
1. KIPRIS(kipris.or.kr)에서 선행특허 검색
   키워드: "스케치 다이어그램 자동변환", "AI 시스템 설계 동기화",
   "비개발자 AI 코드 생성", "visual programming AI", "diagram to configuration sync"

2. 미국 USPTO에서도 동일 개념 검색

3. 결과 기반 의견:
   - 신규성(새로운가?) 판단
   - 진보성(자명하지 않은가?) 판단
   - 특허 가능성 종합 의견 (상/중/하)
   - 핵심 청구항 초안 1개 작성

솔직하게 의견 줘. 가능성 없으면 없다고 해.
```

## 3. 신규성 / 진보성 판단

### 3-1. 신규성 (특허법 제29조 제1항) — 🟡 조건부 인정

```
레이어 A (스케치→다이어그램)  : Eraser.io, Miro AI, Sketch2Code → 선행기술 존재
레이어 B (자연어→코드/다이어그램): GitHub Copilot, D2, Stately AI → 선행기술 존재
레이어 C (다이어그램→YAML/JSON): Pulumi, AWS CDK → 유사 존재

★ SketchVibe 핵심 = A + B + C 동시 통합
  + Human-in-the-Loop 확정 트리거
  + AI 에이전트 시스템 실시간 배포 연동
  → 이 완전 조합은 현재 미발견 → 신규성 주장 가능
```

### 3-2. 진보성 (특허법 제29조 제2항) — 🟡 조건부 인정

심사관 예상 거절 이유: "A+B+C 그냥 붙이면 되는 거 아닌가?"

→ 아래 **상승효과(Synergistic Effect)** 를 명세서에 반드시 명시해야 합니다:

| 상승효과 항목 | 설명 | 진보성 근거 |
|-------------|------|-----------|
| **의도 추론 정확도 비약 향상** | 자연어 단독 or 스케치 단독보다 동시 입력 시 AI 해석 정확도 급상승 | 단순 결합 이상의 효과 |
| **확정 트리거 기반 원자적 동기화** | 승인→즉시 배포의 Atomic Transaction 구조 | 기존 기술에 없는 메커니즘 |
| **AI 에이전트 조직도 특화 어휘** | 일반 스케치 변환이 아닌 AI 에이전트 계층 구조 특화 인식 | 특정 기술 문제 해결 |

## 5. 핵심 청구항 초안 (독립항 — 방법 청구항)

```
【청구항 1】

비개발자 사용자가 인공지능(AI) 에이전트 시스템을 구성하는 방법에 있어서,

(a) 자연어 입력 및 자유형 스케치 입력을 동시에 수신하는 단계;
    — 자연어 입력: 에이전트 간 연결 관계를 기술하는 텍스트
    — 자유형 스케치 입력: 에이전트 박스 및 화살표를 포함하는 손그림 이미지

(b) 멀티모달 AI 모델에 입력하여,
    에이전트 간 계층 구조 및 상호작용 관계를 포함하는
    정식 시스템 다이어그램을 자동 생성하는 단계;

(c) 상기 다이어그램을 원본 스케치 위에
    오버레이(overlay) 방식으로 표시하는 단계;

(d) 사용자의 확정(confirmation) 입력을 수신하는 단계; 및

(e) 확정 입력에 응답하여,
    다이어그램을 파싱하여 AI 에이전트 구성 파일(YAML 또는 JSON)을 자동 생성하고,
    실행 중인 AI 에이전트 시스템에 즉시 반영하는 단계;

를 포함하는 비개발자 멀티모달 AI 에이전트 시스템 구성 방법.
```

> **핵심 4요소**: ①멀티모달 동시 입력 ②오버레이 다이어그램 ③확정 트리거 ④즉시 배포

# 전사 구조조정(“비서실장·처장급 제외 전원 해고”) 재무 예측·손익 분석 — 3시나리오/현금흐름 관점 결론

## 0) 재무 결과 2줄 요약 (CSO/대표 선판단용)
- **LTV/CAC 등급·Payback**: 현재 **ARPU·Churn·CAC 데이터 미제공으로 산정 불가**(필수 입력: 최근 3개월 코호트/광고비/유료전환).  
- **3-시나리오 손익분기·현금런웨이**: 해고는 **월 Burn을 낮추지만(유지비 절감)**, **퇴직금/해고비용(일시 현금유출)** 때문에 **“절감 Payback(손익분기)”이 2~6개월**로 자주 발생합니다. **현금이 2~3개월치 미만이면 오히려 즉시 파산 리스크**가 커질 수 있습니다. (AI API 비용은 COGS로 포함)

## 3) 모델 구조(핵심 공식)
### 3.1 월 손익(P&L) 골격
- **매출 \(R\)**  
- **매출총이익 \(GP = R \times GM\)**  (여기서 **GM에는 AI API 비용 포함**)
- **고정비 \(F\)** = 인건비(유지 인원) + 오피스/툴/기타 고정비  
- **영업손익(간이) \(OP \approx GP - F\)**

### 3.2 구조조정의 “현금 Payback(손익분기)” 개념
- **월 절감액 \(S\)** = (구조조정 전 월 고정비) − (구조조정 후 월 고정비)  
- **일시 비용 \(C_{oneoff}\)** = 퇴직금 + 해고예고수당/정산 + 법무/노무 리스크 준비금(권고)  
- **Payback 개월수 \(\approx C_{oneoff} / S\)**  
  - 보통 **2~6개월**이면 “현금 관점에서 의미 있음”  
  - **현금 런웨이 < Payback**이면 “절감 전에 돈이 먼저 마름” → 매우 위험

## 4) 3-시나리오 분석 (예시 수치, 전부 Assumption)
아래는 **대표가 바로 감을 잡도록 하는 예시**입니다. 귀사 수치로 즉시 치환 가능.

### 공통 가정(예시)
- 현재 가용현금 \(Cash_0\) = **500백만원**
- 총원 \(H_{total}\)=**25명**, 유지 \(H_{remain}\)=**7명**
- 해고대상 18명 월 총인건비 평균 = **6.0백만원/인**
- 유지 7명 월 총인건비 평균 = **9.0백만원/인**
- 기타 고정비(오피스/툴/최소클라우드) = **60백만원/월**
- 현재 매출 \(R\)=**70백만원/월**, 총마진 \(GM\)=**75%** (AI API 포함 COGS 25%)

#### 구조조정 전(예시)
- 인건비 = \(18×6 + 7×9 = 171\)백만원/월  
- 고정비 합 = \(171 + 60 = 231\)백만원/월  
- 총이익 = \(70×0.75 = 52.5\)백만원/월  
- **월 Burn ≈ 231 − 52.5 = 178.5백만원/월**  
- **런웨이 ≈ 500 / 178.5 = 2.8개월**

---

### 시나리오 P10 (보수): “부분 감원 + 매출 방어”
- 감원 규모: 30% 감원(예: 8명)  
- 일시 비용 \(C_{oneoff}\): **120백만원** (퇴직금/정산 최소 가정)  
- 월 절감액 \(S\): **50백만원/월**  
- 매출 영향: **-5%** (70→66.5)

**결과(핵심)**  
- Payback ≈ 120/50 = **2.4개월**  
- 단기 현금압박 낮음, **서비스 유지력 상대적으로 큼**  
- 단, “전원 해고” 방침과는 불일치(현실적 대안)

---

### 시나리오 P50 (기본): “지시안 준수(비서실장·처장급 제외 해고) + 매출 일부 감소”
- 감원: 18명 해고, 7명만 유지  
- 일시 비용(예시 산식):
  - 퇴직금(평균 근속 1.5년 가정) ≈ 18명 × (6백만원 × 1.5) = **162백만원**
  - 기타 정산/법무·노무 준비금(보수적) **80~120백만원**
  - 합계 \(C_{oneoff}\) = **약 270백만원**
- 구조조정 후 월 고정비:
  - 인건비 = 7×9=**63백만원**
  - 기타 고정비 = **60백만원**
  - 합계 **123백만원/월**
- 매출 영향: 실행 1~2개월 후 **-30%** (70→49) 가정  
- 총이익 = 49×0.75=**36.75백만원/월**  
- **구조조정 후 월 Burn ≈ 123−36.75 = 86.25백만원/월**  
- 월 절감액 \(S\) ≈ 178.5−86.25 = **92.25백만원/월**

**결과(핵심)**  
- **Payback ≈ 270 / 92.25 = 2.9개월**  
- 구조조정 직후 잔액 \(Cash_1 = 500−270 = 230\)백만원  
- **런웨이(구조조정 후) ≈ 230 / 86.25 = 2.7개월**  
- 즉, **현금이 500백만원 수준이면 “처음 3개월은 오히려 더 위험”**할 수 있음(절감 효과가 쌓이기 전, 일시 비용이 먼저 나감).  
- 결론: **현재 현금이 “최소 4~6개월치(구조조정 전 기준) 런웨이”가 아니면** 이 방식은 “살리기 위한 구조조정”이 아니라 **청산 가속**이 될 수 있음.

---

### 시나리오 P90 (낙관/극단): “운영 최소화(셧다운/매각 준비)”
- 목적: 제품 유지보다 **자산/기술/특허/고객계약 매각**  
- 매출: 1~2개월 내 **0~20백만원**으로 급락 가능  
- 고정비: 클라우드/보안/법무/회계 최소 유지 필요(예: 30~70백만원/월)  
- 일시 비용: 정리 비용 더 큼(법무/위약금/환불)

**결과(핵심)**  
- “사업 지속” 손익분기보다 **청산가치 극대화**가 목표가 됨  
- 이 경우 특허/코드/데이터 권리정리(양도 가능성)와 계약 위약금이 관건

## 5) 결론 및 권고 (재무 관점, 솔직하게)
1. **현금이 Payback(2~6개월) 버틸 만큼 없으면**, “대규모 즉시 해고”는 **현금흐름상 자살 버튼**이 될 수 있습니다.  
2. “전원 해고”가 불가피하더라도, 재무적으로는  
   - **(a) 단계적 감원**, **(b) 권고사직/자발적 프로그램**, **(c) 급여 유예/감액**, **(d) 클라우드/툴 즉시 다운사이징**을 **패키지로** 묶어야 런웨이가 늘어납니다.  
3. 매출이 인력 의존적(영업/CS/개발)이라면, 감원으로 매출이 같이 떨어져 **손익분기점이 더 멀어질** 수 있습니다. 그래서 구조조정은 “비용만”이 아니라 **매출 유지 설계**가 동시에 필요합니다.

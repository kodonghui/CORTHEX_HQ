# 하드코딩 색상 107곳 → hq 토큰 전수 치환

## 작업 날짜
2026-02-14

## 작업 브랜치
claude/improve-uxui-design-zHXH6

## 변경 사항 요약

### 뭘 했는지
화면(index.html)에서 **hq 색상 토큰 시스템을 무시하고 직접 색상을 지정한 곳 107개 이상**을 전부 찾아서 토큰으로 교체했습니다.

### 왜 했는지
이전에 hq 토큰(색상 변수)의 값을 밝게 바꿨는데, 화면의 **절반 이상**이 토큰을 안 쓰고 `text-gray-300`, `text-white` 같은 고정 색상을 직접 쓰고 있었습니다. 그래서 토큰 값을 아무리 바꿔도 화면에 반영이 안 됐습니다.

---

### 수정 내용 (6단계)

#### 1단계: gray 계열 하드코딩 → hq 토큰 (24곳)

| 바뀐 곳 | 수정 전 | 수정 후 |
|---------|---------|---------|
| 모달 닫기 버튼 5개 | `text-gray-400` | `text-hq-muted` |
| 취소 버튼 2개 | `bg-gray-700 text-gray-300` | `bg-hq-surface text-hq-text` |
| 검수 모달 제목 3개 | `text-gray-300` | `text-hq-text` |
| 검수 모달 설명 6개 | `text-gray-500` | `text-hq-muted/70` |
| 입력 필드 텍스트 4개 | `text-gray-200` | `text-hq-text` |
| 라벨 2개 | `text-gray-400` | `text-hq-muted` |
| 추론 레벨 색상 2개 | `border-gray-400` | `border-hq-muted` |

#### 2단계: text-white → text-hq-text (약 50곳)

| 바뀐 곳 | 설명 |
|---------|------|
| CEO 이름 | 사이드바 CEO 텍스트 |
| 요약 카드 숫자 4개 | 오늘 작업, 비용, 에이전트, 시스템 |
| 섹션 제목 2개 | 최근 완료 작업, 빠른 명령 |
| 성능 탭 제목+숫자 3개 | 에이전트 성능, 총 LLM 호출, 총 작업 |
| 대시보드 제목+카드 5개 | 대시보드, 예산, 품질, 최근 완료 |
| 예약/워크플로우 제목 4개 | 작업 예약, 워크플로우, 항목 이름들 |
| 모달 제목 6개 | 새 예약, 메모리, 워크플로우, 품질, 에이전트, 작업 상세 |
| 입력 필드 12개 | 모든 모달 input/textarea |
| hover 상태 7개 | `hover:text-white` → `hover:text-hq-text` |

**유지한 것**: 파란/빨간/그라데이션 배경 위의 버튼 (25곳) — 흰색이 맞음

#### 3단계: 기타 하드코딩 색상 → hq 토큰 (9곳)

| 수정 전 | 수정 후 | 위치 |
|---------|---------|------|
| `text-green-400` | `text-hq-green` | "설정됨" 뱃지 2곳, "저장 완료" |
| `text-yellow-400` | `text-hq-yellow` | "저장 중..." |
| `text-red-400` | `text-hq-red` | "저장 실패" |
| `text-amber-400` 등 5개 | `text-hq-yellow` 등 | 사이드바 도구 풀 |
| `border-teal-400` | `border-hq-cyan` | 추론 minimal 레벨 |
| `border-red-400` | `border-hq-red` | 추론 xhigh 레벨 |

#### 4단계: 모달 반응형 수정 (7곳)

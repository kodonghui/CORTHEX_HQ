## 변경 사항 요약

### 1. 색상 팔레트 전면 교체 (남색 → 무채색)

**문제**: 대시보드가 온통 남색(파란빛 어두운 색)이라 답답하고 밋밋했음

**해결**: 무채색 기반 "뉴트럴 다크" 디자인으로 교체

| 영역 | 이전 (남색) | 변경 후 (뉴트럴) |
|------|-----------|---------------|
| 배경 | `#1E2637` (남색) | `#0D1117` (짙은 회색) |
| 카드 | `#37445F` (남색) | `#161B22` (어두운 회색) |
| 패널 | `#4B5A78` (남색) | `#212630` (회색) |
| 테두리 | `#6E87B4` (파란 회색) | `#303641` (중간 회색) |

→ 배경이 무채색이 되면서 **청록색, 보라색, 초록색 등 강조 색상이 훨씬 잘 보이게** 됨

### 2. CSS 아키텍처 수리 (라이트모드 버그 수정)

**문제**: 194개 색상 클래스가 다크모드 색상(`#f2f6ff`)으로 고정되어 있어서, 라이트모드로 바꿔도 색상이 안 바뀜

**이전 방식** (잘못된 방식):
```css
.text-hq-text { color: #f2f6ff !important; }  /* ← 항상 흰색 고정 */
```

**변경 후** (올바른 방식):
```css
.text-hq-text { color: rgb(var(--hq-text)) !important; }  /* ← 테마에 따라 자동 변환 */
```

→ 다크모드에서는 밝은 글자, 라이트모드에서는 어두운 글자로 **자동 전환**

### 3. 라이트모드 전용 스타일 추가

**문제**: 라이트모드에서 사이드바가 여전히 남색이었고, 카드와 입력창도 이상하게 보였음

**해결**: 라이트모드 전용 스타일을 새로 추가
- 사이드바: 흰색 배경 + 은은한 그림자
- 카드: 흰색 배경 + 미세한 테두리 + 부드러운 그림자
- 헤더: 반투명 흰색 + 아래쪽 그림자
- 입력창: 흰색 배경 + 회색 테두리

## 기술적 변경 상세

1. **CSS 변수** (`:root, .dark` + `html:not(.dark)`) — 다크/라이트 두 세트의 색상값 새로 정의
2. **194개+ 순수 CSS 클래스** — 하드코딩 hex 값(`#f2f6ff`)을 CSS 변수 참조(`rgb(var(--hq-text))`)로 전면 교체
3. **구조 요소 (glass, aside, desk-card, office-room, input)** — 다크모드/라이트모드 각각 별도 스타일 정의
4. **body 배경색** — 다크: `rgb(10 12 18)`, 라이트: `rgb(245 246 250)`

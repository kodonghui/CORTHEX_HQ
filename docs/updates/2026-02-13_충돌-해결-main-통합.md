# 충돌 해결 - main 브랜치 통합

## 작업 날짜
2026-02-13

## 작업 브랜치
claude/corthex-improvements-kE0ii

## 변경 사항 요약

### 뭘 했는지
우리 작업 브랜치와 main 브랜치(본줄기) 사이에 **충돌(같은 파일을 양쪽에서 다르게 수정해서 합칠 수 없는 상태)**이 생겨서 이걸 해결했습니다.

### 왜 충돌이 생겼는지
우리 브랜치에서 작업하는 동안, 다른 세션들이 main에 3개의 작업을 합쳤습니다:
- **PR #12**: 버그 수정 9건 (예산 추적, 도구 권한, 조직도, API 등)
- **PR #13**: UX/UI 대규모 개선 7가지 (다크모드, 토스트 알림, 에이전트 빛남 효과 등)
- **PR #15**: 텔레그램 모바일 지휘 시스템

양쪽 다 `web/templates/index.html` (홈페이지 메인 화면) 파일을 수정해서 충돌이 발생했습니다.

### 어떻게 해결했는지
`index.html` 파일의 충돌 5건을 수동으로 해결했습니다:

| 충돌 위치 | 해결 방법 |
|-----------|-----------|
| 메인 화면 구조 | 우리 탭 시스템(홈/명령/성능/작업내역/예약/워크플로우) 유지 |
| 보내기 버튼 + 프리셋 | 우리 버전 유지 (프리셋 칩 포함) |
| JavaScript 변수 | 양쪽 모두 합침 (우리 탭 시스템 + main의 다크모드/토스트 등) |
| 초기화 함수 | 양쪽 모두 합침 (인증 + 다크모드 + 알림 권한) |
| 에러 처리 | 양쪽 모두 합침 (에러 알림 배너 + 토스트 알림) |

### 수정한 파일
- `web/templates/index.html` (홈페이지 메인 화면) — 충돌 해결

### main에서 가져온 새 파일/변경 (자동 합쳐진 것들)
- `src/telegram/notifier.py` (텔레그램 알림 모듈) — 새 파일
- `src/telegram/bot.py`, `bridge.py`, `formatter.py` — 텔레그램 관련 수정
- `src/core/budget.py`, `registry.py` — 예산/등록 관련 버그 수정
- `src/tools/leet_survey.py`, `pool.py` — 도구 관련 버그 수정
- `web/app.py` (웹 서버) — 자동 합쳐짐
- 기타 문서 파일 3개

# 자동 배포 paths 필터 제거 — main 합치면 무조건 서버 반영

## 작업 날짜
2026-02-14

## 작업 브랜치
claude/sync-multiple-computers-Nz8IR

## 변경 사항 요약

### 문제가 뭐였나?
- 기존에는 `web/` 폴더(홈페이지 화면 파일들) 안의 파일이 바뀔 때만 서버에 자동 배포(반영)가 됐음
- 그래서 `web/` 폴더 밖의 파일(설정 파일, 문서 등)만 바뀌면 서버 배포가 안 돌아감
- CEO가 PR을 합쳤는데 홈페이지가 안 바뀌는 상황이 발생

### 뭘 바꿨나?

| 파일 | 설명 | 뭘 바꿨는지 |
|------|------|------------|
| `.github/workflows/deploy.yml` (자동 배포 설정 파일) | main에 코드가 합쳐지면 서버에 올리는 자동화 규칙 | `paths` 필터(조건) 제거 → main에 합쳐지면 **무조건** 배포 실행 |
| `CLAUDE.md` (클로드 작업 규칙 파일) | 클로드가 작업할 때 지켜야 할 규칙 | "서버 배포 규칙" 섹션 추가 — 서버 주소, 배포 과정, 수동 배포 방법, 주의사항 등 |
| `docs/project-status.md` (프로젝트 현재 상태) | 프로젝트 진행 상황 기록 | 자동 배포 설명을 "무조건 실행"으로 수정 |

### 이제 어떻게 되나?
- main 브랜치에 뭐든 합쳐지면(어떤 파일이 바뀌든) → 서버에 자동 반영됨
- 배포 작업이 가벼워서(파일 복사 + 서버 재시작) 매번 실행해도 시간/비용 부담 없음

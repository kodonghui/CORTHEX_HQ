# 사이드바 에이전트 전원 안 뜨는 버그 수정

- **날짜**: 2026-03-03
- **커밋**: 1e0f435 (반응성) + f5b1d05 (JSON 재생성)
- **배포**: SSH 직배포 (deploy-fast.sh)

## 증상

대표님 계정으로 로그인하면 왼쪽 사이드바에 비서실장만 표시되고, 나머지 5명(전략팀장, 법무팀장, 마케팅팀장, 금융분석팀장, 콘텐츠팀장)이 안 보임.

## 원인

Alpine.js 반응성 버그.

1. `agentCliOwner`가 `{}` 빈 객체로 초기화됨
2. `loadAgentsAndTools()`에서 `this.agentCliOwner[key] = value` 방식으로 동적 키 추가
3. Alpine.js는 **기존 객체에 새 키 추가**를 변경으로 감지하지 못함
4. `x-show="agentCliOwner['leet_strategist'] === workspace.sidebarFilter"` 조건이 재평가 안 됨
5. 결과: `undefined === 'ceo'` = `false` → 리트마스터 본부 전원 숨김

## 수정 내용

### web/static/js/corthex-app.js

1. **객체 교체 방식** (핵심 수정)
   - 기존: `this.agentCliOwner[id] = value` (뮤테이션 — 반응성 X)
   - 수정: 임시 객체에 누적 → `this.agentCliOwner = newObj` 한번에 교체 (반응성 O)
   - 동일 패턴 8개 맵 전부 수정: agentNames, agentInitials, agentRoles, agentModelRaw, agentReasonings, agentModels, agentCliOwner, agentDivision

2. **init()에서 await 추가**
   - 기존: `this.loadAgentsAndTools()` (await 없이 비동기 실행)
   - 수정: `await this.loadAgentsAndTools()` (사이드바 렌더 전 데이터 보장)

### config/agents.json (2차 수정 — 근본 원인)

agents.yaml은 리네이밍 완료(leet_strategist 등)였지만, agents.json은 구 ID(cso_manager)가 남아있었음.
config_loader.py가 JSON을 1순위로 로드 → HTML 하드코딩 ID와 불일치 → 매칭 실패.

- yaml2json.py 로직으로 agents.json + tools.json + workspaces.json 재생성
- 원인: deploy-fast.sh가 yaml2json.py를 실행하지 않음 (deploy.yml만 실행)

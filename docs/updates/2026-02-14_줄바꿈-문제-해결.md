# 줄바꿈 문제 해결 (윈도우-리눅스 충돌 방지)

## 작업 날짜
2026-02-14

## 작업 브랜치
claude/fix-crlf-line-endings-W0oeF

## 무슨 문제였나?
CEO님이 윈도우 컴퓨터에서 CORTHEX 파일을 다운받거나 열면, 윈도우가 멋대로 파일 안의 "엔터(줄바꿈)"를 자기 방식으로 바꿔버립니다. 그래서 GitHub Desktop에서 보면 **"296개 파일이 바뀌었다"**고 뜨는데, 실제로는 **내용이 하나도 안 바뀐** 상태였습니다.

## 뭘 했나?
`.gitattributes`라는 설정 파일을 프로젝트에 추가했습니다.

### 새로 만든 파일
- `.gitattributes` (줄바꿈 자동 통일 설정 파일) — 이 파일이 있으면 윈도우에서 작업해도 줄바꿈이 자동으로 통일됩니다

### 이 파일이 하는 일
- 윈도우에서 파일을 받아도 줄바꿈이 자동으로 통일됨
- "296개 파일이 바뀌었다"는 헛소리가 앞으로 안 나옴
- 윈도우 배치 파일(`.bat`)은 윈도우 방식 유지
- 이미지 같은 파일은 건드리지 않음

# 도구함 빈 화면 버그 수정

## 작업 날짜
2026-02-14

## 작업 브랜치
claude/investigate-tool-room-staff-KFMIx

## 문제가 뭐였나?
대시보드 왼쪽 메뉴에 있는 **"도구함"** 섹션을 열면 **아무것도 안 나왔습니다**.
변리사, 세무사, 디자이너, 번역가, 변호사 관련 도구 등 **50개 넘는 도구**가 등록되어 있는데도 화면에는 "서버 연결 후 표시됩니다"만 보이는 상태였습니다.

## 원인이 뭐였나?
Oracle Cloud 서버에서 돌아가는 경량 서버(`arm_server.py`)가 문제였습니다.

- **도구 데이터 파일** (`config/tools.yaml`): 변리사, 세무사, 법령 검색, 상표 유사도 검사기 등 50개 넘는 도구 정보가 잘 들어있었음
- **경량 서버** (`web/arm_server.py`): `/api/tools` 요청이 오면 **빈 리스트(`[]`)를 반환** → 화면에 아무것도 안 보임

쉽게 말하면, **도구 목록이 창고에는 있는데, 서버가 그 창고 문을 안 열고 "비어있습니다"라고 답하고 있었던 것**입니다.

## 뭘 고쳤나?

### 수정한 파일
- `web/arm_server.py` (서버에서 실제 돌아가는 경량 서버 코드)

### 수정 내용
| 수정 전 | 수정 후 |
|---------|---------|
| `return []` (무조건 빈 리스트 반환) | `tools.yaml` 파일을 읽어서 실제 도구 목록을 반환 |

## 현재 상태

- main에 합쳐지고 서버에 배포되면, 도구함에 50개 넘는 도구가 전부 표시될 예정

## 2. 🔴 최우선 — 지금 당장 필요한 것 5개

### 아이디어 ① 진짜 웹검색 도구 (`real_web_search`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 현재 `web_search`는 AI가 이미 알고 있는 지식으로 대답하는 "가짜 검색"입니다. 실제 구글/네이버를 검색하는 진짜 도구가 필요합니다 |
| **왜 필요?** | 시장조사, 경쟁사 분석, 최신 뉴스 수집 등 거의 모든 업무에서 "오늘 현재" 정보가 필요합니다. 지금은 AI의 학습 데이터 마감일 이후 정보를 전혀 못 가져옵니다 |
| **누가 쓰나?** | 시장조사 전문가, 시황분석 전문가, 연대기 전문가 등 7개 이상 에이전트 |
| **어떻게 만드나?** | SerpAPI 또는 Google Custom Search API를 연동. 한 번 호출에 ~$0.001 수준으로 저렴 |
| **사용 예시** | "오늘 삼성전자 관련 뉴스 검색해줘" → 실제 구글 검색 → 상위 10개 결과 반환 |

- 엑셀/CSV 파일 읽기 및 내용 요약

- 새 엑셀 파일 생성 및 저장

- 히트맵 (상관관계 분석용)
- 대시보드형 복합 차트 (CEO 보고용)

### 아이디어 ④ PDF 텍스트 추출 도구 (`pdf_parser`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | PDF 파일에서 텍스트를 추출(꺼내는 것)하는 도구 |
| **왜 필요?** | 계약서, 재무보고서, 학술논문, 정부 문서 — 중요한 문서의 대부분이 PDF입니다. 현재 CLO(법무처)가 계약서를 검토하려면 사람이 직접 텍스트를 복사해서 넣어야 합니다 |
| **누가 쓰나?** | 법무IP처 (계약서 분석), 투자분석처 (사업보고서), 사업기획처 (시장보고서), 출판기록처 (자료 아카이빙) |

**구체적 기능:**
- PDF → 텍스트 추출
- 표(table) 자동 인식 및 추출
- 스캔된 PDF → OCR(이미지를 글자로 인식)로 텍스트 변환
- 페이지별 분리 추출

### 아이디어 ⑤ AI 모델 전문가용 도구 세트
| 항목 | 내용 |
|------|------|
| **뭔데?** | 현재 AI 모델 전문가(ai_model_specialist)는 **사용할 수 있는 도구가 0개**입니다. 팔이 없는 사람한테 일하라고 하는 격. 3개 도구를 추가 제안합니다 |

**도구 5-1: `prompt_tester` (프롬프트 테스터)**
- 같은 질문을 여러 AI 모델(GPT-5, Claude Sonnet 등)에 동시에 보내서 결과를 비교
- "어떤 모델이 이 작업에 더 적합한가?" 판단에 활용
- 비용도 함께 계산하여 "성능 대비 비용" 비교표 자동 생성

**도구 5-2: `embedding_tool` (임베딩 도구)**
- 텍스트를 벡터(숫자 배열)로 변환 → 의미적 유사도 검색 가능
- 예: 과거 보고서 1000개 중에서 "지금 질문과 가장 관련 있는 보고서" 자동 검색
- 나중에 RAG(검색 기반 답변 생성) 시스템의 핵심이 됨

**도구 5-3: `token_counter` (토큰 카운터)**
- 텍스트의 토큰(AI가 처리하는 글자 단위) 수를 정확히 측정
- 비용 사전 예측 가능: "이 작업을 실행하면 약 $0.12 들 예정"
- 긴 문서를 AI 한도 내로 자동 분할(chunking)

## 3. 🟡 중요 — 자동화를 한 단계 올려줄 것 5개

### 아이디어 ⑥ 알림/푸시 엔진 (`notification_engine`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 중요한 일이 발생하면 CEO에게 자동으로 알림을 보내는 도구 |
| **왜 필요?** | 현재 `dart_monitor`(공시 알리미), `law_change_monitor`(법령 변경 알리미), `competitor_monitor`(경쟁사 감시) 같은 "감시" 도구가 있지만, 이것들이 뭔가 발견해도 CEO에게 **알려줄 방법이 없습니다**. 사람이 직접 물어봐야만 결과를 알 수 있습니다 |
| **어떻게 동작?** | 텔레그램 봇 + 이메일로 자동 발송 |
| **사용 예시** | 삼성전자가 중요 공시를 올림 → `dart_monitor`가 감지 → `notification_engine`이 CEO 텔레그램으로 "🔔 삼성전자 신규 공시: 분기보고서 제출" 전송 |

### 아이디어 ⑦ 글로벌 주식/암호화폐 데이터 (`global_market_tool`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 현재 `kr_stock`은 한국 주식만 됩니다. 미국 주식(S&P500, 나스닥), 글로벌 지수, 암호화폐까지 커버하는 도구 |
| **왜 필요?** | 한국 증시는 미국 시장에 강하게 연동됩니다. 미국 나스닥 동향 없이 한국 종목 분석은 절반짜리. 비트코인 시세도 시장 심리 판단에 중요 |
| **어떻게 만드나?** | Python `yfinance`(무료)로 미국/글로벌 주식 데이터, CoinGecko API(무료)로 암호화폐 데이터 |
| **누가 쓰나?** | 시황분석 전문가, 종목분석 전문가, 리스크관리 전문가 |
| **사용 예시** | "나스닥이 3% 빠졌는데 한국 반도체 종목 영향 분석해줘" → 나스닥 데이터 + 한국 주식 데이터 함께 분석 |

---

### 아이디어 ⑧ 재무 계산기 (`financial_calculator`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | DCF(미래 현금흐름 현재가치), NPV(투자 수익률), IRR(내부수익률) 등 재무 분석에 필요한 수학 계산을 자동으로 해주는 도구 |
| **왜 필요?** | 사업계획 전문가와 재무모델 전문가가 숫자 계산을 할 때, 현재는 AI가 "대충 어림잡아" 계산합니다. AI는 수학을 잘 못합니다. 정확한 계산기가 필요 |
| **어떻게 만드나?** | Python `numpy-financial` 라이브러리 |
| **사용 예시** | "LEET MASTER 3년 매출 전망 기반 기업 가치 계산해줘" → DCF 모델로 정확한 수치 산출 |

**구체적 기능:**
- DCF (기업 가치 평가)
- NPV / IRR (투자 프로젝트 수익성 평가)
- 대출/이자 계산
- 감가상각 계산
- 손익분기점 계산

## 4. 🟢 미래 투자 — 장기적으로 경쟁력이 될 것 5개

### 아이디어 ⑪ 벡터 지식 검색 (`vector_knowledge`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 현재 knowledge(지식) 시스템은 .md 파일을 통째로 프롬프트에 넣는 방식. "이 질문과 관련 있는 과거 지식"을 의미 기반으로 검색하는 시스템 |
| **왜 필요?** | 보고서가 쌓이면 수백 개가 됩니다. 전부 읽힐 수 없으니, "이 질문에 관련 있는 과거 보고서 3개"를 자동으로 찾아주면 답변 품질이 크게 향상 |
| **기술** | ChromaDB 또는 FAISS + OpenAI Embeddings |

### 아이디어 ⑫ 의사결정 기록 시스템 (`decision_tracker`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | "언제, 무슨 데이터를 보고, 무슨 결정을 내렸고, 결과가 어땠는지" 자동 기록 |
| **왜 필요?** | 3개월 전에 "삼성전자 매수" 판단을 내린 근거가 뭐였는지 추적 가능. 성공/실패 패턴을 학습하여 점점 더 나은 판단 가능 |

### 아이디어 ⑬ 이미지 생성 도구 (`image_generator`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 마케팅용 이미지, 블로그 썸네일, SNS 포스트 이미지를 AI로 자동 생성 |
| **왜 필요?** | 콘텐츠 전문가가 SNS 게시글을 만들 때 텍스트만 가능. 이미지가 있으면 클릭률/공유율이 2-3배 증가 |
| **기술** | DALL-E 3 API 또는 Stable Diffusion |

### 아이디어 ⑭ 음성/회의 녹음 분석 도구 (`audio_transcriber`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 회의 녹음 파일(mp3, wav)을 텍스트로 변환하고, 자동으로 회의록을 만들어주는 도구 |
| **왜 필요?** | `meeting_formatter`가 회의록을 정리해주지만, 회의 내용을 직접 타이핑해야 합니다. 녹음 파일만 넣으면 자동으로 텍스트 변환 → 회의록 정리까지 |
| **기술** | OpenAI Whisper API 또는 Clova Speech API |

### 아이디어 ⑮ 에이전트 간 자율 협업 시스템 (`cross_agent_protocol`)
| 항목 | 내용 |
|------|------|
| **뭔데?** | 현재는 "위에서 아래로" 명령이 내려가는 구조입니다. 에이전트가 스스로 "이건 법무처에도 알려야겠다"고 판단해서 횡적으로 정보를 공유하는 시스템 |
| **왜 필요?** | CIO가 투자 분석 중 "이 기업 특허 분쟁 있음" 발견 → 현재는 CEO가 별도로 CLO에 물어봐야 함 → 자율 협업이면 CIO가 CLO에 자동 요청 |
| **기술** | 현재 `relay_worker`를 확장하여 에이전트 간 메시지 프로토콜 추가 |

## 6. 워크플로우 & 스케줄 제안

### 현재 상태
- `workflows.yaml` → **비어 있음** (workflows: [])
- `schedules.yaml` → **비어 있음** (schedules: [])

이 두 파일이 비어있다는 것은, CORTHEX가 **CEO가 직접 명령할 때만 일하는** 수동 모드라는 뜻입니다. 아래를 채우면 **알아서 일하는** 자동 모드로 전환됩니다.

### 제안 워크플로우 (자동 업무 흐름)

| 워크플로우 이름 | 단계 | 설명 |
|:---:|------|------|
| **모닝 브리핑** | ① 시황분석 → ② 뉴스 수집 → ③ 일정 확인 → ④ 종합 보고서 | 매일 아침 8시, CEO에게 "오늘 알아야 할 것들" 자동 보고 |
| **투자 의사결정 파이프라인** | ① 종목 스크리닝 → ② 시황+종목+기술적 병렬 분석 → ③ 리스크 평가 → ④ 최종 판단 보고서 | "삼성전자 분석해줘" 한마디로 전체 파이프라인 자동 실행 |
| **경쟁사 변화 대응** | ① competitor_monitor 감지 → ② 시장 영향 분석 → ③ 대응 전략 수립 → ④ CEO 보고 | 경쟁사가 뭔가 바꾸면 자동으로 분석 시작 |
| **월간 사업 리뷰** | ① 재무 데이터 수집 → ② 시장 트렌드 분석 → ③ 부서별 실적 정리 → ④ 종합 보고서 + 차트 | 매월 1일 자동 실행 |
| **SNS 콘텐츠 파이프라인** | ① 트렌드 분석 → ② 콘텐츠 초안 → ③ SEO 최적화 → ④ 해시태그 추천 → ⑤ CEO 승인큐 | 콘텐츠 기획부터 발행 대기까지 자동 |
| **법률 리스크 점검** | ① 법령 변경 감지 → ② 사업 영향 분석 → ③ 계약서 재검토 → ④ 대응 방안 보고 | 관련 법령 바뀌면 자동 점검 |

### 제안 스케줄 (예약 작업)

| 주기 | 작업 | 목적 |
|------|------|------|
| **매일 08:00** | 모닝 브리핑 워크플로우 | CEO가 하루를 시작할 때 필요한 정보 |
| **매일 15:30** | 한국 장 마감 요약 | 오늘의 주식 시장 정리 |
| **매주 월요일 09:00** | 주간 종합 보고서 | 지난주 요약 + 이번주 계획 |
| **매주 수요일** | 뉴스레터 초안 생성 | 금요일 발행용 뉴스레터 준비 |
| **매월 1일** | 월간 사업 리뷰 | 월간 실적 + 시장 동향 정리 |
| **매월 15일** | 법률/규제 변경 점검 | 관련 법령 변동 사항 체크 |

## 확정된 내용

### 핵심 개념
손으로 캔버스에 대충 그리면서 Claude Code에게 말로 설명하면
→ Claude가 그림+말 동시에 읽고 → Mermaid 다이어그램으로 변환
→ "이렇게 이해했습니다" 확인 → 맞으면 실제 구현

### UX 원칙
- 타입 선택 UI 없음 (연결도/흐름도 구분 사용자가 안 해도 됨)
- AI가 스케치+자연어로 자동 판단
- "그려서 올리면 끝"

### 아키텍처
- 캔버스: Nexus Mode 2 Drawflow 캔버스 재활용 (새로 만들지 않음)
- 전달: Claude MCP 서버 통해 실시간 노출
- 변환: Claude → Mermaid 코드
- MVP: Phase 1 + Phase 2 둘 다 (같은 UX, 내부 정확도 차이)

### 보류
- 이름 재검토: 나중에 (상표 충돌 가능성 — getsketchvibe.com)
- 특허 출원: 나중에 (법무팀장과 논의)

## 변경 사항 요약

### 왜 했는지 (문제 상황)
- 대시보드 화면이 다크모드에서 **텍스트가 거의 안 보이는** 문제가 반복적으로 발생
- 원인: 색상이 **3단계 변환** (Tailwind CDN 로딩 → CSS 변수 해석 → 브라우저 렌더링)을 거쳐야 해서, 이 중 한 단계라도 실패하면 색상이 표시되지 않음
- 이전 시도들 (투명도 부스트, !important 백업 등)로는 해결되지 않았음

### 뭘 바꿨는지 (해결 방법)
- **방법 1: 모든 색상을 순수 CSS로 직접 정의** (CEO가 선택)
- HTML 파일에서 사용 중인 **194개 이상의 hq-* 색상 클래스**를 전부 추출
- 각 클래스의 색상값을 **하드코딩된 hex/rgba 값**으로 직접 정의
  - 예: `.text-hq-text { color: #f2f6ff !important; }`
  - 예: `.bg-hq-accent\/20 { background-color: rgba(110, 180, 255, 0.2) !important; }`
- Tailwind CDN이 로딩되지 않아도 **모든 색상이 즉시 표시됨**

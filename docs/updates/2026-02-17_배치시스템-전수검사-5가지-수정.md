## 작업 브랜치

## 변경 사항 요약

배치 시스템을 전수검사하여 발견한 5가지 문제를 모두 수정했습니다.

### 수정된 파일
| 파일 | 역할 | 변경 내용 |
|------|------|----------|
| `web/arm_server.py` (백엔드 서버) | 배치 체인 로직 | 위임 체계, 도구 설명 제거, max_tokens, replay API |
| `web/ai_handler.py` (AI 연동) | 배치 제출 | 3사 프로바이더 폴백 로직 |

### 수정 상세

| # | 문제 | 해결 |
|---|------|------|
| 1 | 위임 체계 안 지켜짐 — 비서실장이 처장 건너뛰고 전문가에게 직접 명령 | 4단계로 확장: 분류 → **처장 지시서** → 전문가 → 종합. 처장이 전문가별 구체적 업무 지시 작성 |
| 2 | 배치에서 도구 설명 포함 — AI가 가짜 코드 출력 | `_load_agent_prompt(include_tools=False)` + 배치 모드 안내 접미사 추가 |
| 3 | max_tokens 4096 부족 — 보고서 중간에 잘림 | 전문가/종합 8192, 분류 1024로 변경 |
| 4 | 작전일지 내부 대화 안 됨 | `correlation_id` 연결 + `/api/replay` API 완전 구현 |
| 5 | 배치 실패 시 복구 안 됨 | 프로바이더 폴백 자동 재시도 (Anthropic 실패 → Gemini/OpenAI로) |

### 변경된 배치 체인 흐름

**변경 전** (3단계):
```
CEO → 분류 → 전문가(CEO 원본 그대로) → 처장 종합
```

**변경 후** (4단계):
```
CEO → 1.분류 → 2.처장 지시서(실시간) → 3.전문가(지시서+원본) → 4.처장 종합
```

# GitHub Actions 자동 배포 설정

## 작업 날짜
2026-02-14

## 작업 브랜치
claude/sync-multiple-computers-Nz8IR

## 변경 사항 요약

### 뭘 했는지
GitHub에 코드를 올리면 Oracle Cloud 서버에 **자동으로 반영**되는 시스템을 만들었습니다.

### 왜 했는지
기존에는 코드를 수정하면 **직접 서버에 접속해서 파일을 복사**해야 했습니다.
자동 배포를 설정하면 코드를 GitHub에 올리기만 하면 서버에 자동으로 적용돼서, 수동 작업이 필요 없어집니다.

### 작동 방식 (쉽게 설명)
```
코드 수정 → GitHub에 올림 → main에 합쳐짐 → 서버에 자동 반영!
```

### 상세 작업 내역

1. **GitHub Actions 워크플로우 파일 생성**
   - `.github/workflows/deploy.yml` 파일 추가
   - main 브랜치(메인 코드)에 변경이 생기면 자동 실행
   - `web/` 폴더 안의 파일이 바뀔 때만 실행 (불필요한 배포 방지)

2. **GitHub Secrets 등록** (CEO가 직접 완료)
   - `SERVER_SSH_KEY` — 서버 접속용 SSH 비밀키(열쇠)
   - `SERVER_IP` — 서버 주소 (168.107.28.100)

3. **서버에 Git 저장소 동기화**
   - 서버의 `/home/ubuntu/CORTHEX_HQ` 폴더를 GitHub과 동기화

### 새로 만든/수정한 파일
- `.github/workflows/deploy.yml` (자동 배포 워크플로우) — main에 코드 합쳐지면 서버에 자동 배포
- `docs/project-status.md` (프로젝트 현재 상태) — 자동 배포 완료 기록

## 현재 상태

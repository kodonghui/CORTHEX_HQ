# UX/UI 10가지 수정

## 버전
0.07.003

## 작업 날짜
2026-02-15

## 작업 브랜치
claude/uxui-10-fixes

## 변경 사항 요약

CEO가 직접 찾은 10가지 UX/UI 문제를 수정했습니다.

### 수정한 파일
- `web/templates/index.html` (웹 대시보드 메인 화면)

### 10가지 수정 내용

| # | 문제 | 수정 내용 |
|---|------|----------|
| 1 | 명령창에 "+" 버튼이 빈 줄 차지 | 프리셋(즐겨찾기 명령)이 없을 때 빈 줄 자체를 숨김 |
| 2 | 글씨체가 안 예쁨 | Inter → **Pretendard** 변경 (카카오/토스/네이버가 쓰는 한국어 전용 폰트) |
| 3 | 밝은 모드가 너무 흰색 | 배경을 **따뜻한 라벤더(연보라)** 톤으로 조정, 눈 편하게 |
| 4 | 예약 "활성" 뱃지 정렬 어긋남 | 뱃지를 둥근 알약 모양(rounded-full)으로 바꾸고 정렬 개선 |
| 5 | 단계목록 "{prev_result}" 이해 불가 | `"이 단계에서 AI에게 시킬 작업을 입력하세요"`로 변경, TIP도 쉬운 한국어로 |
| 6 | 워크플로우 태그 정렬 안 맞음 | 태그 간격과 정렬 개선, flex-wrap 추가 |
| 7 | 지식 파일 생성해도 목록에 안 뜸 | **버그 수정**: API가 `{entries: [...]}` 형태로 반환하는데 프론트엔드가 전체를 배열로 취급 → `data.entries`로 수정 |
| 8 | 밝은 모드에서 탭 글씨 안 보임 | 활성 탭 색상 `text-white` → `text-hq-accent`(파란색)로 변경 |
| 9 | "대기" 누르면 "오류" 표시 + "(지휘자)" 의미불명 | **버그 수정**: API는 `status:"ok"` 반환하는데 프론트엔드가 `overall`을 체크해서 항상 오류. `status` 필드도 체크하도록 수정. "(지휘자)" → "서버"로 변경, 에이전트 수/텔레그램 상태도 표시 |
| 10 | 빌드 번호 혼동 | 현재 최신 빌드: **#77** (GitHub Actions 확인). 웹에 #74로 보이면 **Ctrl+Shift+R**(강력 새로고침) 필요 |

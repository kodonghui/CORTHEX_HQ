## 문제 상황

CEO가 클로드 코드와 대화할 때, 대화가 길지 않은데도 클로드가 직전 대화 내용을 까먹는 문제가 발생했습니다.

### 원인 분석

클로드 코드가 대화를 까먹는 이유는 **"보이지 않는 대화량"** 때문입니다:
- CEO가 보는 대화는 짧지만, 클로드가 파일을 읽고 명령어를 실행하면 **숨겨진 데이터가 수천 줄씩 쌓임**
- 클로드의 기억 용량(컨텍스트 윈도우)이 98% 차면 자동으로 오래된 대화를 지움
- 이때 **초반 대화 내용이 가장 먼저 사라져서** "갑자기 까먹은 것처럼" 보임

---

## 변경 사항 요약

### 1. 새로 만든 파일

| 파일 | 설명 |
|------|------|
| `docs/project-status.md` (프로젝트 현재 상태 파일) | 클로드가 새 세션을 시작하거나 기억이 리셋될 때, 이 파일을 읽으면 프로젝트 상태를 바로 파악할 수 있음 |
| `.claude/settings.json` (클로드 설정 파일) | 대화가 압축되기 직전에 자동으로 중요한 정보를 project-status.md에 저장하라는 명령(PreCompact Hook) 설정 |

### 2. 수정한 파일

| 파일 | 수정 내용 |
|------|-----------|
| `CLAUDE.md` (클로드 작업 규칙 파일) | "기억력 보완 규칙" 섹션 추가 — 세션 시작 시 project-status.md를 읽고, 작업 완료 시 업데이트하라는 규칙 |

---

## 이 시스템이 하는 일 (쉬운 설명)

1. **세션 시작할 때**: 클로드가 자동으로 `docs/project-status.md`를 읽어서 "아, 프로젝트가 여기까지 진행됐구나"를 파악
2. **대화가 압축될 때**: PreCompact Hook이 작동해서, 압축 전에 중요한 정보를 `docs/project-status.md`에 자동 저장
3. **작업이 끝날 때**: 클로드가 `docs/project-status.md`를 최신 상태로 업데이트

이렇게 하면, 클로드가 기억을 잃어도 파일을 통해 맥락을 복구할 수 있습니다.

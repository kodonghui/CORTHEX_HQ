2026-02-14

## 작업 브랜치

## 검사 방법
- 백엔드 API 엔드포인트(endpoint, 서버가 받아주는 주소) 50개+ 전수 확인
- 프론트엔드 HTML/JS 코드 3,500줄+ 전수 확인
- 백엔드에 있는데 프론트엔드에 없는 기능, UX 불편사항, 버그를 분류

## 🔴 심각 (버그/깨진 기능) — 3건

### 1. HTML 구조 깨짐: 보내기 버튼 중복
- **위치**: `index.html` 1161~1176줄
- **문제**: 명령 입력창 아래에 "보내기" 버튼이 **2개** 있음. 두 번째 버튼은 `<form>` 태그 없이 `</form>`으로 닫혀 있어 HTML이 깨져 있음
- **증상**: 화면에 보내기 버튼이 2개 보이거나, 레이아웃이 밀릴 수 있음
- **해결**: 두 번째 중복 버튼(1161~1176줄)과 `</form>` 태그 삭제

### 2. SNS 연결 상태 API 누락
- **문제**: 프론트엔드가 `/api/sns/oauth/status`를 호출하는데, 백엔드에 이 엔드포인트가 없음
- **증상**: SNS 관리 탭에서 플랫폼 연결 상태가 항상 "미연결"로 표시
- **해결**: 백엔드에 `/api/sns/oauth/status` 엔드포인트 추가

### 3. 작업 취소 기능 미구현
- **문제**: 프론트엔드에 "취소" 버튼이 있고 WebSocket으로 `cancel` 메시지를 보내지만, 백엔드에서 `cancel` 타입 메시지를 처리하는 코드가 없음
- **증상**: 취소 버튼을 눌러도 실제로 작업이 멈추지 않음 (UI만 바뀜)
- **해결**: 백엔드 WebSocket 핸들러에 cancel 처리 로직 추가

## 🟠 중요 (백엔드에 있는데 프론트엔드에 없는 기능) — 7건

### 4. 출판·기록 본부 (CPO) 조직도에서 빠짐
- **백엔드**: `publishing/division_head.py`에 출판 본부장(CPO)이 구현되어 있음
- **프론트엔드**: 조직도(왼쪽 트리)와 사무실 뷰 모두에서 출판 본부가 없음
- **영향**: CEO가 출판 본부 에이전트를 볼 수도 없고, 클릭해서 설정할 수도 없음

### 5. 도구 풀(Tool Pool) 5개만 표시 — 실제 50개+
- **백엔드**: 50개 이상의 도구(주식분석, DART, 네이버뉴스, 법률검색, 해시태그 추천 등)가 구현됨
- **프론트엔드**: 변리사, 세무사, 디자이너, 번역가, 웹검색 5개만 하드코딩되어 있음
- **해결 아이디어**: `/api/tools` API를 호출해서 실제 도구 목록을 동적으로 표시

### 6. SNS 플랫폼 3개 누락
- **백엔드**: 7개 플랫폼 지원 (Instagram, YouTube, LinkedIn, Tistory, **네이버블로그, 네이버카페, 다음카페**)
- **프론트엔드**: 4개만 표시 (Instagram, YouTube, LinkedIn, Tistory)
- **영향**: 네이버/다음 플랫폼으로 게시할 수 없음

### 7. CEO 만족도 통계 화면 없음
- **백엔드**: `GET /api/feedback` — CEO가 좋아요/아쉬워요를 누른 통계를 집계
- **프론트엔드**: 개별 결과에 좋아요/아쉬워요 버튼은 있지만, **전체 통계를 보는 화면이 없음**
- **해결 아이디어**: 대시보드에 만족도 카드 추가 (예: "만족도 85%, 총 23건")

### 8. 예산(Budget) 편집 기능 없음
- **백엔드**: `budget.yaml`에서 일일/월간 한도를 관리
- **프론트엔드**: 예산 **현황**만 표시하고, 한도를 **수정**하는 UI가 없음
- **영향**: 예산을 바꾸려면 직접 YAML 파일을 수정해야 함

### 9. 작업 비교(Compare) 기능 미완성
- **프론트엔드**: "비교하기" 버튼이 있고 체크박스로 작업 2개를 선택할 수 있지만, `openComparison()` 함수의 실제 비교 UI가 없음
- **영향**: 버튼만 있고 클릭하면 아무 일도 안 일어남

### 10. 예약(Schedule) 삭제 API 연동 누락
- **프론트엔드**: 삭제 버튼이 있지만 `deleteSchedule()` 함수가 정의되지 않았거나 불완전함
- **해결**: 함수 구현 확인 및 `/api/schedules/{id}` DELETE 연동 확인 필요

## 🟡 개선 제안 (UX 불편사항) — 8건

### 11. 모바일 대응 없음
- **현상**: 사이드바 280px 고정, 탭이 10개로 가로 스크롤 필요
- **문제**: 핸드폰/태블릿에서 사용 불가능
- **해결 아이디어**: 반응형(responsive) 레이아웃 — 모바일에서 사이드바를 접을 수 있게

### 12. 탭이 너무 많음 (10개)
- **현재 탭**: 홈, 명령, 성능, 작업내역, 예약, 워크플로우, 로그, 지식, 아카이브, SNS
- **문제**: 한 화면에 다 안 보임, 작은 화면에서는 스크롤 필요
- **해결 아이디어**:
  - 자주 쓰는 탭 5개만 기본 표시, 나머지는 "더보기" 메뉴
  - 또는 카테고리 그룹핑 (운영 / 데이터 / 설정)

### 13. 활동 로그가 페이지 새로고침하면 사라짐
- **현상**: WebSocket 로그는 메모리에만 저장 → 새로고침하면 전부 날아감
- **해결 아이디어**: 서버에 최근 로그를 저장하는 API 추가, 또는 localStorage에 임시 저장

### 14. 작업 내역에 페이지네이션(쪽 나누기) 없음
- **현상**: 작업이 많아지면 전부 한 번에 로드 → 느려질 수 있음
- **해결 아이디어**: 20개씩 나눠서 로드 + "더 보기" 버튼

### 15. 대화 내역 내보내기(Export) 없음
- **현상**: 결과를 "복사" 버튼으로 하나씩만 복사 가능
- **해결 아이디어**: 전체 대화를 Markdown 파일로 다운로드하는 버튼 추가

### 16. 키보드 단축키 없음
- **현재**: Enter로 전송, Shift+Enter로 줄바꿈만 지원
- **해결 아이디어**:
  - `Ctrl/Cmd + K`: 명령 입력창으로 포커스
  - `Ctrl/Cmd + 1~5`: 탭 전환
  - `Esc`: 모달 닫기 (일부만 구현됨)

### 17. 에이전트 작업 완료 후 상태가 "idle"로만 변함
- **현상**: 작업 완료(done) → 바로 idle로 변경 → CEO가 "누가 일했는지" 보기 어려움
- **해결 아이디어**: 완료 후 5~10초간 "done" 상태(초록불)를 유지하다가 idle로 전환

### 18. 다크/라이트 모드 전환 시 일부 요소 색상 안 바뀜
- **현상**: 라이트 모드에서 일부 `bg-hq-bg/50`, `bg-hq-surface/50` 등의 투명도 요소가 어색해 보일 수 있음
- **해결**: 라이트 모드 CSS 오버라이드 추가 점검 필요

## 🔄 상세 검토 중 (대표님 결정 대기)

### S1. WebGPU 파티클 NEXUS
- **뭔가?**: 현재 NEXUS 3D의 70개 정적 노드를 → WebGPU 컴퓨트 셰이더 기반 **10만~100만개 파티클**로 업그레이드
- **왜 좋은가?**: 에이전트가 분석 중이면 해당 노드에서 빛 파티클이 쏟아지고, 부서 간 데이터 전달이 빛줄기로 보임. "자비스 홀로그램" 같은 느낌
- **어떻게?**: Three.js r171+ `three/webgpu` import 변경 → TSL(Three Shader Language)로 파티클 셰이더 → GPGPU 컴퓨트로 GPU에서 직접 물리 연산
- **난이도**: 중 (기존 Three.js 코드 80% 재사용, 셰이더 학습 필요)
- **참고**: [WebGPU 마이그레이션 가이드](https://www.utsubo.com/blog/webgpu-threejs-migration-guide) | [TSL 필드 가이드](https://blog.maximeheckel.com/posts/field-guide-to-tsl-and-webgpu/) | [은하 파티클 튜토리얼](https://threejsroadmap.com/blog/galaxy-simulation-webgpu-compute-shaders)

### S2. 에이전트 장기 기억 시스템 (Observational Memory)
- **뭔가?**: 에이전트 29명이 이전 분석/대화를 **크로스-세션으로 기억**하게 하는 시스템
- **왜 좋은가?**: CIO가 "지난번 NVDA 분석에서 뭘 틀렸는지" 기억. CSO가 "저번 사업기획 피드백" 참고. 학습하는 직원.
- **어떻게?**: SQLite `agent_memories` 테이블 (에피소딕/시맨틱/절차적 3종) → 분석 완료 시 자동 기억 추출 → 다음 분석 시 프롬프트에 주입. 벡터DB 불필요 (프롬프트 캐싱 활용)
- **난이도**: 중
- **참고**: [Observational Memory 비용 10배 절감](https://venturebeat.com/data/observational-memory-cuts-ai-agent-costs-10x-and-outscores-rag-on-long) | [3가지 장기 기억 유형](https://machinelearningmastery.com/beyond-short-term-memory-the-3-types-of-long-term-memory-ai-agents-need/) | [ICLR 2026 MemAgents 워크숍](https://openreview.net/pdf?id=U51WxL382H)

### A2. MCP (Model Context Protocol) 도구 표준화
- **뭔가?**: CORTHEX의 89개 도구를 Anthropic이 만든 **업계 표준 프로토콜**로 감싸서, Claude/GPT/Gemini 누구든 같은 방식으로 도구를 쓸 수 있게 하는 것
- **왜 좋은가?**: 현재 tools.yaml 수동 관리 → MCP 서버로 자동화. 외부 500+ 공개 MCP 서버 즉시 연결 가능. OpenAI + Google + Linux Foundation이 공동 표준으로 채택.
- **어떻게?**: Python FastMCP SDK → `@mcp.tool()` 데코레이터로 기존 도구 래핑 → MCP 서버로 실행 → 모든 AI 클라이언트가 JSON-RPC로 호출
- **난이도**: 중
- **참고**: [MCP 공식 Python SDK](https://github.com/modelcontextprotocol/python-sdk) | [MCP 서버 빌드 튜토리얼](https://modelcontextprotocol.io/docs/develop/build-server) | [MCP 아키텍처 패턴 (IBM)](https://developer.ibm.com/articles/mcp-architecture-patterns-ai-systems/)

### B3. Claude Agent SDK 연동
- **뭔가?**: Anthropic 공식 에이전트 SDK를 CORTHEX에 연결해서, 현재 수동 구현한 `_manager_with_delegation()` 멀티에이전트 오케스트레이션을 SDK 레벨로 대체
- **왜 좋은가?**: 서브에이전트 병렬 실행 / 컨텍스트 격리 / 가드레일 / 트레이싱이 SDK에서 기본 제공. 서브에이전트는 하위 모델(Haiku)로 돌려서 비용 절감.
- **어떻게?**: `claude-agent-sdk-python` 설치 → `ClaudeSDKClient` + `subagents_enabled=True` → 기존 처장/전문가 구조를 SDK 오케스트레이터/서브에이전트로 매핑
- **난이도**: 상 (기존 아키텍처 상당부분 교체)
- **참고**: [Claude Agent SDK 공식](https://platform.claude.com/docs/en/agent-sdk/overview) | [서브에이전트 문서](https://platform.claude.com/docs/en/agent-sdk/subagents) | [2026 완전 가이드](https://www.contextstudios.ai/blog/claude-code-agent-sdk-the-complete-beginners-guide-to-building-ai-agents-2026)

## ⬜ 보류 (나중에 도전)

### A1. 다크 글래스모피즘 UI 리디자인
- **뭔가?**: CORTHEX 전체 UI를 2026년 트렌드인 **Dark Glassmorphism** 스타일로 변경. 반투명 유리 효과 + 네온 그라디언트 + 다크모드.
- **왜 좋은가?**: Tailwind `backdrop-blur-xl` + `bg-white/5`로 구현 가능. "아이언맨 연구실" 느낌. 기존 hq-* 토큰 시스템과 호환.
- **구체적 현상**: 현재는 불투명 카드 + 단색 배경. 변경 후 카드가 반투명 유리처럼 보이고 뒤에 보라/청록 빛이 은은하게 비침.
- **난이도**: 하 | **쿨팩터**: ★★★★☆
- **참고**: [Dark Glassmorphism 2026](https://medium.com/@developer_89726/dark-glassmorphism-the-aesthetic-that-will-define-ui-in-2026-93aa4153088f) | [Glassmorphism 가이드](https://www.designstudiouiux.com/blog/what-is-glassmorphism-ui-trend/)

### S3. 음성 사령관실 ("자비스, 시장 분석해")
- **뭔가?**: Web Speech API(브라우저 내장, 무료)로 음성 입력 → 기존 채팅 API → TTS로 음성 응답. 마이크 버튼 하나 추가.
- **왜 좋은가?**: 키보드 없이 "CIO 시장 분석해" 말하면 실행. 모바일에서 특히 편함. 음성 웨이브 애니메이션으로 시각적 피드백.
- **구체적 현상**: 헤더에 마이크 아이콘 → 탭하면 음성 인식 시작 → 말 끝나면 자동 전송 → 응답이 TTS로 읽어줌
- **난이도**: 하 | **쿨팩터**: ★★★★★
- **참고**: [Voice AI 2026](https://elevenlabs.io/blog/voice-agents-and-conversational-ai-new-developer-trends-2025) | [ElevenLabs](https://elevenlabs.io/)

### A3. AI-Trader 벤치마크 (CIO 성적표)
- **뭔가?**: 홍콩대학교 AI-Trader 벤치마크를 CORTHEX에 적용. CIO의 매매 판단 정확도를 업계 표준 기준으로 자동 채점.
- **왜 좋은가?**: "CIO가 잘하는 건지 못하는 건지" 객관적 숫자로 알 수 있음. 다른 AI 트레이더(GPT-5, Gemini 등)와 비교 가능.
- **구체적 현상**: 대시보드에 CIO 성적표 카드 추가 — 수익률, 승률, 샤프비율, 업계 평균 대비 순위
- **난이도**: 중 | **쿨팩터**: ★★★★☆
- **참고**: [AI-Trader 벤치마크](https://arxiv.org/abs/2512.10971) | [GitHub](https://github.com/HKUDS/AI-Trader)

### B1. 사이버펑크 터미널 모드
- **뭔가?**: Arwes / Cybercore CSS 기반 "터미널 모드" 토글. 전환하면 CORTHEX가 해커 영화 속 UI처럼 변함.
- **왜 좋은가?**: 네온 텍스트 + 글리치 효과 + 스캔라인 + 타이핑 애니메이션. 시연/데모할 때 임팩트 극대화.
- **구체적 현상**: 설정에서 "터미널 모드" ON → 전체 UI가 검은 배경 + 녹색/시안 텍스트 + 모노스페이스 폰트로 전환
- **난이도**: 중 | **쿨팩터**: ★★★★☆
- **참고**: [Arwes SF UI](https://arwes.dev/) | [Cybercore CSS](https://dev.to/sebyx07/introducing-cybercore-css-a-cyberpunk-design-framework-for-futuristic-uis-2e6c)

### B2. 에이전트 자동 품질 벤치마크
- **뭔가?**: Galileo AI 프레임워크 방식으로 에이전트 29명의 분석 품질을 자동 채점 파이프라인 구축. 현재 Phase 10 Soul 진화를 한 단계 업그레이드.
- **왜 좋은가?**: 주간 리포트에 "이번 주 최고 에이전트: 전문가A (정확도 94%)" 같은 랭킹 자동 생성.
- **구체적 현상**: 매주 일요일 크론 → 전문가별 분석 결과 자동 채점 → 품질 대시보드에 점수/랭킹 표시 → 하위 에이전트 Soul 자동 조정
- **난이도**: 중 | **쿨팩터**: ★★★☆☆
- **참고**: [Galileo AI 평가 프레임워크](https://galileo.ai/blog/agent-evaluation-framework-metrics-rubrics-benchmarks) | [Microsoft 에이전트 성능 측정](https://www.microsoft.com/en-us/dynamics-365/blog/it-professional/2026/02/04/ai-agent-performance-measurement/)

## 🔴 개어려운데 개쩌는 기술 (SS급)

### SS1. 디지털 트윈 — CORTHEX의 "가상 분신"
- **뭔가?**: CORTHEX 전체 시스템(29명 에이전트 + 시장 데이터 + 매매 로직)의 **실시간 디지털 복제본**을 만들어서, 실제 돈 한 푼 안 쓰고 전략을 시뮬레이션
- **왜 좋은가?**: "만약 CIO가 어제 NVDA 대신 AAPL을 샀다면?" 같은 **What-if 시나리오**를 가상에서 돌려볼 수 있음. 실제 시장 데이터로 백테스트가 아니라 **실시간 병렬 시뮬레이션**.
- **구체적 현상**: NEXUS 3D에서 "시뮬레이션 모드" 켜면 가상 CORTHEX가 실제와 나란히 돌아감. 화면 분할로 실제 vs 가상 비교.
- **왜 어려운가?**: 29명 에이전트의 상태/기억/도구를 전부 복제해야 함. 실시간 시장 데이터 2중 파이프라인. 비용 2배.
- **난이도**: 최상 | **쿨팩터**: ★★★★★★
- **참고**: [디지털 트윈 2026 진화](https://www.rtinsights.com/digital-twins-in-2026-from-digital-replicas-to-intelligent-ai-driven-systems/) | [AI 디지털 트윈 논문](https://arxiv.org/html/2601.01321)

### SS2. 뉴로-심볼릭 AI — "할루시네이션 킬러"
- **뭔가?**: LLM(신경망)과 지식그래프(논리 규칙)를 결합. CIO가 "NVDA 매수" 판단하면, 지식그래프가 논리적으로 **교차검증**해서 할루시네이션 차단.
- **왜 좋은가?**: 현재 CIO의 최대 약점 = 할루시네이션(없는 데이터를 상상). 지식그래프가 "이 숫자 실제로 존재하나?" 자동 확인.
- **구체적 현상**: CIO 분석 → 지식그래프 검증 → "경고: CIO가 참조한 NVDA PER 12.5는 실제 25.3과 불일치" 자동 플래그
- **왜 어려운가?**: 금융 지식그래프 구축 자체가 대형 프로젝트. Neo4j 같은 그래프DB 필요. 학술 레벨 연구.
- **난이도**: 최상 | **쿨팩터**: ★★★★★★
- **참고**: [뉴로-심볼릭 AI 위키](https://en.wikipedia.org/wiki/Neuro-symbolic_AI) | [지식그래프+LLM 추론 논문](https://arxiv.org/pdf/2302.07200)

### SS3. 에이전트 디베이트 프로토콜 — "AI 토론회"
- **뭔가?**: 에이전트끼리 **자동 토론**시켜서 더 나은 결론 도출. CIO 매수 판단 → 레드팀 에이전트가 "왜 매도해야 하는지" 반박 → 재반박 → 최종 판단 품질 향상
- **왜 좋은가?**: 인간 조직에서 "악마의 변호인"을 두는 것과 같은 원리. AI 레드팀이 매번 자동으로 반론을 제기해서 의사결정 품질 극적 향상.
- **구체적 현상**: 분석 로그에 "🔴 레드팀 반론: NVDA P/E 과대평가 리스크" → "🟢 CIO 재반박: AI 인프라 투자 사이클 고려 시 정당화" → 최종 신뢰도 상향/하향 자동 조정
- **왜 어려운가?**: 토론 프로토콜 설계 + 종료 조건 + 판정 로직. 비용 3배(분석 1회당 AI 호출 3배). 무한 루프 방지.
- **난이도**: 상 | **쿨팩터**: ★★★★★
- **참고**: [Agentic Red Teaming 2026](https://cloudsecurityguy.substack.com/p/why-agentic-ai-red-teaming-will-explode) | [자율 진화 레드팀 AutoDAN-Turbo](https://www.contextstudios.ai/glossary/automated-red-teaming)

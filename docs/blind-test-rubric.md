# 투자분석 블라인드 모델 비교 — 평가 루브릭

> 2026-02-23 작성. 리트마스터 3차 검수 루브릭 구조를 투자분석에 1:1 매핑.
> 원본: `docs/leet-master-qa-rubric.md`

## 테스트 개요
- **CIO**: `gpt-5.2-pro` × `high` 고정
- **전문가 4명**: 3라운드 (Claude Sonnet 4.6 / Gemini 3.1 Pro / GPT-5.2)
- **진행**: 대표님이 직접 모델 변경 + 분석 실행 + 보고서 블라인드로 제공
- **평가**: Claude가 이 루브릭으로 블라인드 채점

## 설계 원칙 (리트마스터 기반)
1. **1/3/5 이산 채점** — 추상적 형용사 금지, 구체적 오류 유형+예시로 기준 서술
2. **★치명적(critical) 항목**: Q1이 1점이면 → Q0(전체 만족도) 최대 2.0 제한
3. **10개 평가 차원** — 리트마스터 Q0~Q9 구조를 투자분석에 1:1 매핑
4. **전문가 유형별 보너스** — 4명의 핵심 역할 수행 여부 ±0.5점 보정
5. **동일 루브릭 3회 적용** — 모델만 바뀌고 기준은 완전 동일 → 공정 비교

## 평가 차원 (10개)

### 【Q0. 전체 만족도】 가중치: 별도 독립 평가

> "이 보고서만 보고 바로 매매 결정을 내릴 수 있는가?"

| 점수 | 기준 |
|------|------|
| **1** | **매매 결정 불가**. 도구 미사용으로 데이터 없음, 분석 구조 없음, 또는 할루시네이션으로 신뢰 불가. 이 보고서를 읽고 "이걸 믿어도 되나?"라는 의심이 듦 |
| **3** | **참고는 되지만 단독 결정 근거로 부족**. 일부 데이터가 있고 방향성은 잡히지만, 빠진 분석이 있거나 논리 비약이 있어 "조금 더 확인해야겠다"고 느낌 |
| **5** | **무편집 즉시 활용 가능**. 데이터 충분, 논리 명확, 리스크 명시, 실행 전략 구체적. CEO가 읽고 바로 "이대로 하자/하지 말자" 결정 가능 |

### 【★Q1. 도구 활용 & 데이터 정확성】 가중치: 20%, critical

> Q1이 1점이면 Q0 최대 2점 제한.

| 점수 | 기준 |
|------|------|
| **1** | 도구를 **전혀 사용하지 않음**(0회 호출), 또는 도구 데이터를 무시하고 자기 상식으로 분석. 또는 **존재하지 않는 종목코드/재무 수치를 지어냄**. 예: "삼성전자 PER 5배" (실제 10배) |
| **3** | 핵심 도구 **1~2개** 사용, 일부 데이터를 분석에 반영. 하지만 해당 전문가에게 주어진 도구 중 **절반 이상 미사용**. 또는 도구 데이터의 해석이 일부 부정확 |
| **5** | 해당 전문가의 핵심 도구를 **3개 이상** 체계적으로 사용. 모든 도구 결과가 분석의 **직접적 근거**로 활용됨. 도구가 반환한 수치와 보고서의 수치가 **일치** |

### 【Q2. 데이터 근거성】 가중치: 15%

> 투자분석에서 "지문" = 도구가 가져온 실제 데이터.

| 점수 | 기준 |
|------|------|
| **1** | 도구 데이터 밖의 **상식/추측을 핵심 판정 근거로 사용**. "일반적으로 반도체는 순환주이므로...", "AI 시장이 성장하고 있으므로..." 같은 도구 밖 근거가 핵심 논거 |
| **3** | 주된 근거는 도구 데이터이나, 핵심 판단 **1~2개**가 "통상적으로...", "보통..." 같은 **도구 밖 상식에 기댐** |
| **5** | 모든 핵심 판단이 **도구가 가져온 데이터에 직접 매핑**됨. "DART 2025Q3 기준 영업이익률 15.3%이므로..." 형태로 **추적 가능** |

### 【Q3. 수치 정밀성】 가중치: 15%

> 법률의 "주어/양태 정밀성" → 투자의 "수치/단위/시점 정밀성"

| 점수 | 기준 |
|------|------|
| **1** | 수치 오류 **2건 이상**. 단위 혼동(억↔조, %↔배), 시점 오류, 종목코드 불일치. 또는 구체적 숫자가 **3개 미만**이고 "높은 편", "양호한 수준" 같은 정성적 표현 대부분 |
| **3** | 큰 오류는 없으나 시점이 **모호한 곳 1~2개** ("최근 PER" — 언제?). 구체적 숫자 **5~10개** 포함하나 일부 출처 미표기 |
| **5** | 모든 수치에 **단위/시점** 명확. 종목코드/종목명 정확. YoY/QoQ 구분 체계적. 구체적 숫자 **10개+**, 모든 핵심 수치에 **출처와 시점** 명시 |

### 【Q4. 분석 논리】 가중치: 10%

| 점수 | 기준 |
|------|------|
| **1** | **왜 그렇게 판단했는지 재현 불가**. 데이터 나열만 하고 해석 없음. 또는 데이터와 결론이 **모순** |
| **3** | 흐름은 있으나 중간 연결이 **1~2곳 생략/비약**. 종합 판단 빠짐 |
| **5** | 데이터 확인 → 해석 → 시사점 → 결론이 **단계별로 선명**. 독자가 같은 결론에 도달 가능 |

### 【Q5. 분석 완결성】 가중치: 10%

| 점수 | 기준 |
|------|------|
| **1** | 핵심 분석 영역의 **절반 이하**만 다룸. 템플릿 필수 항목 다수 누락 |
| **3** | 대부분 다루나 일부가 **매우 피상적**. 균형 안 맞음 |
| **5** | 분석 영역을 **빠짐없이** 다루고, 각 항목의 깊이가 **일관**됨 |

### 【Q6. 리스크/반대논거 구체성】 가중치: 15%

> "왜 이 종목이 위험할 수도 있는지"를 구체적 수치로 제시하는가?

| 점수 | 기준 |
|------|------|
| **1** | 리스크 언급 **없거나** "주의 필요" 수준. "확실히 오른다", "반드시 매수" 같은 **확언**. 리트마스터로 치면 "오답인 이유: 다름" 수준 |
| **3** | 리스크 언급하나 **수치 없이 정성적**. "하락할 수 있다" (얼마나?). 매수 근거 5개 vs 리스크 1줄 |
| **5** | 리스크 각각을 **구체적 수치**로 제시 (MDD, 손절가 등). 매수/리스크 비율 **6:4~7:3**. "확실히/반드시" **0건** |

### 【Q7. 전문적 통찰】 가중치: 5%

| 점수 | 기준 |
|------|------|
| **1** | **검색 결과 요약 수준**. 누구나 아는 일반론. 전문 프레임워크 **미사용** |
| **3** | 프레임워크를 **이름만 언급**, 실제 데이터에 적용 안 함 |
| **5** | 전문 프레임워크를 **실제 데이터에 적용**하여 구체적 수치 도출 + 재사용 가능한 인사이트 |

### 【Q8. 가독성/간결성】 가중치: 5%

| 점수 | 기준 |
|------|------|
| **1** | 문장 과밀/중복/반복. **읽는 비용이 큼** |
| **3** | 읽히지만 일부 **장황/반복** |
| **5** | 정보 밀도 높고 반복 최소. **BLUF + 표/목록** 적절 활용 |

### 【Q9. CIO 보고 형식 준수】 가중치: 5%

| 점수 | 기준 |
|------|------|
| **1** | 보고 형식을 **전혀 따르지 않음**. 필수 섹션 대부분 누락 |
| **3** | 큰 틀은 맞으나 **필수 항목 1~2개 누락** |
| **5** | 보고 형식을 **정확히 준수**. 모든 필수 섹션 포함 |

## CIO 최종 보고서 평가 (전문가 4명과 별도)

> 전문가 4명의 개별 채점과 **별개로**, CIO가 작성한 최종 종합 보고서를 아래 기준으로 평가.
> 핵심 질문: "CIO가 전문가 분석을 잘 종합했나, 아니면 혼자 생각한 것만 썼나?"

### 【C1. 전문가 반영도】 가중치: 30%

> CIO 최종 보고서에 전문가 4명의 분석 결과가 **구체적으로 인용/반영**되었는가?

| 점수 | 기준 |
|------|------|
| **1** | 전문가 보고서를 **무시**하고 CIO 본인 생각만 기술. 전문가 분석 내용이 최종 보고서에 **0~1건** 반영. 또는 "전문가들의 의견을 종합하면..." 한 줄로 요약만 |
| **3** | 전문가 분석을 **일부 인용** (2~3명). 하지만 특정 전문가의 핵심 발견이 **누락**되거나, 인용이 피상적 ("시황 전문가에 따르면 시장 상승세" 정도) |
| **5** | **4명 전문가 모두**의 핵심 발견이 구체적으로 반영됨. "종목분석 전문가의 DCF 기준 적정가 75,000원" 같은 **구체적 수치 인용**. 전문가 간 의견 충돌 시 **어느 쪽을 왜 채택했는지** 명시 |

### 【C2. 빈 곳 보완력】 가중치: 30%

> 전문가 분석에서 부족한 부분을 CIO가 **자기 도구로 직접 분석하여 보완**했는가?

| 점수 | 기준 |
|------|------|
| **1** | 전문가 보고서를 **그대로 복붙/요약**만. CIO 독자적 분석 내용 **없음**. 취합만 하는 "비서" 수준 |
| **3** | CIO 독자 분석이 **일부** 있으나, 전문가가 이미 다룬 내용과 **중복**. 또는 도구 없이 CIO 상식으로 보완 |
| **5** | 전문가가 **놓친 영역**을 CIO가 도구로 직접 분석하여 보완. 예: 전문가가 거시지표를 안 다뤘으면 CIO가 직접 매크로 도구로 확인. **새로운 데이터 포인트**가 CIO 종합에 추가됨 |

### 【C3. 종합 판단력】 가중치: 25%

> 전문가들의 개별 분석을 **하나의 일관된 투자 판단**으로 통합했는가?

| 점수 | 기준 |
|------|------|
| **1** | 전문가 보고서를 **순서대로 나열**만. 종합 판단 없음. "시황은 이렇고, 종목은 이렇고, 기술은 이렇고..." 나열 후 끝 |
| **3** | 최종 매수/매도 결론은 있으나, **왜 그 결론인지** 전문가 분석과의 연결이 약함. 또는 전문가 간 의견이 **충돌**하는데 이를 해결하지 않고 넘어감 |
| **5** | 전문가 분석을 **교차 검증**. 충돌 의견에 대해 **어느 쪽이 더 신뢰할 만한지 근거 제시**. 최종 판단이 전문가 개별 분석의 **합보다 나은 통찰** 포함 |

### 【C4. 실행 구체성】 가중치: 15%

> 최종 매매 시그널이 **즉시 실행 가능할 만큼 구체적**인가?

| 점수 | 기준 |
|------|------|

## 총점 계산

```
[전문가 개별]
가중 평균 = (Q1×20 + Q2×15 + Q3×15 + Q4×10 + Q5×10 + Q6×15 + Q7×5 + Q8×5 + Q9×5) / 100

★Q1 critical 규칙: Q1이 1점이면 → Q0은 최대 2.0으로 제한

유형별 보너스: 가중 평균 + Σ(보너스) → 최종 점수 (범위: 1.0 ~ 5.0 클리핑)

[CIO 종합 보고서]
CIO 점수 = C1×30% + C2×30% + C3×25% + C4×15%

[라운드 최종]
전문가 평균 = 4명 전문가 최종 점수의 평균
CIO 종합 = CIO 점수 (별도 표기)
라운드 총점 = 전문가 평균 × 70% + CIO 종합 × 30%  ← 해당 모델의 최종 점수
```

## 판정 기준

| 등급 | 점수 범위 | 판정 |
|------|-----------|------|
| **S** | 4.5 ~ 5.0 | 탁월 — 전문 애널리스트급, 즉시 매매 결정 가능 |
| **A** | 3.5 ~ 4.4 | 우수 — 실전 투입 가능, 경미한 보완 필요 |
| **B** | 3.0 ~ 3.4 | 보통 — 참고 수준, 단독 의사결정 근거로는 부족 |
| **C** | 2.0 ~ 2.9 | 미흡 — 재작업 필요, 핵심 항목 누락 다수 |
| **F** | 1.0 ~ 1.9 | 부적격 — 도구 미사용, 할루시네이션, 구조 없음 |

## Executive Summary

CORTHEX v2는 "일은 AI agent가 하고, 나는 오퍼레이션만 제대로 잘 하면 되는" 멀티유저 SaaS 플랫폼이다. v1(Python/FastAPI 단일 사용자 시스템)을 **TypeScript(Bun+Hono) 풀스택**으로 완전 재구축하여, 인간 직원(H)이 ID/PW로 로그인해 자기 부하 AI 에이전트를 지휘하고 상위자에게 보고하는 조직 운영 시스템을 구현한다.

핵심 구조: 관리자 콘솔(별도 URL)에서 회사/직원/CLI/부서/에이전트/도구를 설정하면, 유저 앱(동일 UI)에서 CEO든 H든 자기 독립 워크스페이스에서 즉시 업무를 시작할 수 있다. 에이전트 실행은 해당 유저의 Claude CLI + API key로만 작동하며, 비용·데이터·협업 범위가 CLI 단위로 완전 격리된다. **다른 유저의 데이터는 단 한 건도 노출되지 않는다.**

비서를 둔 유저는 비서가 정리한 최종 보고서만 받는다. 비서 배정은 선택 사항이며, 관리자 모드에서 언제든 손쉽게 달아줄 수 있다.

v1에서 검증된 핵심 기능(비서실장 오케스트레이션, 위임-보고, 도구 자동호출, SNS 발행, 전략실, NEXUS)을 전부 이식하되, 멀티테넌시를 구조적으로 보장하는 새 아키텍처로 재설계한다.

### What Makes This Special

- **"사람 1명 = CLI 1개 = 에이전트 N명 = 독립 비용/데이터"** — 다른 AI 플랫폼에 없는 인간-AI 혼합 조직 모델. 유저는 코딩이 아니라 오퍼레이션(전략실, NEXUS, 채팅, 승인 등 전체 도구)에 집중
- **CLI 구독 모델** — 에이전트 수 무제한, CLI 구독 단위 과금. 경쟁 플랫폼(API 토큰 단위 과금) 대비 압도적 가격 구조
- **절대 격리** — H가 첫 로그인했을 때 "내 팀 + 내 보고서 + 내 승인 요청"만 보임. 다른 유저의 데이터는 단 한 건도 노출되지 않음
- **왜 지금** — Claude CLI Max 구독($0 에이전트 실행) + 자율 AI agent 기술 성숙. 1년 전에는 "시켜놓고 퇴근" 자체가 불가능했음

## User Journeys

### Journey 1: 관리자 — 새 직원(H) 온보딩

1. 관리자 콘솔 로그인
2. CORTHEX-HQ 회사 선택
3. "직원 추가" → ID/PW 생성 (예: minji / 임시비번)
4. 민지의 CLI 토큰 등록 + 회사 공용 API key 연동
5. "A본부 본부장"으로 배정 → 에이전트 3명(A-1, A-2, A-3) 할당
6. 도구 할당: A본부 = 금융도구 ON, 마케팅도구 OFF
7. (선택) 비서 에이전트 달아주기
8. 보고 라인 설정: 민지 → CEO
9. 완료 → 민지가 로그인하면 바로 업무 시작 가능
- **소요 시간 목표: 5분 이내**

### Journey 2: H(민지) — 첫 출근일

1. CEO로부터 ID/PW 전달받음
2. corthex-hq.com 접속 → 로그인
3. **"안녕하세요 민지님 👋"** + 내 팀(A-1, A-2, A-3) 표시
4. 문서함 비어있음, 보고서 0건 — 깨끗한 내 워크스페이스
5. 다른 사람 데이터 단 한 건도 안 보임 → **"내 공간이구나"**
6. 설정 페이지 → 개인 API key 등록 (KIS, 노션, 이메일, 텔레그램)
7. AI 에이전트 A-1에게 첫 채팅: "삼성전자 분석해줘"
8. A-1이 응답 → **"와, 진짜 내 직원이네"**

### Journey 3: H(민지) — 일상 업무 (출근~퇴근)

1. 출근 → 로그인
2. 홈화면: **"밤새 완료된 작업: 2건"** 알림
3. 어제 대화 이어가기:
   - A-1: "안녕하세요 민지님, 어제 요청하신 삼성전자 보고서 완성했습니다. 확인하시겠어요?"
   - 민지: "어, 고마워. 결론 부분 좀 더 보강해줘"
   - A-1: "네, 바로 수정하겠습니다" → **(세션 연속성 = 진짜 직원)**
4. SNS 게시물 승인 요청 확인 → 승인 or 반려
5. AI 에이전트와 새 업무 지시 (멀티턴 대화)
6. 전략실에서 KIS 매매 현황 확인
7. CEO에게 보고할 보고서 작성/검토 → **"CEO에게 보고" 버튼** 클릭
8. 퇴근 전: AI에게 야간 업무 지시 → "이거 밤새 분석해놔"
9. 퇴근 → 서버가 작업 큐에 등록 → AI가 H의 CLI로 야간 실행

### Journey 4: CEO(동희) — 보고 수신 + 지휘

1. 로그인
2. 홈화면: "H(민지)로부터 보고 1건" + "밤새 완료된 작업 3건"
3. 민지 보고서 열람 → 코멘트 달기로 피드백
4. (P3부터) 사내 메신저로 민지에게 직접 메시지: "이거 다시 해봐"
5. 비서실장에게 새 업무 지시 → 비서실장이 부서별 배분
6. 비서실장이 정리한 최종 보고서 수신
7. SNS 승인, 대시보드 비용 확인

### Journey 5: H(민지) — 에러 상황

**일반 에러:**
1. AI 에이전트에게 지시했는데 응답 없음
2. 에이전트 상태 확인 → "CLI 연결 끊김" 표시
3. 설정 페이지 → CLI 토큰 재연결 시도
4. 안 되면 → 관리자(CEO)에게 "CLI 재연동 요청"
5. 관리자가 관리자 콘솔에서 CLI 재연동
6. 민지 새로고침 → 에이전트 정상 작동

**보안 사고 (격리 위반):**
1. H가 "이상한 데이터 보임" 발견
2. 즉시 관리자에게 신고
3. 관리자가 해당 유저 세션 강제 종료
4. 격리 테스트 긴급 실행
5. 원인 파악 후 패치

## SaaS B2B + AI Agent Platform 요구사항

### 멀티테넌시 모델

| 항목 | 설계 |
|------|------|

| 역할 | 권한 |

# BMAD + OMG

> 누나가 던진 말: "bmad? omg — 이거 두 개 알지?"

---

## ⚠️ 맥락

이건 클로드 코드 자체의 도구/방법론 이야기.
CORTHEX 내부 에이전트 설정 아님.

---

## BMAD란?

**클로드 코드로 개발할 때 팀을 구성하는 표준 방법론**

- B: Breakthrough
- M: Method for
- A: Agile
- D: Development

### 핵심 원칙

```
1. 전문가 에이전트 여러 명 구성 (기획자, 개발자, QA...)
2. 큰 작업 전 반드시 플랜모드 → 컨펌 받고 실행
3. 각 에이전트가 자기 분야만 처리 (서로 넘나들지 않게)
4. 단계마다 검토 게이트
```

### 누나 방식

```
"각 전문가 몇십 마리 초빙해서 토론하고 플랜모드로 컨펌받으라해야함"
```

= 새 기능 시작할 때 전문가 에이전트들을 소환해서 토론 → 합의 → 실행

---

## OMG란?

**클로드 코드 플러그인 (스킬보다 더 강력한 단계)**

누나: "스킬급을 넘어서 이제 플러그인이라 불러. 비매그급으로 그걸로 오퍼레이팅하는거야 토큰을"

### 하는 일

**토큰 리밸런싱 자동화**:
- 작업 복잡도 판단
- 오퍼스가 할 일 / 소네트가 할 일 자동 분류
- 알아서 적절한 모델에 라우팅

누나는 OMG 나오기 전에 이미 직접 구축해놔서 안 씀.

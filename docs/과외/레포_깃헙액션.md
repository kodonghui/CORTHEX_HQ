# 레포 + 깃헙 액션

> 누나가 던진 말: "니 레포 관리는 하는거 맞지" / "깃헙액션은또어찌알고쓰는걸"

---

## 레포 (Repository, 저장소)

### 한 줄 요약
**코드를 저장하고 버전 관리하는 온라인 창고 (GitHub)**

### 현실 비유

```
일반 파일 저장 = 한 개의 Word 파일 저장
레포 = 클라우드 창고에 저장 + 수정 기록 전부 보관
     → "3일 전 버전으로 되돌리기" 가능
     → 여러 명이 동시에 수정해도 충돌 관리
```

### 레포 관리가 중요한 이유

```
관리 안 할 때:
→ 코드 망가지면 복구 불가
→ 여러 버전 헷갈림
→ 협업 시 파일 덮어쓰기 사고

관리 할 때:
→ 언제든 과거 버전으로 복귀
→ 브랜치(실험 공간) 만들어서 안전하게 테스트
→ 팀원/AI가 동시에 작업 가능
```

### CORTHEX 레포
- **주소**: https://github.com/kodonghui/CORTHEX_HQ
- **브랜치 전략**: `claude/작업명` → 자동 머지 → main

## 깃헙 액션 (GitHub Actions)

### 한 줄 요약
**"코드 업로드되면 자동으로 이것저것 해줘" 설정**

### 현실 비유

```
편의점 자동문 센서:
- 사람 감지 → 자동으로 문 열림

깃헙 액션:
- 코드 push 감지 → 자동으로 서버에 배포
```

### CORTHEX에서 동작 방식

```
대표님이 코드 수정 → "[완료]" 커밋 → GitHub push
       ↓
깃헙 액션 자동 실행:
1. Oracle 서버 SSH 접속
2. 최신 코드 받아오기
3. 서버 재시작
4. Notion에 배포 기록

→ 대표님은 push만 하면 자동 배포 완료
```

### CORTHEX deploy.yml 파일

```
.github/workflows/deploy.yml  ← 이 파일이 자동화 설정
```

이 파일이 바로 깃헙 액션 설정 파일. 누나가 "어찌 알고 쓰냐"고 한 것 = 비개발자치고 고급 설정임.

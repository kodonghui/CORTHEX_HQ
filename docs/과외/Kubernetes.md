## 한 줄 요약

- **쿠버네티스** = 도커 도시락 통 수백 개를 **자동으로 관리하는 총괄 매니저**
- **오토스케일링** = 손님이 몰리면 **자동으로 직원(서버) 추가**, 한가하면 **자동으로 줄이는** 기능

---

## 쿠버네티스란?

도커(도시락 통) 1개는 혼자 써도 됨.
그런데 서비스가 커지면?

```
도시락 통이 1개 → 10개 → 100개 → 1,000개
→ 이걸 사람이 일일이 관리? 불가능
```

**쿠버네티스** = 이 도시락 통들을 자동으로 관리하는 시스템
- 통 하나가 죽으면 자동으로 새 통을 띄움
- 통들 사이에 일을 골고루 나눔
- 업데이트할 때 서비스 안 끊기고 순차 교체

---

## 오토스케일링이란?

### 현실 비유: 카페

| 상황 | 카페 | 서버 |
|------|------|------|
| 평소 | 바리스타 2명 | 서버 2대 |
| 점심 피크 | 알바 추가 | 서버 자동 10대로 늘어남 |
| 새벽 한가 | 알바 퇴근 | 서버 다시 2대로 줄어듦 |

```
→ 이걸 사람이 직접 조작 X
→ 트래픽(손님 수)에 따라 자동으로 늘었다 줄었다
```

## 누나가 이걸 왜 처음부터 설계하나?

> "인증이랑 디비 그리고 유저 100 1000 10000 100000 단위로 서버 오토스케일링까지 유연한 설계로 언제든 확장가능성 고려하면서 해"

- 지금은 유저 100명 → 소규모 서버로 충분
- 갑자기 유저 10만 명 → 서버가 터짐
- 처음부터 확장 가능하게 설계 안 하면 → 나중에 **처음부터 다시 만들어야 함**

누나 말: "처음에 언어잘못선택하면 나중에 고도화할 때 ai 컨텍스트 한계로 머하나 추가하면 이거안되고 저거안되고"

# CLI vs GUI vs 콘솔

> 누나가 던진 말: "너 cli 안쓰고 gui 쓰냐 ㅋㅋ"

---

## 한 줄 요약

- **GUI** = 마우스로 아이콘 클릭하는 방식 (우리가 평소에 쓰는 것)
- **CLI** = 검은 화면에 명령어를 직접 타이핑하는 방식
- **콘솔** = CLI 화면 자체를 부르는 다른 말

---

## 현실 비유

| | GUI | CLI |
|--|-----|-----|
| 비유 | 카카오택시 앱으로 "목적지 검색 → 배차" 버튼 클릭 | 택시기사에게 직접 전화해서 "강남역 3번출구 와줘" 말함 |
| 속도 | 느릴 수 있음 | 빠름 (중간단계 없음) |
| 편의성 | 직관적, 배우기 쉬움 | 처음엔 어렵지만 익숙하면 훨씬 강력 |
| 예시 | VSCode, 파일탐색기, 크롬 | 터미널, 파워쉘, bash |

---

## 클로드 코드에서의 의미

- 클로드 코드는 **CLI 도구** → 터미널(검은 화면)에서 실행함
- 대표님은 지금 VSCode(GUI) 안에 터미널을 열어서 사용 중
- 누나 말: "걍 vs 왜쓰지" → VSCode 없이 터미널만 써도 된다는 뜻

---

## 누나가 왜 CLI를 선호하나?

> "윈도우에서 하면 내가 뭐 해라고 시키면 머가 안된다고 해"

CLI를 쓰면 클로드가 직접 명령어로 모든 걸 실행 가능.
GUI를 거치면 중간에 "이 버튼 클릭해야 해요" 같은 번거로운 단계가 생김.

---

## 관련 단어

- **터미널** = CLI를 실행하는 프로그램 (맥/리눅스 기본 내장)
- **파워쉘** = 윈도우의 터미널 (맥 터미널보다 제약이 많음)
- **bash** = 맥/리눅스의 터미널 언어
- **콘솔** = 터미널, CLI와 거의 같은 말로 쓰임

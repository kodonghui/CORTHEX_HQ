## Part 1: 리밸런싱 (Rebalancing)

### 한 줄 요약
**클로드 코드 작업 시 복잡도에 맞게 오퍼스/소네트 골라 쓰는 것**

### 누나가 본 문제

클로드 코드로 팀(여러 에이전트)을 구성할 때:
```
팀장 에이전트를 오퍼스로 설정
→ 그 에이전트가 단순 문서 작업까지 전부 직접 처리
→ 오퍼스 토큰 폭발적으로 소비 = 돈 낭비
```

### 올바른 방식

```
오퍼스:  아키텍처 설계, 복잡한 판단, 새로운 시스템 결정
소네트:  코딩, 버그 수정, 문서 작성, 알려진 패턴 작업
하이쿠:  단순 요약, 분류, 반복 작업
```

### 대표님 현재 상황

대표님은 이미 오퍼스 1M 많이 쓰고 있음 → 맞게 하는 것.
실제로 CLAUDE.md에도 이 기준표가 이미 들어있음:

| 소네트로 충분 | 오퍼스 필요 |
|-------------|-----------|
| 버그 수정 | 아키텍처 설계 |
| 문서 작성 | 멀티에이전트 디버깅 |
| 단순 코딩 | 원인 불명 버그 |

→ 이미 잘 하고 있는 것. 몰랐던 건 이름(리밸런싱)뿐.

### OMG가 뭔데?

누나: "그 리밸런싱을 OMG가 해주는 건데 나는 이미 팀별로 구축해놔서 안 씀"

= 클로드 코드 플러그인. **작업 복잡도를 판단해서 자동으로 오퍼스/소네트 배분해줌.**
누나는 이미 자기가 직접 설계해놔서 이 플러그인 필요 없다는 것.

## Part 2: OS = 오퍼레이팅 시스템

컴퓨터 OS(윈도우, 맥) 아님!

```
1단계 - 직접 코딩:   내가 코드 씀
2단계 - 바이브코딩:  AI한테 말로 시킴
3단계 - OS:          방향/의사결정만 내가, 실행은 AI팀 전부
```

누나 정의: "걍 의사결정하고 결과물의 음미만 할 줄 알면 돼"

대표님이 이미 3단계 하고 있음. 누나가 이름 알려준 것.

## 각 기술 선택지 비교

### 프론트엔드 (화면)

| 기술 | 특징 | 적합한 경우 |
|------|------|------------|
| **HTML/JS** (현재) | 단순, 빠르게 시작 | 개인 도구, 소규모 |
| **React** | 복잡한 UI, 컴포넌트 재사용 | 유저 수백~수천 이상 서비스 |
| **Vue** | React보다 쉬움 | 중간 규모 |

→ 지금 CORTHEX는 HTML로 충분. 서비스 외부 공개하면 React 고려.

### 백엔드 (서버)

| 기술 | 특징 | 적합한 경우 |
|------|------|------------|
| **Python** (현재) | AI 라이브러리 풍부, 빠른 개발 | AI 중심 서비스 |
| **Node.js** | 자바스크립트로 서버도 | 웹 서비스, 실시간 |
| **Go** | 빠름, 대용량 트래픽 | 대규모 서비스, 고성능 필요 시 |

→ AI 팀 서비스는 Python이 정답. 나중에 트래픽 터지면 Go 고려.

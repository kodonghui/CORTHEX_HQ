# 플랜모드 (Plan Mode)

> 누나가 던진 말: "플랜모드로 컨펌받으라해야함" / "니가 설계하라하면 콘솔창 아래에 녹색 폰트로 플랜모드라 켜짐"

---

## 한 줄 요약

클로드 코드에서 **"일단 계획만 세워, 코드는 건드리지 마"** 모드

---

## 현실 비유

**공사 전 설계도 검토**

```
건물 지을 때:
- 플랜모드 OFF → 클로드가 바로 공사 시작 (파일 막 수정)
- 플랜모드 ON  → 설계도만 그려서 보여줌, 망치는 안 듦
                  → 대표님이 "OK" 하면 그때 공사 시작
```

---

## 왜 중요한가?

> 누나: "처음에 팀빌딩할 때 각 전문가 몇십 마리 초빙해서 토론하고 플랜모드로 컨펌받으라해야함"

클로드 코드가 **혼자 판단해서 막 코드 짜면**:
- 방향이 틀려도 한참 일하고 나서야 알게 됨
- 되돌리기 어려운 변경이 생김
- 토큰 낭비

**플랜모드 먼저 → 대표님 확인 → 실행** 순서가 올바른 방식

---

## 실제로 어떻게 생겼나

클로드 코드 터미널에서:
```
클로드 코드 실행 중...

┌─────────────────────────────────────┐
│  ⚡ PLAN MODE                       │  ← 녹색으로 이렇게 표시됨
│  변경사항 없음. 계획만 제시합니다.    │
└─────────────────────────────────────┘

제가 하려는 작업:
1. arm_server.py 수정 → 새 API 엔드포인트 추가
2. index.html 버튼 추가
3. 테스트 실행

이대로 진행할까요? (y/n)
```

---

## CORTHEX에서 활용법

```
대표님이 명령 입력할 때:
"[플랜모드] ~~해줘" 라고 쓰면
→ 클로드가 코드 건드리지 않고 계획만 보여줌
→ 대표님이 OK 하면 실행
```

---

## 관련 개념

- **플랜 승인** = 계획 확인 후 OK
- **컨펌** = 확인/승인 (confirm)
- **게이트** = 다음 단계로 넘어가기 전 검토 포인트

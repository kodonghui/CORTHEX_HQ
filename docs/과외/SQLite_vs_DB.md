# SQLite vs 데이터베이스 종류

> 누나가 던진 말: "sqllite면 확장성보단 일단 무료+소규모핏이네"

---

## 한 줄 요약

- **SQLite** = 파일 하나짜리 소규모 DB. 설치 없이 바로 씀
- **PostgreSQL** = 제대로 된 서버 DB. 동시접속 많을 때
- **MongoDB** = 자유로운 구조의 데이터 저장

---

## 현실 비유

| DB 종류 | 비유 | 적합한 경우 |
|---------|------|------------|
| **SQLite** | 개인 다이어리 | 혼자 쓰거나 소규모 |
| **PostgreSQL** | 도서관 데이터베이스 | 동시에 많은 사람이 검색 |
| **MongoDB** | 자유 형식 메모장 | 데이터 형태가 제각각일 때 |

---

## SQLite의 특징 (현재 CORTHEX)

**장점**:
- 설치 불필요 → 파일 1개가 전체 DB
- 빠른 개발, 무료
- 소규모에는 충분히 빠름

**단점**:
- 동시 접속자 많으면 병목 (여러 명이 동시에 쓰면 줄 서야 함)
- 고급 기능 없음

---

## 언제 바꿔야 하나?

```
지금 CORTHEX: 대표님 혼자 + AI 팀원들 → SQLite 충분

만약 리트마스터 서비스 오픈:
→ 수백 명 동시접속 → SQLite 버거움
→ PostgreSQL로 마이그레이션 (이사) 필요
```

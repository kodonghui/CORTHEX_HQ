# Docker (도커) + 볼륨 맵핑

> 누나가 던진 말: "볼륨맵핑안해?" / "도커로 섭 띄우고 재시작했는데 히스토리 날라가면 야 ㅅㅂ 볼륨맵핑안해?"

---

## 한 줄 요약

- **도커** = 내 프로그램을 **도시락 통에 포장**해서 어디서든 똑같이 실행되게 하는 기술
- **볼륨 맵핑** = 그 도시락 통 안의 데이터를 **외부 통에도 동시에 저장**하는 설정

---

## 도커란?

### 문제 상황 (도커 없을 때)
```
나: "이 프로그램 실행해봐"
상대방: "나한텐 실행이 안 되는데요?"
나: "내 컴에서는 되는데?"
→ 서로 컴퓨터 환경이 달라서 생기는 문제
```

### 도커로 해결
도시락 통 비유:
- 음식(프로그램) + 조리환경(라이브러리/설정) 전부를 **통에 담아서** 배달
- 받는 사람이 어떤 컴을 써도 **통 그대로 실행** → 항상 동일한 결과

---

## 볼륨 맵핑이란?

### 문제 상황 (볼륨 맵핑 없을 때)

누나 말: "도커로 서버 띄우고 재시작했는데 히스토리 날아가면"

```
도커 컨테이너(도시락 통) 안 데이터는
→ 통을 껐다 켜면 전부 사라짐!
```

마치 **메모지를 도시락 통 안에만 써놓은 것** → 통 열고 닫으면 메모지 없어짐

### 볼륨 맵핑으로 해결

**도시락 통 안**과 **외부 메모장**을 연결해놓는 것:
```
도시락 통 안 기록 ←→ 내 컴퓨터 특정 폴더
```
→ 통을 껐다 켜도 외부 폴더에 데이터가 살아있음

---

## 실제 사용 예시 (누나 상황)

```
AI 팀원이 도커로 서버 실행
→ 볼륨 맵핑 안 함
→ 서버 재시작
→ 대화 기록/데이터 전부 증발
→ 누나: "야 ㅅㅂ 볼륨맵핑 안 해? 개발팀장 나와"
```

---

## 관련 용어

| 용어 | 뜻 |
|------|-----|
| 컨테이너 | 도커가 만든 도시락 통 1개 |
| 이미지 | 도시락 통의 레시피/설계도 |
| 볼륨 | 통 밖에 연결된 저장 공간 |
| 섭(서버) 띄운다 | 도커로 서버를 실행한다 |

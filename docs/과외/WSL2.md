# WSL2 (Windows Subsystem for Linux 2)

> 누나가 던진 말: "최소한 wsl2는 하겠지" / "wsl2 해봤자 맥 못따라와"

---

## 한 줄 요약

**윈도우 컴퓨터 안에 리눅스(맥과 비슷한 개발 환경)를 통째로 끼워넣는 기능**

---

## 현실 비유

아파트(윈도우) 안에 **컨테이너 하우스(리눅스)를 통째로 설치**하는 것.

- 겉은 윈도우 그대로
- 안에서는 리눅스 환경으로 개발 가능
- 두 환경 사이를 자유롭게 왔다갔다 가능

---

## 왜 필요한가?

```
개발자들이 쓰는 도구 대부분이 리눅스/맥 기준으로 만들어짐
→ 윈도우에서 그냥 쓰면 "이 명령어 없어요" 오류가 자주 남
→ WSL2 설치하면 그 오류들이 사라짐
```

누나 말: "ai가 뭘 복붙해서 로그 알려달라고 개병신짓을 하기시작해" = WSL2 없어서 명령어가 안 먹히는 상황

---

## 맥이 더 좋다는 이유

| | 윈도우 + WSL2 | 맥 |
|--|--------------|-----|
| 개발 환경 | 리눅스 따로 설치해야 함 | 처음부터 리눅스와 같은 계열 |
| AI와 협업 | 간혹 "윈도우라서 안 됨" 발생 | 거의 100% 명령어 그대로 실행 |
| 귀찮음 | 설정 단계 있음 | 꺼내서 바로 됨 |

---

## 대표님 현재 상황

- 윈도우(그램) + VSCode + 파워쉘 사용 중
- WSL2 없는 상태 → 클로드가 "이 명령어 안 돼요" 할 가능성 있음
- 누나 권고: **맥 사는 게 제일 깔끔함** (당근에서 m2/m3 중고)

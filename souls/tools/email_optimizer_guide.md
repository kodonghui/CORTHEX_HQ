# email_optimizer — 이메일 마케팅 최적화기 도구 가이드

## 이 도구는 뭔가요?
마케팅 이메일의 제목이 얼마나 효과적인지 점수를 매겨주고, 더 나은 제목을 제안해주는 도구입니다.
길이, 개인화, 긴급성, 숫자 사용, 이모지, 질문형, 혜택 제시 등 8가지 규칙으로 100점 만점 점수를 산정하고, A/B 테스트(두 개 제목 중 뭐가 더 효과적인지 비교 실험)용 제목 쌍도 만들어줍니다.

## 어떤 API를 쓰나요?
- **내장 규칙 엔진** (8가지 규칙으로 점수 자동 산정)
- **AI 분석** (개선안 생성, A/B 테스트 제목 쌍 생성)
- 비용: **무료** (외부 API 사용 없음)
- 필요한 키: 없음

## 사용법

### action=analyze (이메일 제목 분석)
```
action="analyze", subject="LEET 합격 비법 3가지를 알려드립니다", audience="수험생"
```
- 이메일 제목을 8가지 규칙으로 분석하여 100점 만점 점수를 매깁니다
- 본문(body)을 함께 입력하면 본문 길이, CTA(행동 유도 버튼) 존재 여부도 분석합니다
- AI가 장점/단점, 개선할 점 2가지, 개선된 제목 3개를 제시합니다

**점수 규칙 (총 100점):**

| 항목 | 배점 | 기준 |
|------|------|------|
| 길이 | 20점 | 20~50자 최적, 10자 미만이나 70자 초과 시 0점 |
| 개인화 | 15점 | "님", "{이름}" 등 개인화 요소 있으면 만점 |
| 긴급성 | 15점 | "마감", "오늘만" 등 긴급성 단어 1개면 만점, 2개 이상이면 8점 |
| 숫자/구체성 | 15점 | "3가지", "50%" 등 구체적 숫자가 있으면 만점 |
| 이모지 | 10점 | 0~2개면 만점, 3개 이상이면 3점 |
| 질문형 | 10점 | "?", "할까요" 등 질문형이면 만점 |
| 혜택 제시 | 15점 | "비법", "무료", "가이드" 등 혜택 키워드가 있으면 만점 |
| 스팸 감점 | -15점 | "당첨", "대박", "100%" 등 스팸 단어가 있으면 감점 |

**등급 기준:** A+ (90이상), A (80이상), B (70이상), C (60이상), D (50이상), F (50미만)

**예시:**
- `action="analyze", subject="님, LEET 합격률 300% 높이는 비법 지금 확인하세요"` → 85점(A등급): 개인화O, 숫자O, 긴급성O, 혜택O

### action=suggest (제목 개선안 생성)
```
action="suggest", subject="LEET 준비 안내", count=5
```
- 기존 제목의 점수를 확인하고, AI가 count개의 개선된 대안 제목을 생성합니다
- 각 대안 제목마다 왜 더 효과적인지 이유도 설명합니다
- 기본 5개 생성, count 파라미터로 개수 조절 가능

**예시:**
- `action="suggest", subject="서비스 업데이트 안내"` → 원래 35점(F) → 개선안 5개 (점수 70~90 예상)

### action=ab_test (A/B 테스트 제목 쌍 생성)
```
action="ab_test", topic="LEET 교재 출시", pairs=3
```
- 주제에 대해 서로 다른 스타일의 A/B 테스트용 제목 쌍을 생성합니다
- 4가지 스타일 비교: 긴급형, 호기심형, 혜택형, 사회적 증거형
- 실행 가이드도 함께 제공 (배분 방법, 최소 표본 수, 판단 기준)

**스타일 유형:**
- **긴급형**: 시간 제한 강조 ("마지막 기회!", "단 24시간 남았습니다:")
- **호기심형**: 궁금증 유발 ("아무도 모르는", "숨겨진 비밀")
- **혜택형**: 실질적 가치 제시 ("완벽 가이드", "5가지 팁")
- **사회적 증거형**: 다른 사람의 선택 ("1,000명이 선택한", "합격생들의")

**예시:**
- `action="ab_test", topic="LEET 모의고사"` → 3쌍의 A/B 테스트 제목 + 실행 가이드

## 이 도구를 쓰는 에이전트들

### 1. CMO 마케팅처장 (cmo_manager)
**언제 쓰나?** 마케팅 이메일 캠페인의 제목 전략을 수립할 때
**어떻게 쓰나?**
- analyze로 기존 이메일 제목 효과 검증
- ab_test로 A/B 테스트 전략 수립
- 콘텐츠 Specialist에게 제목 가이드라인 전달

**실전 시나리오:**
> CEO가 "이번 달 뉴스레터 제목 좀 잘 뽑아줘" 라고 하면:
> 1. `action="ab_test", topic="2월 LEET 준비 가이드"` 로 A/B 테스트 쌍 생성
> 2. 가장 높은 점수의 제목을 CEO에게 추천
> 3. 발송 후 개봉률(open rate) 비교 결과 보고

### 2. 콘텐츠 Specialist (content_specialist)
**언제 쓰나?** 이메일 콘텐츠 작성 시 제목을 최적화할 때
**어떻게 쓰나?**
- analyze로 작성한 제목의 점수 확인
- suggest로 더 나은 대안 제목 생성
- 점수가 B등급(70점) 이상이 될 때까지 반복 개선

**실전 시나리오:**
> CMO가 "뉴스레터 제목 만들어" 라고 하면:
> 1. 초안 제목 작성 후 `action="analyze", subject="초안 제목"` 실행
> 2. 점수가 낮으면 `action="suggest"` 로 개선안 받기
> 3. 최종 제목 CMO에게 보고

## 주의사항
- 스팸 단어("무료", "대박" 등)는 혜택 키워드이면서 동시에 스팸 감점 대상이므로, 혜택 15점 + 스팸 -15점 = 상쇄될 수 있음
- 점수는 규칙 기반이므로 실제 개봉률(사람들이 메일을 열어보는 비율)과 100% 일치하지는 않음 — 참고 지표로 활용
- 한국어 이메일 기준으로 최적화되어 있음 (영문 이메일은 정확도가 낮을 수 있음)
- A/B 테스트는 최소 1,000명 이상에게 발송해야 통계적으로 의미있는 결과를 얻을 수 있음

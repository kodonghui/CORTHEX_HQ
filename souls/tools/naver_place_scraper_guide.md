# naver_place_scraper — 네이버 플레이스 리뷰 수집기 도구 가이드

## 이 도구는 뭔가요?
네이버 지도(네이버 플레이스)에서 매장, 학원, 식당 등의 리뷰를 수집하고 분석하는 도구입니다.
"이 학원 리뷰가 어떤가?", "고객들이 뭘 좋아하고 뭘 싫어하나?" 같은 질문에 답해줍니다.
리뷰를 모아서 AI가 만족 요인, 불만 요인, 개선 우선순위까지 자동으로 분석해줍니다.

## 어떤 API를 쓰나요?
- **네이버 플레이스 비공식 API** (네이버 지도 내부 API를 활용)
- 비용: **무료**
- 필요한 키: 없음

## 사용법

### action=search (장소 검색)
```
action=search, query="검색어", count=10
```
- `query` (필수): 찾을 장소 이름이나 키워드 (예: "LEET 학원 서울", "강남 법학원")
- `count` (선택, 기본 10, 최대 30): 검색 결과 개수

반환 정보: 장소 이름, ID, 카테고리, 주소, 전화번호, 리뷰 수, 평점

**예시:**
- `action=search, query="LEET 학원 서울"` → 서울 지역 LEET 학원 목록
- `action=search, query="강남 로스쿨 학원"` → 강남 로스쿨 학원 검색

### action=reviews (리뷰 수집)
```
action=reviews, place_id="장소ID", count=100
```
- `place_id` (필수): 장소 ID (search로 먼저 검색해서 확인)
- `count` (선택, 기본 100, 최대 300): 수집할 리뷰 개수

반환 정보: 리뷰 텍스트, 별점, 날짜, 방문 목적, 키워드, 별점 분포 통계

**예시:**
- `action=reviews, place_id="1234567890"` → 해당 장소 리뷰 100건 수집

### action=analyze (리뷰 분석)
```
action=analyze, place_id="장소ID"
```
- `place_id` (필수): 분석할 장소 ID
- 리뷰를 수집한 뒤 AI가 자동으로 종합 분석 보고서를 작성

분석 내용:
1. 고객 만족 요인 TOP 5
2. 고객 불만 요인 TOP 5
3. 자주 언급되는 긍정/부정 키워드
4. 시간별 별점 추이 (개선/악화 방향)
5. 경쟁 우위 포인트
6. 개선 우선순위 제안

**예시:**
- `action=analyze, place_id="1234567890"` → 종합 리뷰 분석 보고서

## 이 도구를 쓰는 에이전트들

### 1. 커뮤니티 Specialist
**언제 쓰나?** 오프라인 매장/학원의 고객 반응을 파악할 때
**어떻게 쓰나?**
- 경쟁 학원/매장의 리뷰를 수집해서 강점/약점 분석
- daum_cafe(온라인 반응)와 함께 온/오프라인 고객 반응 종합 분석
- leet_survey(LEET 특화 서베이)와 함께 LEET 시장 종합 조사

**실전 시나리오:**
> CEO가 "경쟁 학원들 리뷰 분석해줘" 라고 하면:
> 1. `action=search, query="LEET 학원 서울"` → 학원 목록 검색
> 2. 각 학원의 place_id로 `action=analyze` → 학원별 리뷰 분석
> 3. 학원별 만족/불만 요인 비교표 작성
> 4. "A학원은 강사 평가가 좋지만 시설 불만이 많고, B학원은 시설은 좋지만 커리큘럼 불만이 있습니다" 식의 보고

## 주의사항
- 네이버 플레이스 **비공식 API**를 사용하므로, 네이버가 API 구조를 변경하면 작동이 안 될 수 있음
- 리뷰 수집은 장소 ID가 필요하므로, 반드시 `action=search`로 먼저 검색해서 ID를 확인하세요
- 네이버 API 정책으로 접근이 제한될 수 있음 (대량 요청 시)
- 리뷰 텍스트는 최대 300자까지만 수집됨
- 상위 20건만 화면에 표시되지만, 분석에는 수집한 전체 리뷰가 사용됨

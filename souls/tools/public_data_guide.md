# public_data — 공공데이터포털 통계 도구 가이드

## 이 도구는 뭔가요?
정부와 공공기관이 공개하는 통계 데이터(인구수, 고용률, 물가 등)를 조회하는 도구입니다.
실제로 컨설턴트나 리서처가 시장 규모를 추정할 때
"한국 20-30대 인구가 1200만명이니까 TAM은..."  이런 식으로 공공 통계를 근거로 씁니다.

## 어떤 API를 쓰나요?
- **공공데이터포털 API** (data.go.kr)
- 비용: **무료**
- 필요한 키: `PUBLIC_DATA_API_KEY`

## 사용법

### action=search (데이터셋 검색)
```
action=search, query="검색어"
```
- 공공데이터포털에서 데이터셋을 키워드로 검색
- 어떤 데이터가 있는지 탐색할 때 사용

### action=stats (주요 통계 조회)
```
action=stats, category="카테고리"
```
미리 정의된 주요 통계:
| 카테고리 | 어떤 데이터인가 |
|---|---|
| 인구 | 전체 인구, 연령별 인구, 세대수 |
| 고용 | 취업자 수, 실업률, 고용률 |
| 물가 | 소비자물가지수, 생산자물가지수 |
| 교육 | 학생 수, 학교 수, 진학률 |

### action=custom (커스텀 API 호출)
```
action=custom, url="API주소", params="파라미터"
```
- 공공데이터포털의 다른 API를 직접 호출할 때

## 이 도구를 쓰는 에이전트들

### 1. 시장조사 Specialist
**언제 쓰나?** TAM/SAM/SOM 분석, 시장 규모 추정할 때
**어떻게 쓰나?**
- `action=stats, category="인구"` → 타겟 고객층 인구수 파악
- `action=stats, category="교육"` → LEET 관련 시장이면 대학생/로스쿨 지원자 수
- naver_datalab(트렌드)과 함께 → "시장 크기 + 관심도 변화" 종합

**실전 시나리오:**
> "LEET 해설 시장 규모 추정":
> 1. `public_data action=stats, category="교육"` → 로스쿨 정원 2,000명/연
> 2. `naver_datalab action=trend, keywords="LEET"` → 검색량 증가 추이
> 3. **추정:** "매년 LEET 응시자 약 12,000명 x 해설서 구매율 60% x 평균 단가 3만원 = TAM 약 21.6억원"

### 2. 재무모델링 Specialist
**언제 쓰나?** 매출 예측 모델에 시장 데이터를 넣을 때
**어떻게 쓰나?**
- `action=stats, category="인구"` → 인구 감소/증가 반영
- `action=stats, category="고용"` → 경기 상황 반영
- 이 데이터를 매출 예측의 "가정(assumption)"에 활용

## 주의사항
- 공공데이터는 보통 1~6개월 전 데이터입니다. 최신이 아닐 수 있습니다.
- 데이터 형식이 API마다 다를 수 있습니다 (JSON 또는 XML).
- 일부 API는 서비스 키 활성화에 1~2일 걸릴 수 있습니다.

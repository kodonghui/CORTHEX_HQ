# precedent_analyzer — 판례 트렌드 분석기 도구 가이드

## 이 도구는 뭔가요?
특정 법률 분야의 판례(법원 판결 사례)를 대량으로 수집해서,
"최근 몇 년간 이 분야에서 소송이 얼마나 있었는지, 원고(소송을 건 쪽)가 얼마나 이겼는지,
배상금은 보통 얼마인지" 같은 **트렌드(경향)**를 분석해주는 도구입니다.
법무팀이 "이 사업을 하면 소송 위험이 얼마나 되나?"를 판단할 때 씁니다.

## 어떤 API를 쓰나요?
- **국가법령정보센터 API** (https://www.law.go.kr)
- 비용: **완전 무료** (API 키 불필요 — 공개 API)
- 필요한 키: 없음
- LLM 분석: 도구 내부에서 AI(GPT-5 Mini)를 호출하여 종합 분석을 생성합니다

## 사용법

### action=analyze (판례 트렌드 분석)
```
action=analyze, query="검색 키워드", years=5, count=50
```
- **query** (필수): 검색할 법률 분야 키워드
- **years** (선택, 기본값 5): 최근 몇 년간 분석할지
- **count** (선택, 기본값 50): 수집할 판례 건수 (최대)
- 연도별 판례 건수, 원고/피고 승소율, 배상금 통계, 자주 인용되는 법 조문, 핵심 키워드 빈도를 분석
- AI가 법적 트렌드 요약 + 사업 리스크 평가 + 대응 전략까지 생성

**예시:**
- `action=analyze, query="저작권 침해"` → 최근 5년간 저작권 침해 판례 트렌드 분석
- `action=analyze, query="개인정보 유출", years=3, count=30` → 최근 3년간 개인정보 관련 판례 30건 분석
- `action=analyze, query="프랜차이즈 분쟁", years=10` → 최근 10년간 프랜차이즈 판례 트렌드

### action=summary (특정 판례 요약)
```
action=summary, case_id="판례 번호"
```
- **case_id** (필수): 요약할 판례 번호 (예: "2023다12345")
- 사건 개요, 핵심 쟁점, 판결 요지, 시사점을 AI가 정리

**예시:**
- `action=summary, case_id="2023다12345"` → 해당 판례의 4단계 요약 (개요/쟁점/판결/시사점)

### action=risk (법적 리스크 분석)
```
action=risk, topic="분석 주제"
```
- **topic** (필수): 리스크를 분석할 사업/행위 주제
- 관련 판례 수에 따라 리스크 수준을 자동 판정 (상/중/하)
  - 20건 이상: 상 (높음) — 소송 위험이 높은 분야
  - 5~19건: 중 (보통) — 주의가 필요한 분야
  - 5건 미만: 하 (낮음) — 법적 분쟁이 빈번하지 않은 분야
- AI가 핵심 법적 리스크, 관련 법률, 사전 대비 방안, 최악의 시나리오, CEO 권고사항까지 생성

**예시:**
- `action=risk, topic="AI 생성 콘텐츠 저작권"` → AI 콘텐츠의 법적 리스크 분석
- `action=risk, topic="온라인 교육 플랫폼"` → 온라인 교육 사업의 법적 리스크 분석

## 이 도구를 쓰는 에이전트들

### 1. 법무/IP처장 (CLO)
**언제 쓰나?** 새로운 사업이나 기능을 검토할 때, 법적 리스크를 전반적으로 파악할 때
**어떻게 쓰나?**
- `action=risk`로 빠르게 리스크 수준을 파악한 뒤, 필요하면 Specialist에게 상세 분석 지시
- `action=analyze`로 트렌드를 보고 CEO에게 보고할 데이터 확보

**실전 시나리오:**
> CEO가 "AI 교육 서비스를 시작하려는데 법적으로 문제될 게 있어?" 라고 하면:
> 1. `action=risk, topic="AI 교육 서비스"` → 리스크 수준 빠르게 확인
> 2. `action=analyze, query="AI 교육 저작권"` → 관련 판례 트렌드 분석
> 3. 결과를 종합하여 "리스크 수준: 중. 저작권법 제35조의5(공정이용) 관련 판례가 증가 추세입니다. 사전에 라이선스 정책을 수립하세요" 식으로 보고

### 2. 저작권 Specialist
**언제 쓰나?** 저작권/라이선스 관련 구체적 분석이 필요할 때
**어떻게 쓰나?**
- `action=analyze`로 저작권 분야 판례 트렌드를 파악하고, 구체적인 판례는 `action=summary`로 요약
- law_search 도구와 함께 사용하여 법 조문 + 판례 트렌드를 종합 분석

**실전 시나리오:**
> "경쟁사가 우리 콘텐츠를 무단 사용했을 때 승소 가능성은?"
> 1. `action=analyze, query="콘텐츠 무단 복제 저작권"` → 승소율 통계 확인
> 2. `action=summary, case_id="관련 판례번호"` → 유사 판례 상세 요약
> 3. "원고 승소율 67%, 평균 배상금 2,500만원. 승소 가능성 높으나 소송 비용 대비 실익을 먼저 검토하세요"

### 3. 특허/약관 Specialist
**언제 쓰나?** 특허 분쟁 위험을 분석하거나, 약관 관련 판례 동향을 파악할 때
**어떻게 쓰나?**
- `action=analyze`로 특허 분쟁 판례 트렌드 확인
- `action=risk`로 특정 기술 분야의 특허 분쟁 리스크 판단
- kipris 도구(특허 검색)와 함께 사용하여 선행기술 + 판례 동향을 종합

**실전 시나리오:**
> "우리 기술로 특허 출원하면 분쟁 날 가능성은?"
> 1. kipris 도구로 유사 특허 존재 여부 확인
> 2. `action=risk, topic="AI 추천 알고리즘 특허"` → 특허 분쟁 리스크 분석
> 3. `action=analyze, query="특허 무효 AI"` → 관련 특허 분쟁 트렌드 확인

## 주의사항
- **AI 법률 분석은 참고용입니다.** 실제 법률 문제는 반드시 변호사와 상담하세요.
- 판례 검색 결과가 현재 유효한지 확인이 필요합니다 (법이 개정되면 과거 판례의 의미가 달라질 수 있음).
- 법제처 API에서 제공하는 판례 데이터가 완전하지 않을 수 있으므로, 통계(승소율, 배상금)는 참고 수준으로 활용하세요.
- `count` 값을 너무 크게 설정하면 API 호출이 여러 번 발생하여 응답이 느려질 수 있습니다.
- 모든 분석 결과 끝에 면책 안내("이 분석은 참고용이며, 실제 법률 문제는 변호사와 상담하세요")가 자동 포함됩니다.

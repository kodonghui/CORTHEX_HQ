# web_search — 웹검색 도구 가이드

## 이 도구는 뭔가요?
AI(인공지능)의 학습된 지식을 바탕으로 웹 검색처럼 질문에 답변하는 도구입니다.
실제로 인터넷에 접속해서 검색하는 것은 아니고, AI가 알고 있는 정보로 최대한 정확하게 답변합니다.
실시간 최신 정보가 필요하면 `real_web_search` 도구를 대신 사용하세요.

## 어떤 API를 쓰나요?
- **LLM(대규모 언어 모델) 지식 기반 답변** — 외부 API 호출 없음
- 비용: **무료** (AI 모델 호출 비용만 발생)
- 필요한 키: 없음

## 사용법

### 기본 사용 (질문 입력)
```
query="검색할 내용"
```
- `query` 파라미터에 궁금한 내용을 넣으면 AI가 답변합니다
- 출처가 불확실한 정보는 '확인 필요'로 표시됩니다
- 수치 데이터는 대략적인 범위로 제시됩니다

**예시:**
- `query="LEET 시험 최근 동향"` → LEET 시험 관련 정보 답변
- `query="교육 스타트업 시장 규모"` → 교육 시장 규모 추정 답변

## 이 도구를 쓰는 에이전트들

### 1. 비서실장 (Chief of Staff)
**언제 쓰나?** CEO의 일반적인 질문에 빠르게 답변할 때
**어떻게 쓰나?**
- 간단한 배경 정보 조사, 빠른 팩트 확인에 사용
- 더 정확한 최신 정보가 필요하면 real_web_search로 전환

### 2. 정보 보좌관 (Information Aide)
**언제 쓰나?** 보고서 작성 시 배경 정보 수집
**어떻게 쓰나?**
- 보고서에 들어갈 기초 자료 수집
- 뉴스/트렌드 정보 확인

### 3. 검수 보좌관 (Relay Specialist)
**언제 쓰나?** 보고서 내용의 팩트체크(사실 확인) 시
**어떻게 쓰나?**
- 다른 에이전트가 작성한 보고서의 정보가 맞는지 교차 검증

### 4. 시장조사 Specialist
**언제 쓰나?** 시장 규모, 경쟁사 정보 등 기초 조사
**어떻게 쓰나?**
- 시장 동향, 업계 현황 파악에 활용
- naver_datalab, platform_market_scraper 등과 함께 사용

### 5. 법무·IP처장 (CLO)
**언제 쓰나?** 법률/규제 관련 일반적인 정보 확인
**어떻게 쓰나?**
- 법률 개정 동향, 규제 현황 등 기초 정보 수집

### 6. 저작권 Specialist
**언제 쓰나?** 저작권 관련 일반 정보 확인
**어떻게 쓰나?**
- 저작권법, 라이선스 기본 정보 조사

### 7. 설문/리서치 Specialist
**언제 쓰나?** 설문 설계 시 배경 조사, 업계 벤치마크 확인
**어떻게 쓰나?**
- 설문 주제 관련 기초 자료 수집

### 8. 회사연대기 Specialist
**언제 쓰나?** 연대기 작성 시 시대 배경 정보 확인
**어떻게 쓰나?**
- 특정 시기의 업계 동향, 시장 상황 조사

**실전 시나리오:**
> CEO가 "LEET 시장 현황 알아봐" 라고 하면:
> 1. web_search로 LEET 시장 기초 정보 수집
> 2. 더 정확한 최신 정보가 필요하면 real_web_search로 추가 검색
> 3. naver_datalab으로 검색량 추이 확인
> 4. 종합 보고서 작성

## 주의사항
- 실시간 정보가 아닌 AI의 학습 데이터 기반이므로, **최신 정보는 정확하지 않을 수 있음**
- 실시간/최신 정보가 필요하면 반드시 `real_web_search` 사용
- 수치 데이터는 대략적인 범위이므로 정확한 수치가 필요하면 전문 도구(dart_api, ecos_macro 등) 사용
- web_search와 real_web_search의 차이: web_search는 AI 지식 기반(빠르지만 최신 아님), real_web_search는 실제 구글 검색(느리지만 최신)

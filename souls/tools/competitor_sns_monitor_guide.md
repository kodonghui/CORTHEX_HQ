# competitor_sns_monitor — 경쟁사 SNS 모니터 도구 가이드

## 이 도구는 뭔가요?
경쟁사의 SNS 활동(네이버블로그, 인스타그램, 유튜브)을 자동으로 추적하고 분석해주는 도구입니다.
감시 대상을 등록해두면, 블로그 글 목록, 인스타그램 팔로워/게시물 수, 유튜브 구독자 수 등을 수집하여 경쟁사의 SNS 전략을 분석합니다.

## 어떤 API를 쓰나요?
- **네이버 블로그 RSS** (블로그 최근 게시물 자동 수집)
- **인스타그램 웹 스크래핑** (팔로워 수, 게시물 수 조회)
- **유튜브 웹 스크래핑** (채널명, 구독자 수 조회)
- **AI 분석** (경쟁사 전략 종합 분석 보고서 생성)
- 비용: **무료**
- 필요한 키: 없음

## 사용법

### action=add (감시 대상 추가)
```
action="add", name="메가로스쿨", blog_url="https://blog.naver.com/mega_leet", instagram="mega_leet", youtube="https://www.youtube.com/@mega_leet"
```
- 경쟁사를 감시 목록에 추가합니다
- blog_url, instagram, youtube 중 최소 하나는 입력해야 합니다
- 같은 이름이 이미 등록되어 있으면 중복 오류 발생

**예시:**
- `action="add", name="법률저널", blog_url="https://blog.naver.com/lawjournal"` → 감시 대상 추가 완료

### action=remove (감시 대상 제거)
```
action="remove", name="메가로스쿨"
```
- 감시 목록에서 해당 경쟁사를 제거합니다

### action=list (감시 목록 조회)
```
action="list"
```
- 현재 감시 중인 경쟁사 목록을 표로 보여줍니다 (이름, 블로그, 인스타, 유튜브, 마지막 확인일)

### action=check (SNS 활동 확인)
```
action="check"
```
또는 특정 경쟁사만:
```
action="check", name="메가로스쿨"
```
- 등록된 경쟁사의 최근 SNS 활동을 수집합니다
- **블로그**: RSS로 최근 10개 게시물 제목/링크/날짜 수집
- **인스타그램**: 팔로워 수, 게시물 수 조회
- **유튜브**: 채널명, 구독자 수 조회
- name을 지정하지 않으면 전체 감시 목록 대상으로 확인

**예시:**
- `action="check"` → 전체 경쟁사 블로그 최근 글, 인스타 팔로워, 유튜브 구독자 현황

### action=report (전략 종합 보고서)
```
action="report", name="all"
```
또는 특정 경쟁사:
```
action="report", name="메가로스쿨"
```
- 수집된 데이터를 바탕으로 AI가 경쟁사 SNS 전략 종합 보고서를 생성합니다
- 포함 내용: 각 경쟁사 전략 요약, 게시 빈도/콘텐츠 유형 분석, 우리의 강점/약점, 대응 전략 3가지, 벤치마킹 포인트

**예시:**
- `action="report"` → 전체 경쟁사 대비 우리 SNS 전략 비교 분석 보고서

## 이 도구를 쓰는 에이전트들

### 1. CMO 마케팅처장 (cmo_manager)
**언제 쓰나?** 경쟁사 SNS 동향을 파악하고 우리 전략을 조정할 때
**어떻게 쓰나?**
- add로 주요 경쟁사 등록 (최초 1회)
- check로 주기적 활동 모니터링
- report로 전략 보고서 생성 → CEO 보고

**실전 시나리오:**
> CEO가 "경쟁사들이 요즘 SNS에서 뭘 하고 있는지 파악해줘" 라고 하면:
> 1. `action="check"` 로 전체 감시 목록 확인
> 2. `action="report"` 로 종합 보고서 생성
> 3. CEO에게 "경쟁사 A는 주 3회 블로그 포스팅, 우리는 주 1회. 블로그 빈도를 높이는 것을 추천합니다" 보고

## 주의사항
- 감시 목록 데이터는 `data/competitor_sns_watchlist.json` 파일에 저장됩니다 (배포 시 날아갈 수 있음 — DB 마이그레이션 고려 필요)
- 인스타그램/유튜브 스크래핑은 차단 방지를 위해 요청 간 1.5~2초 딜레이가 있어 다수 경쟁사 확인 시 시간이 걸림
- 인스타그램이 비공개 계정이거나 접근을 차단하면 데이터 수집이 실패할 수 있음
- 네이버 블로그 RSS가 비활성화된 블로그는 게시물 수집 불가
- check/report 실행 시 마지막 확인 시간이 자동 갱신됩니다

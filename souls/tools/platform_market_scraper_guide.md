# platform_market_scraper — 플랫폼 시장 조사기 도구 가이드

## 이 도구는 뭔가요?
크몽, 탈잉, 클래스101 같은 온라인 교육/서비스 플랫폼에서 상품 정보(가격, 리뷰, 별점, 수강생 수)를 자동으로 수집하고 분석하는 도구입니다.
"우리 분야에서 다른 사람들은 얼마에 팔고 있나?", "경쟁이 얼마나 치열한가?" 같은 시장 조사 질문에 답해줍니다.

## 어떤 API를 쓰나요?
- **크몽/탈잉/클래스101 웹 크롤링** (웹페이지 자동 수집) — 각 플랫폼 검색 페이지를 직접 파싱(분석)
- 비용: **무료**
- 필요한 키: 없음
- 의존 라이브러리: `httpx`, `beautifulsoup4`

## 사용법

### action=search (플랫폼 서비스 검색)
```
action=search, query="검색어", platform="all", count=20
```
- `query` (필수): 검색할 키워드 (예: "LEET", "법학", "코딩 교육")
- `platform` (선택, 기본 "all"): 검색할 플랫폼
  - `all` — 크몽+탈잉+클래스101 전부
  - `kmong` — 크몽만
  - `taling` — 탈잉만
  - `class101` — 클래스101만
- `count` (선택, 기본 20): 각 플랫폼에서 가져올 상품 개수

반환 정보: 상품명, 가격, 별점, 리뷰/수강생 수

**예시:**
- `action=search, query="LEET"` → 3개 플랫폼에서 LEET 관련 서비스 검색
- `action=search, query="법학 강의", platform="class101"` → 클래스101에서만 검색

### action=analyze (시장 분석)
```
action=analyze, query="검색어"
```
- `query` (필수): 분석할 키워드
- 먼저 3개 플랫폼에서 데이터를 수집한 뒤, AI가 시장 분석 보고서를 작성

분석 내용:
1. 시장 가격 포지셔닝 (가격대별 분포, 평균/최빈 가격대)
2. 경쟁 강도 (경쟁자 수, 포화도 판단)
3. 차별화 기회 (빈틈 있는 영역)
4. 진입 전략 제안 (적정 가격대, 타겟 고객, 콘텐츠 전략)

**예시:**
- `action=analyze, query="LEET 강의"` → LEET 강의 시장 분석 보고서

### action=price_range (가격대 분포 분석)
```
action=price_range, query="검색어"
```
- `query` (필수): 분석할 키워드
- 3개 플랫폼에서 수집한 가격을 구간별로 분류하고, AI가 가격 전략을 분석

가격 구간: 5만원 이하 / 5~10만원 / 10~30만원 / 30만원 이상
통계: 평균/최저/최고 가격

**예시:**
- `action=price_range, query="코딩 교육"` → 코딩 교육 가격대 분포 + 적정 가격 제안

## 이 도구를 쓰는 에이전트들

### 1. 사업기획처장 (CSO)
**언제 쓰나?** 신규 사업 기획 시 시장 규모와 경쟁 현황 파악
**어떻게 쓰나?**
- analyze로 시장 전체 분석
- price_range로 가격 전략 수립
- naver_datalab(검색량 추이)와 함께 시장 매력도 종합 판단

### 2. 시장조사 Specialist
**언제 쓰나?** 경쟁사 서비스 가격/리뷰 조사
**어떻게 쓰나?**
- search로 경쟁 서비스 목록 수집
- app_review_scraper, youtube_analyzer와 함께 경쟁 환경 종합 분석

### 3. 마케팅·고객처장 (CMO)
**언제 쓰나?** 마케팅 전략 수립 시 경쟁 제품 벤치마킹
**어떻게 쓰나?**
- 경쟁사의 가격, 리뷰, 콘텐츠 전략 파악
- youtube_analyzer와 함께 콘텐츠 마케팅 전략 수립

### 4. 설문/리서치 Specialist
**언제 쓰나?** 사용자 리서치 시 시장 가격 벤치마크 확인
**어떻게 쓰나?**
- 설문 설계 전 기존 시장 가격/서비스 수준 파악

### 5. 커뮤니티 Specialist
**언제 쓰나?** 커뮤니티 내 추천 서비스/제품 트렌드 파악
**어떻게 쓰나?**
- 커뮤니티에서 언급된 서비스의 실제 가격/평점 교차 검증

**실전 시나리오:**
> CEO가 "LEET 강의 시장에서 우리가 들어갈 자리가 있나?" 라고 하면:
> 1. `action=analyze, query="LEET 강의"` → 3개 플랫폼 시장 분석
> 2. `action=price_range, query="LEET 강의"` → 가격대별 분포 확인
> 3. 경쟁이 약한 가격대, 차별화 가능한 영역 파악
> 4. "5~10만원대 프리미엄 해설 서비스가 부족합니다. 여기에 진입 가능합니다" 같은 구체적 제안

## 주의사항
- 웹 크롤링(자동 수집) 방식이므로 플랫폼 UI 변경 시 수집이 안 될 수 있음
- 각 플랫폼 요청 사이에 1.5초 딜레이가 있어서 전체 수집에 시간이 좀 걸림
- 가격이 표시되지 않는 서비스는 "가격 미정"으로 나옴
- `beautifulsoup4` 라이브러리가 설치되어 있어야 함
- 수집되는 데이터는 검색 결과 페이지에서 파싱 가능한 정보로 한정됨 (상세 페이지는 미포함)

# law_change_monitor — 법령 변경 알리미 도구 가이드

## 이 도구는 뭔가요?
우리 사업과 관련된 법률이 바뀌었는지(개정/신설/폐지) 자동으로 감시하는 도구입니다.
관심 있는 법령을 "감시 목록"에 등록하면, 나중에 한 번에 변경사항을 확인할 수 있습니다.
법이 바뀌면 "우리 사업에 어떤 영향이 있는지"까지 AI가 분석해줍니다.
마치 "법률 알림 서비스"를 직접 운영하는 것과 같습니다.

## 어떤 API를 쓰나요?
- **국가법령정보센터 API** (https://www.law.go.kr)
- 비용: **완전 무료** (API 키 불필요 — 공개 API)
- 필요한 키: 없음
- LLM 분석: 도구 내부에서 AI(GPT-5 Mini)를 호출하여 변경 영향 분석을 생성합니다
- 감시 목록 저장: `data/law_watchlist.json` 파일에 JSON으로 저장

## 사용법

### action=watch (감시 법령 등록)
```
action=watch, law_name="법령명"
```
- **law_name** (필수): 감시할 법령 이름
- 법제처 API에서 해당 법령의 현재 정보(시행일, 법령 구분, 법령ID)를 조회하여 감시 목록에 저장
- 이미 등록된 법령은 중복 등록 안 됨

**예시:**
- `action=watch, law_name="저작권법"` → 저작권법 감시 등록
- `action=watch, law_name="개인정보보호법"` → 개인정보보호법 감시 등록
- `action=watch, law_name="전자상거래법"` → 전자상거래법 감시 등록

### action=unwatch (감시 해제)
```
action=unwatch, law_name="법령명"
```
- **law_name** (필수): 감시를 해제할 법령 이름
- 감시 목록에서 제거

**예시:**
- `action=unwatch, law_name="저작권법"` → 저작권법 감시 해제

### action=check (변경사항 확인)
```
action=check
```
- 파라미터 없음 — 등록된 모든 감시 법령의 변경사항을 한 번에 확인
- 각 법령의 현재 시행일을 법제처 API에서 조회하여, 등록 당시 시행일과 비교
- 시행일이 바뀌었으면 "변경 감지"로 표시
- 변경이 감지되면 AI가 개정 내용 추정, 사업 영향 분석, 필요 조치, 시급도(즉시/1개월내/3개월내)를 분석
- 확인 후 감시 목록의 시행일 정보를 자동 업데이트

**예시:**
- `action=check` → "저작권법: 변경 감지(시행일 2025-12-01 → 2026-03-01), 개인정보보호법: 변경 없음"

### action=list (감시 목록 조회)
```
action=list
```
- 파라미터 없음 — 현재 감시 중인 법령 목록을 표로 표시
- 법령명, 마지막 시행일, 등록일 정보 포함

**예시:**
- `action=list` → 감시 중인 법령 3건 목록 표시

### action=recent (최근 개정 법령 조회)
```
action=recent, days=30, category="분야"
```
- **days** (선택, 기본값 30): 최근 며칠간의 개정 법령을 조회할지
- **category** (선택, 기본값 "전체"): 분야별 필터

**지원하는 카테고리:**

| 카테고리 | 검색 키워드 |
|---------|-----------|
| **교육** | 교육, 학교, 대학, 학원, 평생교육, 직업훈련 |
| **정보통신** | 정보통신, 전자상거래, 개인정보, 저작권, 인터넷, 소프트웨어, AI, 데이터 |
| **금융** | 금융, 은행, 보험, 증권, 투자, 자본시장 |
| **전체** (기본) | 교육, 정보통신, 저작권, 개인정보, 전자상거래 |

- 법제처 API에서 키워드별 검색 후 중복 제거, 시행일 기준 최신순 정렬
- AI가 법령 변경 트렌드, 교육/IT 사업 관련 영향, 주목할 법령을 분석

**예시:**
- `action=recent` → 최근 30일간 주요 분야 개정 법령 목록
- `action=recent, days=60, category="정보통신"` → 최근 60일간 IT 분야 개정 법령
- `action=recent, category="금융"` → 최근 30일간 금융 분야 개정 법령

## 이 도구를 쓰는 에이전트들

### 1. 법무/IP처장 (CLO)
**언제 쓰나?** 정기적인 법률 변경 모니터링, 새 사업 분야의 법률 환경 파악
**어떻게 쓰나?**
- `action=watch`로 핵심 법령 등록 → 정기적으로 `action=check`로 변경 확인
- `action=recent`로 최근 법령 개정 트렌드를 파악하여 CEO에게 선제적으로 보고
- 변경이 감지되면 Specialist에게 상세 분석 지시

**실전 시나리오:**
> CEO가 "우리 사업에 영향 줄 만한 법 변경이 있어?" 라고 하면:
> 1. `action=check` → 감시 중인 법령 변경사항 확인
> 2. `action=recent, category="정보통신"` → 최근 IT 분야 법령 개정 확인
> 3. "개인정보보호법 시행일이 변경되었습니다. 3개월 내 우리 개인정보 처리방침 업데이트가 필요합니다. 나머지 감시 법령은 변경 없습니다"

### 2. 저작권 Specialist
**언제 쓰나?** 저작권/콘텐츠 관련 법령의 변경을 추적할 때
**어떻게 쓰나?**
- `action=watch, law_name="저작권법"` 등 저작권 관련 법령 등록
- `action=check`로 저작권법 개정 여부 확인
- 개정이 감지되면 law_search 도구로 구체적인 개정 조문 확인

**실전 시나리오:**
> "저작권법이 최근에 바뀌었는지 확인해줘"
> 1. `action=check` → 저작권법 변경 여부 확인
> 2. 변경 감지 시: law_search 도구로 개정 조문 상세 확인
> 3. "저작권법 제35조의5(공정이용)가 개정되었습니다. AI 학습 데이터 관련 조항이 신설되어 우리 AI 서비스에 영향이 있습니다"

### 3. 특허/약관 Specialist
**언제 쓰나?** 특허법, 전자상거래법 등 약관/IP 관련 법령 변경을 추적할 때
**어떻게 쓰나?**
- `action=watch`로 특허법, 전자상거래법, 약관규제법 등록
- `action=recent, category="정보통신"`으로 IT 관련 법령 트렌드 파악
- 법 개정 감지 시 contract_reviewer 도구로 기존 약관이 새 법에 맞는지 재검토

**실전 시나리오:**
> "약관규제법이 바뀌면 우리 이용약관도 바꿔야 할 수 있으니 감시해줘"
> 1. `action=watch, law_name="약관의 규제에 관한 법률"` → 약관규제법 감시 등록
> 2. 다음 번 `action=check` 시 변경 자동 감지
> 3. 변경 감지되면: contract_reviewer 도구로 기존 약관 재검토 + 수정 권고

## 주의사항
- **이 분석은 참고용입니다.** 실제 법률 문제는 반드시 변호사와 상담하세요.
- 감시 목록은 `data/law_watchlist.json` 파일에 저장됩니다. 서버 배포 시 `git reset --hard`로 이 파일이 사라질 수 있으므로, 중요한 감시 목록은 DB(save_setting)로 이전하는 것이 안전합니다.
- `action=check`는 시행일 변경만 감지합니다. 같은 시행일 내에서 세부 조문이 바뀐 경우는 감지하지 못할 수 있습니다.
- 법제처 API는 무료이지만 응답 속도가 느릴 수 있습니다. 감시 목록이 많으면 check에 시간이 걸릴 수 있습니다.
- `action=recent`의 카테고리는 키워드 기반 검색이므로, 특정 법령이 누락될 수 있습니다. 중요한 법령은 `action=watch`로 직접 등록하는 것이 확실합니다.

# 런칭 전략

당신은 SaaS 제품 런칭과 기능 발표 전문가입니다. 모멘텀을 구축하고, 주목을 끌며, 관심을 사용자로 전환하는 런칭을 계획하는 것이 목표입니다.

## 시작하기 전에

**먼저 제품 마케팅 컨텍스트를 확인하세요:**
`.claude/product-marketing-context.md` 파일이 있으면, 질문하기 전에 먼저 읽으세요. 해당 컨텍스트를 활용하고, 아직 다루지 않은 정보나 이 작업에 특화된 정보만 질문하세요.

---

## 핵심 철학

최고의 회사들은 한 번만 런칭하지 않습니다—반복해서 런칭합니다. 모든 새로운 기능, 개선, 업데이트는 주목을 끌고 오디언스와 소통할 기회입니다.

강력한 런칭은 단일 순간이 아닙니다. 다음에 관한 것입니다:
- 제품을 일찍 사용자 손에 넣기
- 실제 피드백에서 배우기
- 매 단계에서 임팩트 만들기
- 시간이 지남에 따라 복리로 늘어나는 모멘텀 구축

## ORB 프레임워크

런칭 마케팅을 세 가지 채널 유형으로 구조화하세요. 모든 것이 궁극적으로 자사 채널로 이어져야 합니다.

### 자사 채널 (Owned)
채널을 소유합니다 (오디언스는 아니지만). 알고리즘이나 플랫폼 규칙 없이 직접 접근합니다.

**예시:**
- 이메일 리스트
- 블로그
- 팟캐스트
- 브랜드 커뮤니티 (Slack, Discord)
- 웹사이트/제품

**중요한 이유:**
- 시간이 지날수록 더 효과적
- 알고리즘 변경이나 유료화 없음
- 오디언스와의 직접적 관계
- 콘텐츠의 복리 가치

**오디언스에 따라 1-2개로 시작:**
- 업계에 양질의 콘텐츠가 부족 → 블로그 시작
- 사람들이 직접 업데이트를 원함 → 이메일에 집중
- 참여가 중요 → 커뮤니티 구축

**예시 - Superhuman:**
초대 전용 대기 명단과 1:1 온보딩 세션으로 수요를 구축했습니다. 모든 신규 사용자가 30분 라이브 데모를 받았습니다. 이것이 독점성, FOMO, 입소문을 만들었습니다—모두 자사 관계를 통해서. 수년 후에도 원래 온보딩 자료가 참여를 이끌고 있습니다.

### 임대 채널 (Rented)
가시성을 제공하지만 통제할 수 없는 플랫폼. 알고리즘이 바뀌고, 규칙이 변하며, 유료화가 증가합니다.

**예시:**
- 소셜 미디어 (Twitter/X, LinkedIn, Instagram)
- 앱 스토어와 마켓플레이스
- YouTube
- Reddit

**올바른 사용법:**
- 오디언스가 활동하는 1-2개 플랫폼 선택
- 자사 채널로 트래픽을 유도하는 데 활용
- 유일한 전략으로 의존하지 말 것

**예시 - Notion:**
생산성 열성팬이 활동하는 Twitter, YouTube, Reddit에서 바이럴리티를 해킹했습니다. 커뮤니티가 템플릿과 워크플로우를 공유하도록 장려했습니다. 하지만 모든 가시성을 자사 자산으로 연결했습니다—모든 바이럴 포스트가 가입으로, 그다음 타겟 이메일 온보딩으로 이어졌습니다.

**플랫폼별 전술:**
- Twitter/X: 대화를 촉발하는 스레드 → 뉴스레터 링크
- LinkedIn: 고가치 포스트 → 게이트 콘텐츠 또는 이메일 가입으로 연결
- 마켓플레이스 (Shopify, Slack): 리스팅 최적화 → 더 많은 정보를 위해 사이트로 유도

임대 채널은 속도를 주지만 안정성은 아닙니다. 사용자를 자사 생태계로 데려와 모멘텀을 포착하세요.

### 차입 채널 (Borrowed)
다른 사람의 오디언스를 활용하여 가장 어려운 부분—주목받기—을 단축합니다.

**예시:**
- 게스트 콘텐츠 (블로그 포스트, 팟캐스트 인터뷰, 뉴스레터 피처)
- 협업 (웨비나, 공동 마케팅, 소셜 테이크오버)
- 강연 (컨퍼런스, 패널, 버추얼 서밋)
- 인플루언서 파트너십

**수동적이 아닌 능동적으로:**
1. 오디언스가 팔로우하는 업계 리더 목록 작성
2. 윈-윈 협업 제안
3. SparkToro나 Listen Notes 같은 도구로 오디언스 겹침 찾기
4. 제휴/추천 인센티브 설정

**예시 - TRMNL:**
유료 스폰서십이 아닌 무료로 e-ink 디스플레이를 유튜버 Snazzy Labs에게 보냈습니다—그가 좋아해주길 바라면서. 그가 50만+ 조회수를 기록한 심층 리뷰를 만들어 $500K+ 매출을 이끌었습니다. 또한 지속적 프로모션을 위한 제휴 프로그램도 설정했습니다.

차입 채널은 즉각적인 신뢰성을 제공하지만, 차입한 관심을 자사 관계로 전환해야만 효과가 있습니다.

## 5단계 런칭 접근법

런칭은 하루짜리 이벤트가 아닙니다. 모멘텀을 구축하는 단계적 프로세스입니다.

### 1단계: 내부 런칭
공개 전에 초기 피드백을 수집하고 주요 문제를 해결합니다.

**액션:**
- 무료 테스트를 위해 얼리 사용자를 1:1로 모집
- 사용성 갭과 누락 기능에 대한 피드백 수집
- 프로토타입이 데모 가능할 정도로 기능적인지 확인 (프로덕션 수준일 필요 없음)

**목표:** 친화적 사용자와 핵심 기능 검증.

### 2단계: 알파 런칭
통제된 방식으로 외부 사용자 앞에 제품을 놓습니다.

**액션:**
- 얼리 액세스 가입 폼이 있는 랜딩 페이지 생성
- 제품의 존재를 알림
- 테스트 시작을 위해 개별적으로 사용자 초대
- MVP가 프로덕션에서 작동해야 함 (아직 진화 중이더라도)

**목표:** 첫 외부 검증과 초기 대기 명단 구축.

### 3단계: 베타 런칭
외부 버즈를 생성하면서 얼리 액세스를 확대합니다.

**액션:**
- 얼리 액세스 리스트 처리 (일부 무료, 일부 유료)
- 해결하는 문제에 대한 티저로 마케팅 시작
- 친구, 투자자, 인플루언서에게 테스트와 공유 요청

**추가 고려:**
- 커밍 순 랜딩 페이지 또는 대기 명단
- 대시보드 내비게이션에 "Beta" 스티커
- 얼리 액세스 리스트에 이메일 초대
- 실험적 기능을 위한 설정 내 얼리 액세스 토글

**목표:** 버즈 구축과 더 넓은 피드백으로 제품 개선.

### 4단계: 얼리 액세스 런칭
소규모 테스트에서 통제된 확장으로 전환합니다.

**액션:**
- 제품 정보 유출: 스크린샷, 기능 GIF, 데모
- 양적 사용 데이터와 질적 피드백 수집
- 참여 사용자와 사용자 리서치 진행 (크레딧으로 인센티브)
- 선택적으로 제품/시장 적합성 설문으로 메시징 개선

**확장 옵션:**
- 옵션 A: 배치별 초대 제한 (5-10%씩)
- 옵션 B: "얼리 액세스" 프레이밍으로 모든 사용자 한 번에 초대

**목표:** 규모에서 검증하고 풀 런칭 준비.

### 5단계: 풀 런칭
본격적으로 공개합니다.

**액션:**
- 셀프서브 가입 오픈
- 과금 시작 (아직 안 했다면)
- 모든 채널에서 정식 출시 발표

**런칭 터치포인트:**
- 고객 이메일
- 인앱 팝업과 제품 투어
- 런칭 자료로 연결되는 웹사이트 배너
- 대시보드 내비게이션에 "New" 스티커
- 블로그 포스트 발표
- 플랫폼 전체 소셜 포스트
- Product Hunt, BetaList, Hacker News 등

**목표:** 최대 가시성과 유료 사용자로의 전환.

## Product Hunt 런칭 전략

Product Hunt는 얼리 어답터에게 도달하는 강력한 수단이 될 수 있지만, 마법은 아닙니다—준비가 필요합니다.

### 장점
- 기술에 능숙한 얼리 어답터 오디언스에 노출
- 신뢰성 향상 (특히 오늘의 제품에 선정되면)
- 잠재적 PR 보도와 백링크

### 단점
- 좋은 순위를 얻기 매우 경쟁적
- 단기적 트래픽 급증
- 상당한 사전 준비 필요

### 성공적 런칭 방법

**런칭 당일 전:**
1. 영향력 있는 지지자, 콘텐츠 허브, 커뮤니티와 관계 구축
2. 리스팅 최적화: 매력적인 태그라인, 세련된 비주얼, 짧은 데모 영상
3. 성공적 런칭 사례를 연구하여 효과적이었던 것 파악
4. 관련 커뮤니티에 참여—피칭 전에 가치 제공
5. 하루 종일 참여를 위한 팀 준비

**런칭 당일:**
1. 하루 종일 이벤트로 취급
2. 모든 댓글에 실시간으로 응답
3. 질문에 답하고 토론 촉발
4. 기존 오디언스의 참여 장려
5. 가입 확보를 위해 트래픽을 사이트로 유도

**런칭 당일 후:**
1. 참여한 모든 사람에게 후속 조치
2. Product Hunt 트래픽을 자사 관계로 전환 (이메일 가입)
3. 런칭 후 콘텐츠로 모멘텀 유지

### 사례 연구

**SavvyCal** (스케줄링 도구):
- 런칭 전 랜딩 페이지와 온보딩 최적화
- 생산성/SaaS 인플루언서와 사전 관계 구축
- 런칭 당일 모든 댓글에 응답
- 결과: 월간 #2 제품

**Reform** (폼 빌더):
- 성공적 런칭 사례를 연구하고 인사이트 적용
- 명확한 태그라인, 세련된 비주얼, 데모 영상 제작
- 런칭 전 커뮤니티 참여 (먼저 가치 제공)
- 런칭을 하루 종일 참여 이벤트로 취급
- 가입 확보를 위해 트래픽 유도
- 결과: 오늘의 #1 제품

## 런칭 후 제품 마케팅

런칭은 발표가 나간다고 끝나지 않습니다. 이제 채택과 리텐션 작업이 시작됩니다.

### 즉각적 런칭 후 액션

**신규 사용자 교육:**
핵심 기능과 사용 사례를 소개하는 자동 온보딩 이메일 시퀀스를 설정합니다.

**런칭 강화:**
놓친 사람들을 위해 주간/격주/월간 라운드업 이메일에 발표를 포함합니다.

**경쟁사 대비 차별화:**
왜 당신이 확실한 선택인지 하이라이트하는 비교 페이지를 게시합니다.

**웹 페이지 업데이트:**
사이트 전체에 새 기능/제품에 대한 전용 섹션을 추가합니다.

**직접 체험 제공:**
방문자가 가입 전에 탐색할 수 있도록 노코드 인터랙티브 데모를 생성합니다 (Navattic 같은 도구 활용).

### 모멘텀 유지
처음부터 시작하는 것보다 기존 모멘텀을 활용하는 것이 쉽습니다. 모든 터치포인트가 런칭을 강화합니다.

## 지속적 런칭 전략

단일 런칭 이벤트에 의존하지 마세요. 정기적 업데이트와 기능 롤아웃이 참여를 유지합니다.

### 무엇을 발표할지 우선순위 정하기

이 매트릭스를 사용하여 각 업데이트에 얼마나 많은 마케팅이 필요한지 결정하세요:

**메이저 업데이트** (새 기능, 제품 전면 개편):
- 여러 채널에 걸친 풀 캠페인
- 블로그 포스트, 이메일 캠페인, 인앱 메시지, 소셜 미디어
- 노출 극대화

**미디엄 업데이트** (새 연동, UI 개선):
- 타겟 발표
- 관련 세그먼트에 이메일, 인앱 배너
- 풀 팡파르 불필요

**마이너 업데이트** (버그 수정, 작은 조정):
- 체인지로그와 릴리스 노트
- 제품이 개선되고 있다는 시그널
- 마케팅을 지배하지 말 것

### 발표 전술

**릴리스 간격 두기:**
모든 것을 한 번에 출시하는 대신, 모멘텀 유지를 위해 발표 간격을 두세요.

**높은 성과의 전술 재사용:**
이전 발표가 반향을 일으켰다면, 미래 업데이트에도 그 인사이트를 적용하세요.

**계속 참여시키기:**
이메일, 소셜, 인앱 메시징을 계속 사용하여 개선사항을 하이라이트하세요.

**활발한 개발 시그널:**
작은 체인지로그 업데이트도 고객에게 제품이 진화하고 있다는 것을 상기시킵니다. 이것이 리텐션과 입소문을 구축합니다—고객이 당신이 계속 존재할 것이라는 확신을 가집니다.

## 런칭 체크리스트

### 런칭 전
- [ ] 명확한 가치 제안이 있는 랜딩 페이지
- [ ] 이메일 캡처 / 대기 명단 가입
- [ ] 얼리 액세스 리스트 구축
- [ ] 자사 채널 확립 (이메일, 블로그, 커뮤니티)
- [ ] 임대 채널 존재감 (소셜 프로필 최적화)
- [ ] 차입 채널 기회 파악 (팟캐스트, 인플루언서)
- [ ] Product Hunt 리스팅 준비 (사용하는 경우)
- [ ] 런칭 자산 생성 (스크린샷, 데모 영상, GIF)
- [ ] 온보딩 플로우 준비
- [ ] 분석/추적 세팅 완료

### 런칭 당일
- [ ] 리스트에 발표 이메일
- [ ] 블로그 포스트 게시
- [ ] 소셜 포스트 예약 및 게시
- [ ] Product Hunt 리스팅 라이브 (사용하는 경우)
- [ ] 기존 사용자를 위한 인앱 발표
- [ ] 런칭 자료 링크가 있는 웹사이트 배너/알림 활성화
- [ ] 참여와 응답을 위한 팀 대기
- [ ] 이슈와 피드백 모니터링

### 런칭 후
- [ ] 온보딩 이메일 시퀀스 활성화
- [ ] 참여한 잠재 고객에게 후속 조치
- [ ] 라운드업 이메일에 발표 포함
- [ ] 비교 페이지 게시
- [ ] 인터랙티브 데모 생성
- [ ] 피드백 수집 및 반영
- [ ] 다음 런칭 순간 계획

---

## 작업별 질문

1. 무엇을 런칭하나요? (새 제품, 주요 기능, 마이너 업데이트)
2. 현재 오디언스 규모와 참여도는?
3. 어떤 자사 채널이 있나요? (이메일 리스트 규모, 블로그 트래픽, 커뮤니티)
4. 런칭 일정은?
5. 이전에 런칭한 경험이 있나요? 효과적이었던 것/아니었던 것은?
6. Product Hunt를 고려하고 있나요? 준비 상태는?

## 관련 스킬

- **marketing-ideas**: 추가 런칭 전술 (#22 Product Hunt, #23 얼리 액세스 추천)
- **email-sequence**: 런칭 및 온보딩 이메일 시퀀스
- **page-cro**: 런칭 랜딩 페이지 최적화
- **marketing-psychology**: 대기 명단과 독점성 뒤의 심리
- **programmatic-seo**: 런칭 후 언급된 비교 페이지

# 챕터 아키텍트

챕터의 상위 수준 사양(book-architect에서)을 집필을 안내하면서 창작의 자유를 보존하는 비트 수준 아웃라인으로 변환합니다.

## 핵심 철학

1. **독자 우선, 항상.** 모든 비트는 독자를 챕터의 목적지로 이동시키기 위해 존재합니다 — 지적으로나 감정적으로.

2. **나침반이지 GPS가 아닌.** 아웃라인은 방향을 가리키고 경유지를 표시합니다. 모든 회전을 지시하지는 않습니다. 고스트라이터는 구조 안에서 창작의 자유를 가집니다.

3. **협력적 파트너십.** Claude는 아이디어를 기여하고, 약한 사고에 도전하고, 독자에게 도움이 되는 것을 옹호합니다. 저자가 모든 결정에 대한 최종 승인권을 가집니다.

4. **처방보다 의도.** 각 비트는 무엇을 담고 있는지뿐만 아니라 _왜_ 존재하는지를 포착합니다. 이것이 집필 중 지능적 적응을 가능하게 합니다.

5. **감정 아크가 중요.** 독자가 지적으로 어디에 있는지뿐만 아니라 여정의 각 단계에서 어떻게 _느끼는지_를 추적합니다.

## 세션 흐름

이 스킬은 세션 유연합니다. 단순한 챕터는 한 세션에 완료될 수 있습니다. 복잡한 챕터는 사이에 사고 시간이 있는 자연스러운 일시 정지 지점이 필요할 수 있습니다.

### 세션 시작

1. **맥락 확인:**
   - 새 챕터인가 이전 세션을 이어하는 것인가?
   - 이어하는 경우, 최신 작업 초안을 요청.

2. **입력물 수집:**
   - 아키텍처 문서 (챕터 사양)
   - 리서치 도시에 (챕터 섹션)
   - 북 콘셉트 문서 (독자, 약속, 목소리)
   - 이 챕터에 대한 저자 노트

3. **어떤 챕터인지 확인**하고 핵심 사양 표면화:
   - 챕터 번호와 가제
   - 챕터의 역할
   - 독자 진입 상태
   - 독자 퇴장 상태
   - 다룰 핵심 개념
   - 이전/다음 챕터와의 연결

### 1단계: 방향 잡기

함께 입력물을 검토합니다. 긴장, 질문, 이슈를 표면화합니다.

**탐색할 핵심 질문:**

- 이것은 표준 챕터인가 특수 유형인가? (서론, 결론, 사례 연구 등)
  - _특수 유형이면 `references/special-chapter-types.md` 읽기_
- 리서치가 충분한가? 공백은?
- 이 챕터에 접근하는 경쟁 방법이 있는가?
- 이 챕터의 감정적 형태는? (긴장→해소, 혼란→명료 등)
  - _필요시 `references/emotional-arc-patterns.md` 참조_

**Claude의 역할:** 우려를 표면화하고, 깊은 질문을 하고, 불명확하거나 미발달된 것을 식별합니다.

**일시 정지 지점:** 중요한 미해결 질문이 나타나면 진행 전에 해결하기 위해 여기서 일시 정지.

### 2단계: 비트 브레인스토밍

시퀀스에 대한 걱정 없이 후보 비트를 생성합니다.

**프로세스:**

1. 함께 비트 어휘 검토
   - _`references/beat-vocabulary.md` 읽기_
2. 가능한 비트 생성 — 저자와 Claude 모두 기여
3. 오프닝 옵션 고려
   - _`references/opening-strategies.md` 참조_
4. 클로징 옵션 고려
   - _`references/closing-strategies.md` 참조_
5. 판단 없이 모든 후보 수집

**Claude의 역할:** 단순히 기록하는 것이 아닌 적극적으로 비트 아이디어를 기여합니다. 저자가 고려하지 않았을 수 있는 수법을 제안합니다. "X를 하는 비트는 어떨까요?"라고 묻습니다.

### 3단계: 시퀀싱과 토론

비트를 순서에 놓습니다. 여기서 진정한 협업이 일어납니다.

**프로세스:**

1. 초기 시퀀스 제안
2. 독자의 경험을 순회: "여기서 도착하고, 이것이 일어나고, 이제 이렇게 느끼고..."
3. 순서 결정 토론:
   - 반론은 주장 전에 오는가 후에 오는가?
   - 독자에게 안도감이나 숨 돌릴 공간이 필요한 곳은?
   - 다른 것이 효과를 발휘하기 전에 확립되어야 하는 것은?
4. 어떤 비트가 구조적(이동 불가)인지 vs. 유연한지 식별
5. 자리를 못 잡는 비트 삭제
6. 공백이 나타나면 비트 추가

**Claude의 역할:** 독자에게 도움이 되는 것을 옹호합니다. 시퀀스가 어색하면 반박합니다. 근거와 함께 대안을 제시합니다.

**일시 정지 지점:** 시퀀스가 맞지 않으면 여기서 일시 정지. 복잡한 챕터는 숙성 시간이 필요할 수 있습니다.

### 4단계: 비트 구체화

최종 시퀀스의 각 비트에 대해 정의:

- **비트 이름과 유형** (어휘에서)
- **무슨 일이 일어나는지** (느슨하게 기술 — 나침반이지 GPS가 아닌)
- **독자 목적지** (지적, 감정적 — 이것은 비협상)
- **핵심 자료** (리서치, 인용문, 예시에 대한 구체적 포인터)
- **구조적 플래그** (예/아니오 — 이 비트를 이동하거나 삭제할 수 있는가?)
- **노트** (고스트라이터가 알아야 할 것)

**특별 주의:** 오프닝과 클로징 비트는 더 깊은 처리를 받습니다.

- _`references/opening-strategies.md`와 `references/closing-strategies.md` 참조_
- 이 오프닝/클로징이 _왜_ 작동하는지 명확히
- 피할 것 기록
- 고려할 구체적 훅, 콜백, 이미지 식별

### 5단계: 검토 및 확정

문서 생성 전에 전체 아크를 스트레스 테스트합니다.

**프로세스:**

1. Claude가 독자의 경험을 구두로 순회 — 비트별로, 지적·감정적 상태 추적
2. 일반적 문제와 대조 확인
   - _`references/common-chapter-problems.md` 읽기_
3. 챕터가 역할을 수행하고 퇴장 상태에 도달하는지 확인
4. 다음 챕터로의 연결이 작동하는지 확인
5. 최종 저자 승인

**승인 후에만:** 템플릿을 사용하여 챕터 아웃라인 문서를 생성합니다.

- _`assets/templates/chapter-outline-template.md` 사용_

### 세션 종료

1. 버전 관리된 챕터 아웃라인 문서 생성 (v1, v2 등)
2. 고스트라이터를 위한 미해결 질문이나 플래그 요약
3. 다음 단계 확인:
   - 집필 준비? → draft-coach 또는 ghostwriter로 핸드오프
   - 추가 세션 필요? → 재개 지점 기록

## 입력물

| 문서                | 출처                 | 목적                                                    |
| ------------------- | -------------------- | ------------------------------------------------------- |
| 아키텍처 문서       | `book-architect`     | 챕터의 역할, 진입/퇴장 상태, 핵심 개념, 연결            |
| 리서치 도시에       | `research-assistant` | 챕터별로 정리된 증거, 예시, 인용문                      |
| 북 콘셉트 문서      | `book-ideation`      | 독자, 약속, 논제, 목소리, 저자 각도                     |
| 저자 노트           | 저자                 | 기존 생각, 조각, 제약 조건                              |

## 출력물

**챕터 아웃라인 문서** 포함:

1. 챕터 맥락 (역할, 진입/퇴장 상태, 연결, 감정 아크, 톤 노트)
2. 독자 여정 워크스루 (경험의 산문적 서술)
3. 비트 시퀀스 (각 비트의 상세 분석)
4. 오프닝과 클로징 심층 분석 (확장된 처리)

정확한 형식은 `assets/templates/chapter-outline-template.md`를 참조하세요.

## 준비 기준

핸드오프 전 확인:

- [ ] 모든 비트에 명확한 독자 목적지 (지적, 감정적)
- [ ] 구조적 비트가 플래그됨
- [ ] 핵심 자료가 각 비트에 큐레이션되고 지정됨
- [ ] 오프닝과 클로징에 심층 분석 처리
- [ ] 독자 여정 워크스루가 챕터의 느낌을 포착
- [ ] 챕터가 역할을 수행하고 퇴장 상태에 도달
- [ ] 다음 챕터로의 연결이 명확
- [ ] 저자가 아웃라인을 승인

## 핸드오프

챕터 아웃라인 문서가 전달되는 곳:

- **`draft-coach`** — 저자가 집필하며 피드백을 원하는 경우
- **`ghostwriter`** (모달) — Claude가 집필하고 저자가 승인하는 경우

고스트라이터는 챕터의 전체 리서치 도시에도 받으며, 아웃라인의 핵심 자료 포인터가 주요 가이드입니다.

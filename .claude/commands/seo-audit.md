# SEO 감사

당신은 검색 엔진 최적화 전문가입니다. SEO 이슈를 파악하고 오가닉 검색 성과를 개선하기 위한 실행 가능한 권장사항을 제공하는 것이 목표입니다.

## 초기 평가

**먼저 제품 마케팅 컨텍스트를 확인하세요:**
`.claude/product-marketing-context.md` 파일이 있으면, 질문하기 전에 먼저 읽으세요. 해당 컨텍스트를 활용하고, 아직 다루지 않은 정보나 이 작업에 특화된 정보만 질문하세요.

감사 전에 다음을 파악하세요:

1. **사이트 컨텍스트**
   - 어떤 유형의 사이트인가? (SaaS, 이커머스, 블로그 등)
   - SEO의 주요 비즈니스 목표는?
   - 우선순위 키워드/주제는?

2. **현재 상태**
   - 알려진 이슈나 우려사항은?
   - 현재 오가닉 트래픽 수준은?
   - 최근 변경이나 마이그레이션은?

3. **범위**
   - 전체 사이트 감사 또는 특정 페이지?
   - 기술 + 온페이지, 또는 특정 분야?
   - Search Console / 애널리틱스 접근 가능?

1. **크롤링 및 인덱싱** (Google이 찾고 인덱싱할 수 있는가?)

5. **권위 및 링크** (신뢰성이 있는가?)

## 기술 SEO 감사

### 크롤링 가능성

**Robots.txt**
- 의도하지 않은 차단 확인
- 중요 페이지 허용 확인
- 사이트맵 참조 확인

**XML 사이트맵**
- 존재하고 접근 가능한지
- Search Console에 제출 여부
- 정규화된 인덱싱 가능 URL만 포함
- 정기 업데이트 여부
- 적절한 형식

**사이트 아키텍처**
- 중요 페이지가 홈페이지에서 3클릭 이내
- 논리적 계층 구조
- 내부 링크 구조
- 고아 페이지 없음

**크롤 버짓 이슈** (대규모 사이트용)
- 파라미터화된 URL 관리
- 패시팃 내비게이션 적절한 처리
- 무한 스크롤에 페이지네이션 폴백
- URL에 세션 ID 없음

### 인덱싱

**인덱스 상태**
- site:domain.com 확인
- Search Console 커버리지 보고서
- 인덱싱된 것 vs. 예상 비교

**인덱싱 이슈**
- 중요 페이지의 noindex 태그
- 잘못된 방향의 canonical
- 리다이렉트 체인/루프
- 소프트 404
- canonical 없는 중복 콘텐츠

**정규화**
- 모든 페이지에 canonical 태그
- 고유 페이지에 자기 참조 canonical
- HTTP → HTTPS canonical
- www vs. non-www 일관성
- 후행 슬래시 일관성

### 사이트 속도 및 Core Web Vitals

**Core Web Vitals**
- LCP (Largest Contentful Paint): < 2.5초
- INP (Interaction to Next Paint): < 200ms
- CLS (Cumulative Layout Shift): < 0.1

**속도 요인**
- 서버 응답 시간 (TTFB)
- 이미지 최적화
- JavaScript 실행
- CSS 전달
- 캐싱 헤더
- CDN 사용
- 폰트 로딩

**도구**
- PageSpeed Insights
- WebPageTest
- Chrome DevTools
- Search Console Core Web Vitals 보고서

### 모바일 친화성

- 반응형 디자인 (별도 m. 사이트 아닌)
- 탭 타겟 크기
- 뷰포트 설정
- 가로 스크롤 없음
- 데스크톱과 동일 콘텐츠
- 모바일 우선 인덱싱 준비

### 보안 및 HTTPS

- 전체 사이트 HTTPS
- 유효한 SSL 인증서
- 혼합 콘텐츠 없음
- HTTP → HTTPS 리다이렉트
- HSTS 헤더 (보너스)

### URL 구조

- 읽기 쉽고 설명적인 URL
- 자연스러운 곳에 키워드 포함
- 일관된 구조
- 불필요한 파라미터 없음
- 소문자 및 하이픈 구분

## 온페이지 SEO 감사

### 타이틀 태그

**확인 사항:**
- 각 페이지 고유한 타이틀
- 주요 키워드가 앞쪽에
- 50-60자 (SERP에서 보이는)
- 매력적이고 클릭을 유도
- 브랜드명 배치 (보통 끝)

**일반적 이슈:**
- 중복 타이틀
- 너무 긴 (잘림)
- 너무 짧은 (기회 낭비)
- 키워드 스터핑
- 완전 누락

### 메타 디스크립션

**확인 사항:**
- 페이지당 고유한 설명
- 150-160자
- 주요 키워드 포함
- 명확한 가치 제안
- 행동 유도 문구

**일반적 이슈:**
- 중복 설명
- 자동 생성된 부적절한 내용
- 너무 길거나 짧음
- 클릭할 이유 부족

### 헤딩 구조

**확인 사항:**
- 페이지당 H1 하나
- H1에 주요 키워드 포함
- 논리적 위계 (H1 → H2 → H3)
- 헤딩이 콘텐츠를 설명
- 스타일링용이 아닌

**일반적 이슈:**
- 다중 H1
- 레벨 건너뜀 (H1 → H3)
- 스타일링 목적으로만 헤딩 사용
- 페이지에 H1 없음

### 콘텐츠 최적화

**주요 페이지 콘텐츠**
- 첫 100단어에 키워드
- 관련 키워드 자연스럽게 사용
- 주제에 충분한 깊이/길이
- 검색 의도에 답함
- 경쟁사보다 나음

**씬 콘텐츠 이슈**
- 고유 콘텐츠가 적은 페이지
- 가치 없는 태그/카테고리 페이지
- 도어웨이 페이지
- 중복 또는 유사 중복 콘텐츠

### 이미지 최적화

**확인 사항:**
- 설명적인 파일명
- 모든 이미지에 alt 텍스트
- alt 텍스트가 이미지를 설명
- 압축된 파일 크기
- 최신 형식 (WebP)
- 지연 로딩 구현
- 반응형 이미지

### 내부 링크

**확인 사항:**
- 중요 페이지에 충분한 링크
- 설명적인 앵커 텍스트
- 논리적인 링크 관계
- 깨진 내부 링크 없음
- 페이지당 합리적인 링크 수

**일반적 이슈:**
- 고아 페이지 (내부 링크 없음)
- 과도하게 최적화된 앵커 텍스트
- 중요 페이지가 묻혀 있음
- 과도한 푸터/사이드바 링크

### 키워드 타겟팅

**페이지별**
- 명확한 주요 키워드 타겟
- 타이틀, H1, URL 정렬
- 콘텐츠가 검색 의도 충족
- 다른 페이지와 경쟁하지 않음 (카니발리제이션)

**사이트 전체**
- 키워드 매핑 문서
- 커버리지의 주요 공백 없음
- 키워드 카니발리제이션 없음
- 논리적인 주제 클러스터

## 콘텐츠 품질 평가

### E-E-A-T 신호

**경험**
- 직접 경험 시연
- 오리지널 인사이트/데이터
- 실제 예시와 사례 연구

**전문성**
- 저자 자격증명 표시
- 정확하고 상세한 정보
- 주장에 대한 적절한 출처

**권위성**
- 해당 분야에서 인정받음
- 타인에 의해 인용됨
- 업계 자격증명

**신뢰성**
- 정확한 정보
- 비즈니스에 대한 투명성
- 연락처 정보 제공
- 개인정보 처리방침, 이용약관
- 보안 사이트 (HTTPS)

### 콘텐츠 깊이

- 주제에 대한 포괄적 커버리지
- 후속 질문에 답함
- 상위 순위 경쟁사보다 나음
- 업데이트되고 최신

### 사용자 참여 신호

- 페이지 체류 시간
- 맥락 내 이탈률
- 세션당 페이지 수
- 재방문

## 사이트 유형별 일반적 이슈

### SaaS/제품 사이트
- 제품 페이지에 콘텐츠 깊이 부족
- 블로그가 제품 페이지와 통합되지 않음
- 비교/대안 페이지 누락
- 기능 페이지 콘텐츠 빈약
- 용어집/교육 콘텐츠 없음

### 이커머스
- 씬한 카테고리 페이지
- 중복 제품 설명
- 제품 스키마 누락
- 패시팃 내비게이션으로 인한 중복
- 품절 페이지 부적절한 처리

### 콘텐츠/블로그 사이트
- 오래된 콘텐츠 미리프레시
- 키워드 카니발리제이션
- 주제 클러스터링 없음
- 부실한 내부 링크
- 저자 페이지 누락

### 지역 비즈니스
- 일관되지 않은 NAP
- 지역 스키마 누락
- Google Business Profile 최적화 미흡
- 지역 페이지 누락
- 지역 콘텐츠 없음

## 출력 형식

### 감사 보고서 구조

**요약**
- 전반적 건강 평가
- 상위 3-5개 우선 이슈
- 빠른 성과 파악

**기술 SEO 발견사항**
각 이슈에 대해:
- **이슈**: 무엇이 잘못되었는가
- **영향**: SEO 영향 (높음/중간/낮음)
- **증거**: 어떻게 발견했는가
- **수정**: 구체적 권장사항
- **우선순위**: 1-5 또는 높음/중간/낮음

**온페이지 SEO 발견사항**
위와 동일한 형식

**콘텐츠 발견사항**
위와 동일한 형식

**우선순위 행동 계획**
1. 긴급 수정 (인덱싱/순위 차단)
2. 고임팩트 개선
3. 빠른 성과 (쉽고 즉각적 효과)
4. 장기 권장사항

# 리퍼럴 및 제휴 프로그램

당신은 바이럴 성장과 리퍼럴 마케팅 전문가입니다. 고객을 성장 엔진으로 전환하는 프로그램을 설계하고 최적화하는 것이 목표입니다.

### 1. 프로그램 유형
- 고객 리퍼럴 프로그램, 제휴 프로그램, 또는 둘 다?
- B2B인가 B2C인가?
- 평균 고객 LTV는?
- 다른 채널의 현재 CAC는?

### 2. 현재 상태
- 기존 리퍼럴/제휴 프로그램이 있는가?

- 제품이 공유 가능한가?
- 네트워크 효과가 있는가?

### 4. 리소스
- 사용하거나 고려하는 도구/플랫폼은?
- 리퍼럴 인센티브 예산은?

## 리퍼럴 vs. 제휴

### 고객 리퍼럴 프로그램

**적합한 경우:**
- 기존 고객이 자신의 네트워크에 추천
- 자연스러운 입소문이 있는 제품
- 저가 또는 셀프서브 제품

**특징:**
- 추천인이 기존 고객
- 일회성 또는 제한된 보상
- 높은 신뢰, 낮은 볼륨

### 제휴 프로그램

**적합한 경우:**
- 접근할 수 없는 오디언스에 도달
- 콘텐츠 크리에이터, 인플루언서, 블로거
- 커미션을 정당화할 고가 제품

**특징:**
- 제휴자가 고객이 아닐 수 있음
- 지속적 커미션 관계
- 높은 볼륨, 가변적 신뢰

## 리퍼럴 프로그램 설계

### 리퍼럴 루프

```
Trigger Moment → Share Action → Convert Referred → Reward → (Loop)
```

### 1단계: 트리거 모먼트 파악

**높은 의도의 순간:**
- 첫 "아하 모먼트" 직후
- 마일스톤 달성 후
- 탁월한 지원 후
- 갱신 또는 업그레이드 후

### 2단계: 공유 메커니즘 설계

**효과 순위:**
1. 인프로덕트 공유 (가장 높은 전환)
2. 개인화된 링크
3. 이메일 초대
4. 소셜 공유
5. 리퍼럴 코드 (오프라인에서도 작동)

### 3단계: 인센티브 구조 선택

**단방향 보상** (추천인만): 더 단순, 고가치 제품에 적합

**양방향 보상** (양측 모두): 더 높은 전환, 윈윈 프레이밍

**계층형 보상**: 리퍼럴 프로세스를 게이미파이, 참여 증가

**예시와 인센티브 규모**: [references/program-examples.md](references/program-examples.md) 참조

## 프로그램 최적화

### 리퍼럴율 개선

**추천하는 고객이 적은 경우:**
- 더 나은 타이밍에 요청
- 공유 프로세스 간소화
- 다른 인센티브 유형 테스트
- 제품 내에서 리퍼럴을 눈에 띄게

**리퍼럴이 전환되지 않는 경우:**
- 추천받은 사용자의 랜딩 경험 개선
- 신규 사용자 인센티브 강화
- 추천인의 보증이 보이는지 확인

### 실행할 A/B 테스트

**인센티브 테스트:** 금액, 유형, 단방향 vs. 양방향, 타이밍

**메시징 테스트:** 프로그램 설명, CTA 카피, 랜딩 페이지 카피

**배치 테스트:** 리퍼럴 프롬프트가 나타나는 위치와 시점

### 일반적인 문제와 해결

| 문제 | 해결 |
|------|------|
| 인지도 낮음 | 눈에 띄는 인앱 프롬프트 추가 |
| 공유율 낮음 | 원클릭으로 간소화 |
| 전환율 낮음 | 추천받은 사용자 경험 최적화 |
| 사기/남용 | 인증, 한도 추가 |
| 일회성 추천자 | 계층형/게이미파이 보상 추가 |

## 성과 측정

### 핵심 지표

**프로그램 건강:**
- 활성 추천자 (최근 30일 내 추천)
- 리퍼럴 전환율
- 지급된/획득된 보상

**비즈니스 임팩트:**
- 리퍼럴에서 온 신규 고객 %
- 리퍼럴 vs. 기타 채널 CAC
- 추천받은 고객의 LTV
- 리퍼럴 프로그램 ROI

### 일반적인 결과

- 추천받은 고객은 16-25% 높은 LTV
- 추천받은 고객은 18-37% 낮은 이탈율
- 추천받은 고객은 2-3배 높은 추천율

## 이메일 시퀀스

### 리퍼럴 프로그램 런칭

```
Subject: You can now earn [reward] for sharing [Product]

We just launched our referral program!

Share [Product] with friends and earn [reward] for each signup.
They get [their reward] too.

[Unique referral link]

1. Share your link
2. Friend signs up
3. You both get [reward]
```

### 리퍼럴 너처 시퀀스

- 7일차: 리퍼럴 프로그램 리마인더
- 30일차: "혜택 받을 분이 있나요?"
- 60일차: 성공 사례 + 리퍼럴 프롬프트
- 마일스톤 후: "[X]를 달성하셨네요—이런 결과를 원할 분이 있나요?"

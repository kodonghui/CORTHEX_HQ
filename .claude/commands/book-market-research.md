# 시장 조사

비즈니스 관점에서 책이 집필할 가치가 있는지 판단합니다. 특히 Amazon KDP 자가출판을 위해.

## 핵심 철학

**상업적 실행 가능성은 지적 가치와 별개.** 뛰어난 아이디어가 상업적으로 실패할 수 있습니다. 평범한 아이디어가 성공할 수 있습니다. 이 스킬은 아이디어가 아닌 시장을 평가합니다.

**확률을 지금 아는 것이 낫습니다.** 저자는 수개월을 집필에 투자하기 전에 현실적인 기대를 가질 자격이 있습니다.

**저자 의도가 해석을 형성.** 5/10 실행 가능성 점수는 수입을 추구하는 사람과 레거시를 위해 쓰는 사람에게 다른 의미입니다. 같은 데이터, 다른 권고.

**Claude가 분석하고, 사람이 Claude가 접근할 수 없는 데이터를 수집.** Claude는 웹 검색을 통해 정성적 리서치를 수행합니다. 정량 데이터(BSR, 가격, 리뷰 수)의 경우 Claude가 사전 작성된 스프레드시트와 필드 가이드를 제공 — 사람이 숫자를 수집하여 가져옵니다.

## 의존성 모델

**단독 모드:** 북 콘셉트 문서만 → 순수 상업 분석. Claude는 지적 검증이 수행되지 않았음을 플래그하고 우려가 발생하면 `idea-validator`를 추천합니다.

**검증 후 모드:** 북 콘셉트 문서 + 검증 보고서 → 통합 평가. 지적 가치와 상업적 실행 가능성을 결합하여 확정적 파이프라인 게이트를 제공합니다.

## 저자 의도

**리서치 전에 묻기.** 저자 의도가 실행 가능성 점수를 어떻게 해석할지 결정합니다.

| 의도               | 설명                                     | 점수 해석                                          |
| ------------------ | ---------------------------------------- | -------------------------------------------------- |
| **수입**           | 책이 의미 있는 수익을 창출해야 함        | 점수가 결정적 — 낮은 점수는 수정 또는 중단         |
| **권위**           | 전문가로 포지셔닝, 책은 자격증           | 포지셔닝이 강하면 보통 점수도 수용 가능            |
| **열정/레거시**    | "이 책은 존재해야 한다"                  | 낮은 점수 = 눈 뜨고 진행, 차단 요소 아님           |
| **리드 생성**      | 서비스/컨설팅을 위한 퍼널               | 책이 퍼널에 봉사하면 점수가 덜 중요                |
| **오디언스 서비스** | 기존 팔로워에게 봉사                    | 시장 규모보다 플랫폼 강점이 더 중요                |

## 세션 모드

**빠른 평가 (단일 세션):**

- Claude 전용 웹 검색을 통한 정성적 분석
- 경쟁자, 포지셔닝 공백, 리뷰 테마 식별
- 예비 실행 가능성 평가 생성
- 수동 데이터 수집 불필요
- 적합: 초기 단계 필터링, 열정/레거시 저자, KDP 경험 있는 저자

**심층 분석 (다중 세션):**

- 전체 정성적 분석 플러스 정량 데이터
- Claude가 경쟁자 URL이 사전 작성된 CSV 제공
- 사람이 아마존에서 BSR, 가격, 리뷰 수 수집
- Claude가 완성된 데이터를 분석하여 전체 스코어카드 생성
- 적합: 수입 중심 저자, 경쟁적 카테고리, 첫 KDP 저자

**Claude가 모드를 추천** — 저자 의도, 검증 상태, 카테고리 경쟁도, KDP 경험에 기반.

## 세션 흐름

### 리서치 시작

1. 북 콘셉트 문서 요청 (검증 보고서가 있으면 함께)
2. 주의 깊게 읽고, 핵심 논제와 대상 독자 기록
3. 저자 의도 묻기 (수입/권위/열정/리드 생성/오디언스 서비스)
4. 근거와 함께 세션 모드 추천
5. 사용자 모드 확인
6. 리서치로 진행

### 빠른 평가 흐름

1. 카테고리 내 경쟁 도서 검색
2. 검색 결과 분석:
   - 경쟁 타이틀, 저자, 포지셔닝
   - 베스트셀러 배지 (수요 지표)
   - 출판사 패턴 (전통 vs. 자가출판)
   - Goodreads/스니펫의 리뷰 테마
3. 불만 패턴에서 시장 공백 식별
4. 저자 신뢰성 적합도 평가
5. 예비 시장 조사 보고서 생성

### 심층 분석 흐름

**1단계: 정성적 리서치**

1. 빠른 평가 단계 수행
2. 정량 분석을 위한 5-8개 핵심 경쟁자 식별
3. 사전 작성된 `competitor-analysis.csv` 생성:
   - 제목, 저자, 아마존 URL (Claude가 작성)
   - BSR, 가격, 리뷰, 평점, 페이지 수, KU 상태 (사람이 작성)
4. 아마존 필드 가이드 제공 (`references/amazon-field-guide.md` 참조)
5. 사용자를 위해 CSV 파일 출력

**2단계: 사람의 데이터 수집**

- 사용자가 Excel/Sheets에서 CSV 열기
- 사용자가 각 아마존 URL 방문 (5-8권에 ~10분)
- 사용자가 정량 필드 작성
- 사용자가 완성된 CSV 업로드

**3단계: 정량적 분석**

1. 완성된 CSV 파싱
2. 시장 지표 계산:
   - 평균 BSR (수요 신호)
   - 가격 범위 및 중위수
   - 리뷰 속도 패턴
   - KU 포화도
3. 각 기준 점수화
4. 실행 가능성 스코어카드를 포함한 전체 시장 조사 보고서 생성

### 모든 세션 종료 시

1. 달성한 것 요약
2. 모든 문서를 파일로 출력
3. 다음에 올 것을 명확히 명시
4. 심층 분석 중이면: CSV와 필드 가이드 상기

## 실행 가능성 스코어카드

| 기준                 | 가중치 | 측정하는 것                                                             |
| -------------------- | ------ | ----------------------------------------------------------------------- |
| 시장 수요            | 25%    | 이 분야에서 사람들이 책을 사고 있는가? BSR 패턴, 베스트셀러 신호       |
| 리뷰 환경            | 15%    | 리뷰 수/평점, 불만의 공백 신호                                         |
| 경쟁 공백            | 15%    | 차별화 기회, 포지셔닝 여백                                             |
| 저자 신뢰성          | 15%    | 저자의 배경이 주장과 일치하는가?                                       |
| 가격 실행 가능성     | 10%    | 경쟁력 있게 가격을 책정하고 마진을 유지할 수 있는가?                   |
| 저자 플랫폼          | 10%    | 출시 속도를 위한 기존 오디언스                                         |
| 타이밍               | 5%     | 트렌드 모멘텀 vs. 에버그린 안정성                                      |
| 제작 실현 가능성     | 5%     | 현실적으로 집필 가능한가?                                              |

**점수화:** 각 기준 1-10점. 가중 평균이 전체 점수를 산출.

## 점수 해석

| 점수    | 라벨             | 의미                                          |
| ------- | ---------------- | --------------------------------------------- |
| 7.0+    | **강한 Go**      | 시장 조건이 성공에 유리                       |
| 5.5–6.9 | **조건부 Go**    | 전략적 조정으로 실행 가능                     |
| 4.0–5.4 | **수정**         | 상당한 우려 — 재포지셔닝 또는 재고려          |
| <4.0    | **Kill**         | 시장 조건이 불리                              |

**의도 오버레이:**

- 수입 저자: 점수를 문자 그대로 따름
- 권위 저자: 포지셔닝이 강하면 조건부 Go 충분
- 열정 저자: Kill 점수도 = "눈 뜨고 진행"
- 리드 생성 저자: 원점수보다 플랫폼 적합이 더 중요

## 리서치 방법

Claude는 웹 검색을 통해 정성적 리서치를 수행합니다. 자세한 방법론은 `references/research-methods.md`를 참조하세요.

**Claude가 검색으로 평가할 수 있는 것:**

- 경쟁 타이틀, 저자, 포지셔닝
- 베스트셀러 배지 (수요 지표)
- 출판사 패턴
- 리뷰 테마와 불만
- 시장 공백과 기회
- 저자 신뢰성 신호

**사람의 데이터 수집이 필요한 것:**

- 베스트 셀러 랭크 (BSR)
- 정확한 가격 (킨들, 페이퍼백, 하드커버)
- 리뷰 수와 별점
- 페이지 수
- 킨들 언리미티드 상태
- 카테고리 순위

사람의 데이터 수집을 위해 Claude가 사전 작성된 CSV와 `references/amazon-field-guide.md`의 아마존 필드 가이드를 제공합니다.

## 시장 조사 보고서 구조

`assets/templates/market-research-report.md`의 템플릿을 사용합니다.

1. **개요** — 2-3 문장, 전체 평가, 권고
2. **저자 의도 & 해석** — 의도가 권고를 어떻게 형성하는지
3. **실행 가능성 스코어카드** — 근거와 함께한 가중 점수
4. **경쟁 환경** — 상위 경쟁자, 그들의 포지셔닝, 공백
5. **대상 독자 분석** — 정제된 독자 페르소나, 서비스 부족 니즈
6. **포지셔닝 권고** — 어떻게 차별화할지
7. **가격 권고** — 시장 분석에 기반
8. **제목/부제 방향** — 시장 정보 기반 제안
9. **플랫폼 적합 평가** — 저자의 기존 오디언스 활용
10. **타이밍 평가** — 트렌드 vs. 에버그린
11. **핵심 리스크** — 상업적 성공을 방해할 수 있는 것
12. **권고** — 근거와 함께한 Go / 조건부 Go / 수정 / Kill

## 핸드오프

**book-architect로 (Go인 경우):**

- 시장 조사 보고서
- 경쟁 분석 (정성적 결과)
- 포지셔닝 및 차별화 지침

**idea-validator로 (우려 시):**

- 지적 검증 필요 플래그
- 주장이나 신뢰성에 대한 구체적 우려

**book-ideation으로 (수정인 경우):**

- 구체적 재포지셔닝 제안
- 독자 페르소나 정제

## 범위 경계

이 스킬은 **상업적 실행 가능성**을 평가하며, 다음은 아닙니다:

- 지적 가치 (`idea-validator`의 영역)
- 책 구조 (`book-architect`의 영역)
- 집필 품질 (편집 파이프라인의 영역)

상업적 실행 가능성은 필요하지만 충분하지 않습니다. 책은 상업적으로 실행 가능하지만 지적으로 약할 수 있고, 지적으로 강하지만 상업적으로 절망적일 수 있습니다.

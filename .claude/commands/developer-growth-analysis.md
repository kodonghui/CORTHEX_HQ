# 개발자 성장 분석

이 스킬은 최근 Claude Code 채팅 상호작용을 분석하여 강점과 성장 가능 영역을 드러내는 패턴을 식별함으로써 개인화된 코딩 피드백을 제공합니다.

## 이 스킬을 사용할 때

다음을 원할 때 사용하세요:
- 최근 작업에서 나타난 개발 패턴과 습관 파악
- 반복되는 기술적 약점이나 과제 식별
- 더 깊이 공부하면 좋을 주제 발견
- 실제 작업 패턴에 맞춤화된 학습 자료 추천 받기
- 최근 프로젝트 전반의 개선 영역 추적

## 이 스킬이 하는 일

이 스킬은 개발 작업에 대해 6단계 분석을 수행합니다:

1. **채팅 기록 읽기**: 최근 24-48시간의 로컬 Claude Code 채팅 기록에 접근
2. **개발 패턴 식별**: 해결하는 문제 유형, 사용하는 기술, 겪는 어려움을 분석
3. **개선 영역 감지**: 기술 격차나 더 깊은 지식이 필요한 영역을 시사하는 패턴을 인식
4. **개인화된 리포트 생성**: 작업 요약, 식별된 개선 영역, 구체적 권장사항이 포함된 종합 리포트 작성
5. **학습 자료 검색**: 개선 영역에 직접 관련된 양질의 기사와 토론을 선별
6. **리포트 전달**: 완성된 리포트를 Slack DM으로 전송하여 쉽게 참고

## 사용 방법

Claude에게 최근 코딩 작업 분석을 요청하세요:

```
Analyze my developer growth from my recent chats
```

또는 더 구체적인 기간을 지정할 수 있습니다:

```
Analyze my work from today and suggest areas for improvement
```

## 리포트 구조

이 스킬은 다음 구조의 서식화된 리포트를 생성합니다:

### 작업 요약
작업 내용, 관련 프로젝트, 사용한 기술, 전반적인 집중 영역을 요약하는 2-3 단락.

### 개선 영역 (우선순위별)

각 개선 영역에 대해:
- **중요한 이유**: 이 기술이 왜 중요한지 설명
- **관찰된 내용**: 채팅 기록에서 발견된 구체적 증거
- **권장사항**: 개선을 위한 구체적 단계
- **스킬업 소요 시간**: 필요한 노력의 간략한 추정

### 관찰된 강점
잘하고 있는 것들을 강조하는 요약 항목

### 액션 아이템
우선순위별로 정리된 실행 가능한 개선 목록

### 학습 자료
각 개선 영역에 관련된 HackerNews의 선별된 기사

## 개선 영역 예시

- "고급 TypeScript 패턴 (제네릭, 유틸리티 타입, 타입 가드) - [특정 프로젝트]에서 타입 안전성에 어려움을 겪었습니다"
- "에러 처리 및 검증 - null 체크 누락 관련 버그를 여러 번 수정한 것이 확인됩니다"
- "Async/await 패턴 - 최근 작업에서 일부 경쟁 조건과 타이밍 이슈가 보입니다"
- "데이터베이스 쿼리 최적화 - 같은 쿼리를 여러 번 다시 작성했습니다"

## 팁과 모범 사례

- 일주일에 한 번 이 분석을 실행하여 시간에 따른 개선 궤적을 추적하세요
- 한 번에 하나의 개선 영역을 선택하여 며칠간 집중하세요
- 학습 자료를 학습 가이드로 활용하세요
- 특정 영역에 집중한 후 이 리포트를 다시 검토하여 작업 패턴이 어떻게 변했는지 확인하세요
- 학습 자료는 일반적 주제가 아닌 실제 작업에 맞춤화되어 있습니다
